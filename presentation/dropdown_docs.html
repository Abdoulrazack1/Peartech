Comprendre
dropdown.js

Ce fichier crée le menu déroulant des catégories dans la navigation. Il génère son HTML dynamiquement depuis la base de données, gère l'ouverture/fermeture, et communique les clics aux autres scripts via des événements personnalisés.

Menu de navigation généré entièrement en JavaScript ¶

dropdown.js — le seul composant de navigation généré à 100% en JS
Contrairement aux menus HTML statiques
Pas de <ul><li> dans le HTML
Pas de données hardcodées
Dépend de PearTechDB (attente async)
Avantages du menu dynamique
Catégories lues depuis data.js
Ajouter une catégorie = juste data.js
Émet categorySelected filtre catalogue

dropdown.js construit le menu des catégories au moment où la page se charge, l'insère dans la navigation, puis gère tous ses comportements : ouvrir au clic, fermer en cliquant ailleurs ou avec Échap, et notifier les autres scripts quand une catégorie est choisie.

Analogie : C'est comme un régisseur de théâtre. Au lever du rideau, il construit le décor (génère le menu). Il installe les mécanismes d'ouverture/fermeture (les événements). Et quand un acteur entre en scène (clic sur une catégorie), il crie l'info en coulisses (CustomEvent) pour que les autres régisseurs réagissent.

Vocabulaire essentiel — lisez ça en premier

Termes utilisés dans ce fichier
CustomEvent
Un événement inventé par le développeur — pas un clic ou un keydown standard. new CustomEvent('nom', {detail: données}) crée un événement sur mesure qu'on peut envoyer à qui on veut et qui transporte des données.
dispatchEvent
La méthode qui "lance" un événement. document.dispatchEvent(event) envoie l'événement sur tout le document. Tous les scripts qui écoutent ce nom d'événement le reçoivent.
createElement
document.createElement('div') crée un élément HTML en mémoire — il n'est pas encore visible dans la page. On le configure, puis on l'insère avec appendChild ou insertBefore.
insertBefore
parent.insertBefore(nouveau, reference) insère nouveau juste AVANT l'élément reference dans le parent. Permet de positionner précisément un élément.
e.stopPropagation()
Arrête la propagation d'un événement. Sans ça, un clic sur le bouton dropdown remonterait jusqu'au document — qui fermerait immédiatement le menu qu'on vient d'ouvrir.
PearTechDBReady
Événement personnalisé émis par data.js quand la base de données finit de se charger. Permet à dropdown.js d'attendre la DB sans boucle de réessai (contrairement à init() dans product_grid.js).
classList.contains
element.classList.contains('active') retourne true si l'élément a la classe "active", false sinon. Utilisé pour savoir si le menu est ouvert ou fermé sans regarder le CSS.

02 — Attendre la base de données

Deux stratégies d'attente : réessai vs événement ¶

product_grid.js et dropdown.js doivent tous deux attendre PearTechDB, mais ils utilisent deux approches différentes. Comprendre pourquoi éclaire beaucoup sur la communication entre scripts.

Deux stratégies d'attente — polling vs événement
Stratégie A — polling (setInterval)
attempts = 0
setInterval(100ms)
window.PearTechDB ?
oui clearInterval init()
non attempts++
attempts > 50 abandon (5s)
Fonctionne même si data.js n'émet pas d'événement
Stratégie B — événement dataReady
document.addEventListener(
'dataReady', () => {
clearInterval(poller)
init()
})
Réaction immédiate si data.js émet l'événement
Les deux écouteurs coexistent. Qui déclenche init() en premier l'autre ne fait rien (clearInterval déjà appelé).
Réessai (product_grid) vs Événement (dropdown) — comparaison

Vérifie jusqu'à 50 fois même si la DB est rapide

Les deux approches sont valides. Le polling est plus autonome mais moins élégant. L'événement est plus précis et coopératif. En pratique, dropdown.js vérifie d'abord si la DB est déjà là (if (window.PearTechDB)) — si oui, pas besoin d'attendre.

Comment data.js émet l'événement

javascript — dans data.js (fichier non documenté ici)
// data.js, après avoir initialisé PearTechDB :
const event = new CustomEvent('PearTechDBReady');
document.dispatchEvent(event);
// Tout script qui fait document.addEventListener('PearTechDBReady', fn)
// reçoit cet événement à ce moment précis
CustomEvent vs Event : new Event('nom') crée un événement simple sans données. new CustomEvent('nom', {detail: {...}}) crée un événement qui transporte des données dans sa propriété .detail. Pour 'PearTechDBReady', pas besoin de données — on veut juste signaler que la DB est prête. Pour 'categorySelected' (plus loin), on transporte la catégorie choisie.

03 — Génération du menu

createDropdownMenu() — construire le HTML dynamiquement ¶

Au lieu d'un menu écrit en dur dans le HTML, dropdown.js le fabrique à la volée depuis les données de PearTechDB. Si la DB est indisponible, il utilise des données statiques de secours.

Analogie : C'est comme un menuisier qui reçoit un plan (les catégories de la DB) et fabrique une armoire sur mesure. Si le plan est perdu, il utilise un modèle standard qu'il connaît de mémoire (les catégories statiques). Dans tous les cas, le résultat est une armoire fonctionnelle.

04 — Gestion des événements

attachDropdownEvents() — trois façons de fermer le menu ¶

Trois comportements sont nécessaires pour un menu déroulant accessible : clic sur le bouton, clic en dehors, et touche Échap.

Les 3 événements installés
Clic sur .nav-dropdown
Toggle — ouvre si fermé, ferme si ouvert. e.stopPropagation() pour ne pas propager au document.
Clic sur document
Si le clic est hors du menu ET hors du bouton ferme. Même logique que recherche.js avec .contains().
Touche Échap
Ferme seulement si le menu est .active. La vérification évite d'appeler closeDropdown() inutilement.

Le problème de propagation — pourquoi stopPropagation ?

Ce qui se passe SANS stopPropagation lors d'un clic sur le bouton
Clic sur .nav-dropdown
l'événement remonte (propagation)
Handler du bouton : toggleDropdown() OUVRE le menu
le même clic continue de remonter jusqu'au document
Handler du document : menu.contains(e.target)? NON (e.target = le bouton, pas le menu) FERME
Résultat : menu ouvert puis immédiatement fermé INVISIBLE
Avec e.stopPropagation() dans le handler du bouton, le clic s'arrête là. Le handler document ne le reçoit jamais. Le menu s'ouvre et reste ouvert.
javascript
function attachDropdownEvents() {
const dropdownMenu = document.querySelector('.dropdown-menu');
if (!dropdownMenu) return;
dropdownButton.addEventListener('click', function(e) {
e.stopPropagation();   //  CRUCIAL : bloque la propagation vers document toggleDropdown();
});
document.addEventListener('click', function(e) {
if (!dropdownMenu.contains(e.target) && !dropdownButton.contains(e.target)) {
closeDropdown();
}
});
document.addEventListener('keydown', function(e) {
if (e.key === 'Escape' && dropdownMenu.classList.contains('active')) {
closeDropdown();
}
});
}

Les 3 façons de fermer le menu — et pourquoi les 3 sont nécessaires

javascript
// 1. Clic sur le bouton toggle (ouvre ET ferme)
dropdownBtn.addEventListener('click', function(e) {
    e.stopPropagation(); // Empêche la propagation vers document
    toggleDropdown();
});

// 2. Clic en dehors du dropdown
document.addEventListener('click', function(e) {
    if (!dropdown.contains(e.target)) closeDropdown();
});

// 3. Touche Échap
document.addEventListener('keydown', function(e) {
    if (e.key === 'Escape') closeDropdown();
});

e.stopPropagation() — Interrompre la propagation

Sans e.stopPropagation() sur le bouton, un clic sur le bouton déclencherait deux handlers dans l'ordre : 1. le listener du bouton ouvre le dropdown. 2. l'événement remonte jusqu'à document — le listener 'clic dehors' voit que l'événement ne vient pas du dropdown (il vient du bouton qui est hors du menu), et le referme immédiatement. Le menu s'ouvre et se referme instantanément — l'utilisateur ne le voit jamais.

dropdown.contains(e.target) — Détecter un clic intérieur

dropdown.contains(e.target) retourne true si l'élément cliqué est le dropdown ou l'un de ses descendants. Un clic sur un item du menu, sur la scrollbar interne, ou sur le titre de catégorie — tous retournent true et ne ferment pas le menu. Seul un clic strictement à l'extérieur ferme le menu.

✓
Les 3 mécanismes de fermeture sont complémentaires : le bouton pour la fermeture intentionnelle, le clic dehors pour la fermeture contextuelle (l'utilisateur clique ailleurs sur la page), et Échap pour la fermeture clavier (accessibilité et habitude utilisateur des interfaces modales).

Les 3 façons de fermer le dropdown

Clic boutondropdownBtn.addEventListener('click', toggleDropdown)
Ouvre si fermé, ferme si ouvert
Clic extérieurdocument.addEventListener('click', e => !dropdown.contains(e.target) && closeDropdown())
Détecte les clics hors du composant
Touche Échapdocument.addEventListener('keydown', e => e.key === 'Escape' && closeDropdown())
Convention clavier universelle
🔍
dropdown.contains(e.target) vérifie si le clic a eu lieu à l'intérieur du composant dropdown (bouton OU menu). Si true, on ne ferme pas — l'utilisateur interagit avec le menu. Si false, le clic est en dehors → fermeture. Sans ce check, cliquer sur un item fermerait le menu avant la navigation.

06 — Communication via CustomEvent

categorySelected — notifier les autres scripts ¶

Quand l'utilisateur clique sur une catégorie du menu, dropdown.js émet un événement personnalisé contenant les données de la catégorie. D'autres scripts peuvent l'écouter et réagir — sans couplage direct entre eux.

Analogie : C'est comme une radio. dropdown.js est la station émettrice : quand quelqu'un choisit une catégorie, il diffuse l'info sur le canal "categorySelected". D'autres scripts (par exemple un script de filtrage) sont des récepteurs branchés sur ce canal. Ils réagissent sans que dropdown.js sache qu'ils existent.
javascript
function attachDropdownItemEvents() {
document.querySelectorAll('.dropdown-item').forEach(function(item) {
item.addEventListener('click', function(e) {
const category = this.getAttribute('data-category');
// this.getAttribute() = lit un attribut HTML
// this.dataset.category aurait aussi fonctionné const event = new CustomEvent('categorySelected', {
detail: {
category: category,  // ex: "Apple" link:     this.href    // ex: "page_catalogue.html?categorie=apple" }
});
document.dispatchEvent(event);
// Envoie l'événement sur tout le document.
// N'importe quel script faisant document.addEventListener('categorySelected', fn)
// recevra cet événement avec les données dans event.detail closeDropdown();
// Ferme le menu après le clic.
// La navigation se fait via le href du lien (comportement natif du <a>).
// Pas besoin de window.location.href ici — le navigateur le fait seul. });
});
}

Pourquoi un CustomEvent plutôt qu'un appel direct

Quand l'utilisateur clique sur une catégorie dans le dropdown, dropdown.js pourrait appeler directement catalogue.js ou filtre.js. Mais cela créerait un couplage fort : dropdown.js devrait importer ou connaître l'API de chaque module destinataire. Le CustomEvent découple les deux : dropdown.js publie, les autres écoutent s'ils veulent.

javascript
// Dans dropdown.js — publication sans connaissance du destinataire
dropdownItems.forEach(item => {
    item.addEventListener('click', function() {
        const category = this.dataset.category;
        closeDropdown();

        document.dispatchEvent(new CustomEvent('categorySelected', {
            detail: { category: category, slug: this.dataset.slug }
        }));
    });
});

// Dans catalogue.js (éventuel) — abonnement
document.addEventListener('categorySelected', function(e) {
    filterByCategory(e.detail.category);
});

CustomEvent vs click sur un lien href

L'alternative simple serait de mettre des <a href="catalogue.html?category=apple"> dans le dropdown — navigation native sans JS. Le CustomEvent est utilisé quand on veut rester sur la même page et déclencher un filtrage en place (comme sur un SPA). Pour une navigation vers une nouvelle page, le lien href est plus approprié et plus accessible.

event.detail — Transporter des données avec l'événement

La propriété detail d'un CustomEvent peut contenir n'importe quel objet JS. Ici { category, slug } transporte les données nécessaires au destinataire sans passer par des variables globales ou le DOM. Les données voyagent dans l'événement lui-même — clean et sans effet de bord.

Le film complet de A à Z ¶

dropdown.js génère entièrement le menu de navigation en JS à partir des catégories PearTechDB. C'est un exemple de rendu dynamique avec gestion asynchrone de dépendance.

Les 6 fonctions — rôle et déclencheur

createDropdownMenu() — Génère le HTML du menu depuis PearTechDB.categories. Inject dans le DOM. Appelle ensuite attachDropdownItemEvents(). Déclenchée après résolution de PearTechDB.

attachDropdownEvents() — Gestion globale : clic sur le bouton toggle, clic hors zone, touche Échap. Déclenchée à l'init.

attachDropdownItemEvents() — Listeners sur chaque item du menu généré. Appelée APRÈS createDropdownMenu() (post-injection).

toggleDropdown() — Bascule ouvert/fermé en vérifiant l'état actuel.

Le pattern asynchrone — PearTechDB disponible ou non

javascript
// data.js peut être chargé avant ou après dropdown.js
if (window.PearTechDB) {
    createDropdownMenu();   // Déjà disponible — génère immédiatement
} else {
    // Pas encore disponible — attendre le signal
    document.addEventListener('PearTechDBReady', createDropdownMenu);
}
// Dans tous les cas, les comportements globaux sont attachés
attachDropdownEvents();

Cette bifurcation gère les deux cas de chargement sans polling ni setTimeout. Si data.js est chargé synchrone avant dropdown.js dans le HTML, la branche if s'exécute. Si data.js est chargé après (async ou defer), la branche else s'exécute quand le signal PearTechDBReady arrive.

06 — Ordre du code

Pourquoi chaque bloc est placé là où il est ¶

dropdown.js gère une dépendance asynchrone (PearTechDB peut charger avant ou après). La règle fondamentale : créer le HTML avant d'attacher les listeners sur ce HTML.

1
Condition d'existencedocument.querySelector('.nav-dropdown') + if (!dropdownButton) return
Le bouton est la condition d'existence de tout le module. Sans bouton dropdown, il n'y a rien à créer ni à écouter. Le guard clause évite d'exécuter des centaines de lignes si la page ne contient pas ce bouton. C'est le pattern fail-fast appliqué à l'entrée du module — la vérification la plus tôt possible.
Sans guard : createDropdownMenu() injecterait un menu HTML dans la page… sans bouton pour le déclencher. attachDropdownEvents() ferait dropdownButton.addEventListener → TypeError sur null.
↓ résoudre la dépendance asynchrone avant de construire
2
Branchement asynchroneif (window.PearTechDB) { init() } else { addEventListener('PearTechDBReady') }
Point de décision central — placé après le guard pour son importance architecturale. Ce bloc résout le problème d'ordre de chargement des scripts : data.js peut être chargé avant ou après dropdown.js selon la configuration HTML. Si PearTechDB existe déjà → appel immédiat. Sinon → écoute de l'événement custom PearTechDBReady émis par data.js à la fin de son initialisation. Sans ce branchement, le menu serait vide si data.js chargeait après dropdown.js.
Les deux branches appellent createDropdownMenu() et attachDropdownEvents() — définies après, hoistées.
↓ comportements globaux — indépendants de PearTechDB
3
Comportements globauxfunction attachDropdownEvents() — clic bouton, clic hors menu, touche Échap
Définie avant createDropdownMenu() par ordre de simplicité. attachDropdownEvents() gère l'ouverture/fermeture du menu — comportement simple (toggle de classe CSS). Elle est placée avant createDropdownMenu() qui est bien plus complexe (génération HTML). Ce choix suit la convention : simple avant complexe. Elle appelle toggleDropdown(), closeDropdown() — définies plus bas, hoistées.
↓ la fonction principale — doit appeler attachDropdownItemEvents en dernier
4
Construction du menufunction createDropdownMenu() — génère le HTML + l'injecte
La fonction principale — appelle attachDropdownItemEvents() impérativement en fin. Elle génère le HTML complet du menu via menu.innerHTML = html, puis appelle attachDropdownItemEvents(). L'ordre interne est une contrainte DOM absolue : les .dropdown-item n'existent dans le DOM qu'après l'injection. Appeler attachDropdownItemEvents() avant retournerait 0 éléments → aucun listener.
Si attachDropdownItemEvents() était appelée avant menu.innerHTML = html : document.querySelectorAll('.dropdown-item') → 0 éléments → forEach sur tableau vide → aucun listener attaché → les items du menu s'affichent mais les clics ne font rien.
↓ manipuler ce que createDropdownMenu a créé
5
Manipulation d'étattoggleDropdown() + openDropdown() + closeDropdown()
Après createDropdownMenu() dans l'ordre de lecture — elles opèrent sur ce qu'elle crée. Ces fonctions ajoutent/retirent la classe active sur .dropdown-menu. Leur querySelector('.dropdown-menu') ne trouve l'élément que si createDropdownMenu() a été appelée en premier. L'ordre de lecture reflète la dépendance : 'créer d'abord, manipuler ensuite'.
Appellent document.querySelector('.dropdown-menu') — retourne null si createDropdownMenu() n'a pas encore été appelée.
↓ attacher les listeners sur les items uniquement après leur injection dans le DOM
6
Listeners sur itemsfunction attachDropdownItemEvents() — clics sur .dropdown-item
Appelée en dernier dans createDropdownMenu() — contrainte DOM absolue. Les .dropdown-item sont créés par menu.innerHTML = html. Cette opération détruit et recrée tout le contenu du menu. attachDropdownItemEvents() fait querySelectorAll('.dropdown-item') — ce sélecteur retourne 0 résultats si appelé avant l'injection. Il n'y a aucune exception à cette règle.
Règle absolue du projet : attacher des listeners sur des éléments injectés par innerHTML uniquement APRÈS l'injection. Cette règle s'applique dans carousel.js, recherche.js, cart.js, faq.js et ici.
Règle générale de ce fichier

Le branchement if (PearTechDB) / else (addEventListener PearTechDBReady) est le pattern de résolution de dépendance asynchrone utilisé dans tout le projet. La règle innerHTML avant addEventListener sur les éléments injectés est une contrainte DOM absolue.

Concepts et termes techniques utilisés dans ce fichier ¶

Définitions des termes importants rencontrés dans cette documentation.

Document Object Model
Représentation en mémoire de la page HTML sous forme d'arbre. Chaque balise devient un nœud. JavaScript modifie cet arbre via getElementById(), querySelector(), innerHTML. Modifier le DOM = modifier ce que l'utilisateur voit en temps réel.
Immediately Invoked Function Expression
Fonction qui s'exécute immédiatement à sa déclaration : (function() { ... })(). Crée un scope isolé — tout ce qui est déclaré dedans reste privé et ne pollue pas window. Tous les fichiers JS du projet utilisent ce pattern.
DOMContentLoaded
Événement déclenché quand le HTML est parsé et le DOM prêt — avant images et CSS. Moment idéal pour initialiser les modules JS. Distinct de window.load qui attend tous les assets réseau.
CustomEvent
Événement personnalisé émis par le code : document.dispatchEvent(new CustomEvent('nom', {detail: données})). Permet à des modules de communiquer sans se connaître — couplage zéro. L'abonné écoute via document.addEventListener('nom', fn).
classList
Interface de manipulation des classes CSS. add(), remove(), toggle(), contains(). Plus fiable que modifier className comme une string. toggle('active', condition) ajoute ou retire selon un booléen.
stopPropagation()
Stoppe la remontée (bubbling) d'un événement. Sans ça, un clic sur un bouton qui ouvre un menu déclencherait aussi le listener document.addEventListener('click', fermerMenu). Utilisé pour que le bouton d'ouverture n'annule pas immédiatement l'ouverture.
Délégation d'événements
Un seul listener sur un parent au lieu d'un listener par enfant. L'événement remonte (bubble) de l'enfant vers le parent. e.target.closest('.btn') identifie l'élément source. Avantage : fonctionne pour les éléments ajoutés dynamiquement après l'initialisation.

Dropdown : gestion d'état, focus et fermeture intelligente ¶

💡
Analogie — un seul dropdown ouvert à la fois : C'est comme une règle dans un bureau : un seul tiroir peut être ouvert à la fois. Quand tu en ouvres un, les autres se ferment automatiquement. Pas besoin de tout vérifier manuellement — la règle s'applique à chaque ouverture.
💡
Analogie — écouter le document pour fermer : Mettre un listener de clic sur document pour fermer le dropdown, c'est comme un vigile qui surveille toute la pièce. Dès que quelqu'un touche quelque chose en dehors du menu, le menu se ferme. Sans ce vigile, le menu resterait ouvert indéfiniment.
🚨 PIÈGE — Le clic sur le bouton ferme immédiatement via le listener document
Si le listener sur document ne vérifie pas e.target.closest('[data-dropdown]'), le clic sur le bouton d'ouverture déclenche deux actions : l'ouverture (listener bouton) puis la fermeture immédiate (listener document). Toujours stopper la propagation ou vérifier si le clic est dans la zone du dropdown.
🚨 PIÈGE — Oublier la gestion clavier (Escape)
Un dropdown accessible doit se fermer avec la touche Escape. Ajouter un listener keydown sur document : if (e.key === 'Escape') closeAllDropdowns(). Sans ça, les utilisateurs clavier sont bloqués dans le menu.
PRO TIP
Utilise aria-expanded="true/false" sur le bouton déclencheur et aria-haspopup="true". Les lecteurs d'écran annoncent alors "bouton, sous-menu, plié/déplié" — essentiel pour l'accessibilité.
PRO TIP
Pour les animations d'ouverture, anime max-height de 0 à une valeur fixe plutôt que display: none → block. La transition CSS peut alors s'appliquer, ce qui est impossible avec display.

État ouvert/fermé

Un booléen ou une classe CSS. Toute la logique découle de cet état unique.

Fermeture globale

Listener sur document pour détecter les clics extérieurs. Utilise .contains().

Un seul ouvert

Fermer tous les autres dropdowns avant d'en ouvrir un nouveau.

Accessibilité

aria-expanded, aria-haspopup, gestion de la touche Escape.

            return;

        // Vérifie si la base de données PearTechDB est déjà chargée
        if (window.PearTechDB) {
            // Si oui, on crée immédiatement le menu et on attache les événements
            createDropdownMenu();
            attachDropdownEvents();
        } else {
            // Sinon, on écoute l'événement personnalisé 'PearTechDBReady' émis par data.js
            document.addEventListener('PearTechDBReady', function() {
                createDropdownMenu();
                attachDropdownEvents();
            });

        // ATTACHEMENT DES ÉVÉNEMENTS GLOBAUX

         * - clic ailleurs pour fermer
         * - touche Échap pour fermer

        function attachDropdownEvents() {
            const dropdownMenu = document.querySelector('.dropdown-menu');
            if (!dropdownMenu) return;

            document.addEventListener('keydown', function(e) {
                if (e.key === 'Escape' && dropdownMenu.classList.contains('active')) {
                    closeDropdown();
                }
            });

            const icon = dropdownButton.querySelector('.material-symbols-outlined');

            menu.classList.remove('active'); // Cache le menu

        }

        /**
         * Attache un événement de clic à chaque item du dropdown
         * pour pouvoir éventuellement réagir à la sélection et fermer le menu
         */
        function attachDropdownItemEvents() {
            document.querySelectorAll('.dropdown-item').forEach(function(item) {
                item.addEventListener('click', function(e) {
                    const category = this.getAttribute('data-category'); // Récupère la catégorie cliquée
                    // Crée un événement personnalisé pour informer d'autres parties de l'application
                    const event    = new CustomEvent('categorySelected', {
                        detail: { category: category, link: this.href }
                    });
                    document.dispatchEvent(event);
                    closeDropdown(); // Ferme le menu après le clic
                });
            });
        }

        console.log('Dropdown module initialisé');
    });

})();

① Ouverture — fermer les autres d'abord

javascript
function openDropdown(dropdown) {
  // ── 1. Fermer tous les autres dropdowns ouverts ───────────────────
  document.querySelectorAll('.dropdown.open').forEach(d => {
    if (d !== dropdown) closeDropdown(d);
    // Exclut le dropdown qu'on est en train d'ouvrir.
    // Sans cette vérification, on fermerait le dropdown immédiatement après l'avoir ouvert.
  });

  // ── 2. Ouvrir le dropdown cible ───────────────────────────────────
  dropdown.classList.add('open');

  const trigger = dropdown.querySelector('.dropdown-trigger');
  trigger?.setAttribute('aria-expanded', 'true');
  // ?. car le trigger peut être absent dans certaines configurations.
}

function closeDropdown(dropdown) {
  dropdown.classList.remove('open');
  dropdown.querySelector('.dropdown-trigger')?.setAttribute('aria-expanded', 'false');
}

② Fermeture au clic extérieur — contains() expliqué

javascript
document.addEventListener('click', function(e) {
  document.querySelectorAll('.dropdown.open').forEach(dropdown => {
    if (!dropdown.contains(e.target)) {
      closeDropdown(dropdown);
    }
  });
  // contains(node) : retourne true si node est un descendant.
  // dropdown.contains(e.target) :
  //   — clic sur le bouton trigger (dans dropdown) → true → on NE ferme PAS
  //   — clic sur une option (dans dropdown)         → true → on NE ferme PAS
  //   — clic ailleurs sur la page                   → false → on ferme ✓
  //
  // Astuce : contains() retourne aussi true si e.target === dropdown lui-même.
});

document.addEventListener('keydown', function(e) {
  if (e.key === 'Escape') {
    document.querySelectorAll('.dropdown.open').forEach(closeDropdown);
    // Ferme tous les dropdowns ouverts avec Escape.
    // Obligatoire pour l'accessibilité clavier (WCAG 2.1 pattern).
  }
});

③ Le problème stopPropagation — pourquoi on évite

javascript
// ❌ Solution naïve : stopPropagation sur chaque dropdown
trigger.addEventListener('click', function(e) {
  e.stopPropagation(); // Empêche le listener document de recevoir ce clic
  toggleDropdown(dropdown);
});
// Problème :
// - Casse tout autre listener sur document qui dépend de ce clic
// - Impossible à débugger : le clic "disparaît" silencieusement
// - Si on ajoute un analytics tracker sur document, il ne reçoit pas les clics dropdown

// ✓ Solution PearTech : pas de stopPropagation, contains() à la place
trigger.addEventListener('click', function(e) {
  toggleDropdown(dropdown); // L'événement continue à remonter normalement
});
// Le listener document reçoit bien le clic.
// Mais dropdown.contains(e.target) = true → on ne ferme pas. ✓
