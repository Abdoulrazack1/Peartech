01 — Stockage

load() / save() — un tableau d'IDs entiers ¶

favoris.js stocke uniquement les IDs des produits (des entiers), pas les données complètes. Les données produits sont récupérées à la demande depuis PearTechDB quand nécessaire.

Cycle read-modify-write des favoris

Le try/catch dans load() protège contre un JSON corrompu : si quelqu'un a modifié manuellement localStorage, on obtient [] plutôt qu'une exception fatale.

load() et save() — Les deux primitives de persistence

javascript
function loadFavoris() {
    try {
        return JSON.parse(localStorage.getItem('peartech-favoris')) || [];
    } catch(e) {
        return []; // JSON corrompu → tableau vide
    }
}

function saveFavoris(favs) {
    try {
        localStorage.setItem('peartech-favoris', JSON.stringify(favs));
    } catch(e) {
        console.warn('Impossible de sauvegarder les favoris :', e);
    }
}

JSON.parse avec try/catch — Protection contre le JSON corrompu

Si localStorage.getItem('peartech-favoris') retourne une chaîne corrompue (ex: "[{id:1" sans fermeture), JSON.parse() lève une SyntaxError. Sans try/catch, cette erreur remonterait et planterait tout favoris.js. Le catch retourne [] — les favoris sont perdus mais le module continue de fonctionner.

|| [] — Valeur par défaut pour getItem null

localStorage.getItem('peartech-favoris') retourne null si la clé n'existe pas (premier lancement). JSON.parse(null) retourne null. null || [] → []. La chaîne est concise : lire → parser → fallback vide, sans if explicite.

Pourquoi load() relit localStorage à chaque appel

Au lieu de garder une variable en mémoire et de n'appeler load() qu'une fois, favoris.js relit localStorage à chaque opération. C'est légèrement moins efficace mais toujours synchronisé : si un autre onglet du même navigateur modifie les favoris, favoris.js verra les changements sans mécanisme de synchronisation supplémentaire.

function toggle(id) {
id = parseInt(id);      // Force l'entier (sécurité) const ids = load();     // Charge l'état actuel const idx = ids.indexOf(id); // -1 si absent, index si présent if (idx === -1) {
ids.push(id);        // Absent  on ajoute à la fin } else {
ids.splice(idx, 1); // Présent  on retire exactement 1 élément à l'index trouvé }
save(ids);
updateAllBadges();      // Synchronise le compteur dans le header return idx === -1;      // true = ajouté, false = retiré
}
indexOf vs includes — quelle différence ?
includes(val)
Retourne true/false. Utilisé dans isFavori() quand on veut juste savoir si l'ID est présent — simple, lisible.
indexOf(val)
Retourne l'index (-1 si absent). Utilisé dans toggle() car on a besoin de l'index pour le splice(). Faire includes() puis indexOf() serait deux parcours là où indexOf() seul suffit.
splice(idx, 1)
Modifie le tableau en place : retire exactement 1 élément à la position idx. splice(2, 1) sur [1,7,42] donne [1,7].

Pourquoi chaque bloc est placé là où il est ¶

favoris.js suit une architecture en couches ascendantes : données → badges → composants → orchestration → exposition. Chaque couche s'appuie uniquement sur les couches inférieures.

1

En tout premier — utilisées dans toutes les couches suivantes. FAV_KEY est la clé localStorage utilisée dans load() et save(). La centraliser évite les fautes de frappe (une faute ici = tous les favoris perdus). FALLBACKS est un objet statique d'URLs d'images de secours par catégorie — utilisé dans le rendu des cards de la page favoris.
↓ API CRUD — fondation de tout le module
2
API interneload() + save() + isFavori() + toggle() + getAll() + clear()

Elles sont définies en premier car toutes les fonctions suivantes les appellent. toggle() appelle load(), save(), et updateAllBadges(). En les plaçant au sommet, chaque fonction suivante peut être comprise en isolation — elle délègue les détails à cette couche de base.
load() et save() utilisent FAV_KEY. toggle() appelle load(), save(), updateAllBadges() (étape 3).
↓ pont entre données et UI

Concepts et termes techniques utilisés dans ce fichier ¶

Définitions des termes importants rencontrés dans cette documentation.

Document Object Model
Représentation en mémoire de la page HTML sous forme d'arbre. Chaque balise devient un nœud. JavaScript modifie cet arbre via getElementById(), querySelector(), innerHTML. Modifier le DOM = modifier ce que l'utilisateur voit en temps réel.
Immediately Invoked Function Expression
Fonction qui s'exécute immédiatement à sa déclaration : (function() { ... })(). Crée un scope isolé — tout ce qui est déclaré dedans reste privé et ne pollue pas window. Tous les fichiers JS du projet utilisent ce pattern.
DOMContentLoaded
Événement déclenché quand le HTML est parsé et le DOM prêt — avant images et CSS. Moment idéal pour initialiser les modules JS. Distinct de window.load qui attend tous les assets réseau.
localStorage
Stockage clé-valeur persistant dans le navigateur — survit à la fermeture. Capacité ~5 Mo. Clé/valeur uniquement en string : les objets passent par JSON.stringify() avant stockage et JSON.parse() à la lecture.
JSON — JavaScript Object Notation
Format texte pour sérialiser des objets. JSON.stringify(obj) → string stockable. JSON.parse(str) → objet utilisable. Obligatoire pour stocker/lire des objets dans localStorage ou sessionStorage.
dataset
Interface qui expose les attributs data-* d'un élément. el.dataset.productId lit data-product-id. Permet de stocker des données métier directement dans le HTML sans variables globales. La conversion camelCase est automatique : data-product-id → dataset.productId.
Template literal
Chaîne avec backtick `...` qui permet d'insérer des expressions avec ${variable} et des retours à la ligne. Remplace la concaténation de strings. Utilisé massivement pour générer le HTML des cards dynamiquement.
Opérateur spread [...]
Étale les éléments d'un itérable. [...arr] crée une copie superficielle. [...a, ...b] fusionne. Utilisé pour copier un tableau avant de le trier sans modifier l'original.

Favoris : toggle, persistance et synchronisation UI ¶

💡
Analogie — le tableau de bord : La liste des favoris en localStorage est comme un tableau de bord de cuisine. Les icônes cœur sur les produits sont les voyants lumineux correspondants. Quand tu coches un plat, le voyant s'allume. La liste et les voyants doivent toujours être synchronisés — si l'un ment, l'utilisateur est perdu.
💡
Analogie — Set vs Array pour les IDs : Stocker des IDs dans un Set plutôt qu'un Array, c'est comme une liste de membres avec contrôle d'accès : tu ne peux pas entrer deux fois, et vérifier si quelqu'un est membre est instantané (has()) plutôt que de faire la queue (includes()).
🚨 PIÈGE — Oublier de mettre à jour tous les boutons favoris de la page
Si un produit apparaît plusieurs fois (catalogue + section "recommandés"), toggler un favori doit mettre à jour tous ses boutons. Utilise document.querySelectorAll('[data-product-id="'+id+'"]') et mets à jour chacun, pas seulement le bouton cliqué.
🚨 PIÈGE — Stocker des objets produits entiers dans les favoris
Ne stocker que les IDs en favoris, pas les objets complets. Les données produits peuvent changer (prix, stock). Au chargement, récupérer les détails depuis PearTechDB.getById(id). Des favoris stables avec des données fraîches.
PRO TIP
Utilise un Set en mémoire comme cache des IDs favoris. Set.has(id) est O(1) — vérification instantanée — là où Array.includes() est O(n). Sur 1000 favoris, la différence est mesurable.
PRO TIP
Anime le cœur au clic avec une classe CSS temporaire : el.classList.add('pop') + setTimeout(() => el.classList.remove('pop'), 300). Un petit effet de rebond donne un feedback physique satisfaisant.

(function () {
    'use strict'; // Mode strict activé

    const FAV_KEY = 'peartech-favoris'; // Clé de stockage dans localStorage

    // Images de fallback par catégorie si les images produits ne se chargent pas
    const FALLBACKS = {
        'cat_apple':    'https://images.unsplash.com/photo-1591337676887-a217a6970a8a?w=400&q=80',
        'cat_android':  'https://images.unsplash.com/photo-1610945415295-d9bbf067e59c?w=400&q=80',
        'cat_wearables':'https://images.unsplash.com/photo-1523275335684-37898b6baf30?w=400&q=80',
        'cat_tablets':  'https://images.unsplash.com/photo-1587033411391-5d9e51cce126?w=400&q=80',
    };

    // ============================================
    // API INTERNE - Fonctions de gestion des favoris
    // ============================================

    // Charge la liste des IDs favoris depuis localStorage
    function load() {
        try {
            return JSON.parse(localStorage.getItem(FAV_KEY)) || []; // Désérialise ou retourne tableau vide
        } catch {
            return []; // En cas de JSON corrompu, retourne un tableau vide
        }
    }

    // Sauvegarde la liste des IDs favoris dans localStorage
    function save(ids) {
        localStorage.setItem(FAV_KEY, JSON.stringify(ids)); // Sérialise et sauvegarde
    }

    // Vérifie si un produit est dans les favoris
    function isFavori(id) {
        return load().includes(parseInt(id)); // Convertit en entier car les IDs peuvent être des strings
    }

    // Ajoute ou retire un produit des favoris (toggle)
    function toggle(id) {
        id = parseInt(id); // Force la conversion en entier
        const ids = load();         // Charge les favoris actuels
        const idx = ids.indexOf(id); // Cherche l'ID dans la liste (-1 si absent)
        if (idx === -1) {
            ids.push(id); // Produit absent : on l'ajoute
        } else {
            ids.splice(idx, 1); // Produit présent : on le retire (1 élément à partir de l'index)
        }
        save(ids);          // Sauvegarde la liste mise à jour
        updateAllBadges();  // Met à jour le compteur dans le header
        return idx === -1;  // Retourne true si ajouté, false si retiré
    }

    // Retourne tous les IDs favoris
    function getAll() {
        return load();
    }

    // Vide complètement les favoris
    function clear() {
        save([]);          // Sauvegarde un tableau vide
        updateAllBadges(); // Remet le badge à 0
    }

    // ============================================
    // BADGES HEADER - Compteur de favoris
    // ============================================

    function updateAllBadges() {
        const count = load().length; // Nombre de favoris actuels
        // Met à jour tous les badges dans le DOM (header desktop + mobile)
        document.querySelectorAll('#fav-count, .fav-badge').forEach(el => {
            el.textContent = count > 0 ? count : ''; // Affiche le nombre ou vide si 0
            el.dataset.count = count;                // Stocke la valeur en data attribute
            el.style.display = count > 0 ? '' : 'none'; // Cache le badge si 0
        });
    }

    // ============================================
    // BOUTON CŒUR SVG
    // ============================================

    // Crée un bouton cœur pour une carte produit
    function renderHeartBtn(productId, isFav) {
        const btn = document.createElement('button');
        btn.className = 'btn-fav';             // Classe CSS du bouton
        btn.dataset.favBtn = productId;        // Stocke l'ID pour la délégation d'événements
        btn.setAttribute('aria-label',
            (isFav ? 'Retirer des favoris' : 'Ajouter aux favoris')); // Label accessible
        btn.innerHTML = heartSVG(isFav);       // Injecte le SVG cœur (plein ou vide)
        return btn;
    }

    // Génère le SVG d'un cœur selon l'état (plein = favori, vide = non favori)
    function heartSVG(filled) {
        return `<svg viewBox="0 0 24 24" width="18" height="18"
                     fill="${filled ? '#ef4444' : 'none'}"
                     stroke="${filled ? '#ef4444' : 'currentColor'}"
                     stroke-width="1.8" stroke-linecap="round" stroke-linejoin="round"
                     aria-hidden="true">
                    <path d="M20.84 4.61a5.5 5.5 0 0 0-7.78 0L12 5.67l-1.06-1.06
                             a5.5 5.5 0 0 0-7.78 7.78l1.06 1.06L12 21.23l7.78-7.78
                             1.06-1.06a5.5 5.5 0 0 0 0-7.78z"/>
                </svg>`;
        // fill rouge si favori, transparent sinon
        // stroke rouge si favori, couleur du thème sinon
    }

    function updateHeartBtn(btn, isFav) {

    function buildCard(product) {
        const price    = product.basePrice; // Prix actuel
        const oldPrice = (product.oldPrice && product.oldPrice > price) ? product.oldPrice : null; // Ancien prix si supérieur
        const discount = oldPrice ? Math.round((1 - price / oldPrice) * 100) : 0; // Pourcentage de réduction

        // Badge (Nouveau / -X% / Meilleure vente / rien)
        const badge = product.isNew
            ? '<span class="product-badge badge-new">Nouveau</span>'
            : discount > 0
                ? `<span class="product-badge badge-promo">-${discount}%</span>`
                : product.isBestSeller
                    ? '<span class="product-badge badge-bestseller">Meilleure vente</span>'
                    : '';

        // Alerte stock bas
        const stockHtml = product.stock <= 3
            ? `<span class="stock-alert">Plus que ${product.stock} en stock !</span>`
            : product.stock <= 10
                ? '<span class="stock-low">Stock limité</span>'
                : '';

        // Texte des spécifications (processeur · RAM · stockage)
        const specsText = [product.specs.processor, product.specs.ram, product.specs.storage]
            .filter(Boolean).join(' · ')
            || Object.values(product.specs).slice(0, 2).join(' · ')
            || '';

        // Formatage des prix en français
        const priceFmt    = price.toLocaleString('fr-FR', { minimumFractionDigits: 2 }) + ' €';
        const oldPriceFmt = oldPrice
            ? `<span class="product-old-price">${oldPrice.toLocaleString('fr-FR', { minimumFractionDigits: 2 })} €</span>`
            : '';

        // Génération des étoiles (pleines, demi, vides)
        const rating    = product.rating || 0;

        const fallback = FALLBACKS[product.categoryId] || '';

// Couche 1 : Set en mémoire (rapide, accès O(1))
let favoritesSet = new Set();
// Set : collection de valeurs uniques. Pas de doublons possibles.
// .has(id) : O(1) — aussi rapide avec 1 élément qu'avec 10 000.
// Array.includes() : O(n) — parcourt tout le tableau à chaque fois.

function loadFavorites() {
  try {
    const stored = localStorage.getItem('peartech-favorites');
    const ids    = stored ? JSON.parse(stored) : [];
    // ids est un tableau de nombres : [1, 5, 12, 42]
    // On ne stocke PAS les objets produits — uniquement les IDs.

    favoritesSet = new Set(ids);
    // new Set([1, 5, 12]) → Set{1, 5, 12}
    // Conversion directe du tableau en Set via le constructeur.
  } catch(e) {
    favoritesSet = new Set(); // Repart vide si données corrompues
  }
}

function saveFavorites() {
  const ids = [...favoritesSet];
  // Spread operator : convertit Set → Array pour JSON.stringify.
  // JSON.stringify ne sait pas sérialiser un Set directement.

  localStorage.setItem('peartech-favorites', JSON.stringify(ids));
}

function toggleFavorite(id) {
  if (favoritesSet.has(id)) {
    favoritesSet.delete(id);
    // delete sur un Set : retire l'élément. Retourne true si trouvé, false sinon.
  } else {
