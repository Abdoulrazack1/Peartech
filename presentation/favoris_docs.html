Comprendre
favoris.js

Le système de favoris complet de PearTech. Il gère le stockage des IDs en localStorage, les boutons cœur SVG dynamiques sur toutes les pages via délégation globale, la page dédiée page_favoris.html avec grille animée et suppression au clic, les toasts de notification, et expose une API publique window.Favoris.

01 — Stockage

load() / save() — un tableau d'IDs entiers

favoris.js stocke uniquement les IDs des produits (des entiers), pas les données complètes. Les données produits sont récupérées à la demande depuis PearTechDB quand nécessaire.

Cycle read-modify-write des favoris
load()
getItem → JSON.parse → [3, 7, 42] ou []
↓ toggle(id) appelé
toggle()
ids = load() → push ou splice → save(ids)
save()
setItem('peartech-favoris', JSON.stringify(ids))
Le try/catch dans load() protège contre un JSON corrompu : si quelqu'un a modifié manuellement localStorage, on obtient [] plutôt qu'une exception fatale.
javascript — API de stockage interne
const FAV_KEY = 'peartech-favoris'; // Clé localStorage

function load() {
    try {
        return JSON.parse(localStorage.getItem(FAV_KEY)) || [];
    } catch {
        return []; // JSON corrompu → tableau vide plutôt qu'une erreur fatale
    }
}

function save(ids) {
    localStorage.setItem(FAV_KEY, JSON.stringify(ids));
}

function isFavori(id) {
    return load().includes(parseInt(id)); // parseInt car les IDs peuvent arriver en string depuis le DOM
}
Ce qui est stocké dans localStorage
// Clé : "peartech-favoris"
// Valeur : tableau JSON d'IDs entiers
[1, 7, 42, 15]
// Exemple après 4 cœurs cliqués
// PAS les noms, prix ou images — juste les IDs
// Les données complètes sont lues depuis PearTechDB à la demande
💡Stocker seulement les IDs plutôt que les objets complets présente deux avantages : compacité (quelques octets vs potentiellement des Ko), et fraîcheur des données — si un prix change dans data.js, la page favoris affichera toujours le prix actuel puisqu'elle relit depuis PearTechDB à chaque affichage.
02 — Ajouter / Retirer

toggle() — splice vs push selon indexOf

toggle(id) — splice vs push selon indexOf
Produit déjà favori → splice (retirer)
ids = [3, 7, 12]
indexOf(7) = 1
splice(1, 1)
→ ids = [3, 12]
save() → ❤️ retiré
Nouveau produit → push (ajouter)
ids = [3, 12]
indexOf(42) = -1
push(42)
→ ids = [3, 12, 42]
save() → ❤️ ajouté
const idx = ids.indexOf(id) · if(idx > -1) ids.splice(idx, 1) · else ids.push(id) · save(ids) · return ids.includes(id)
javascript
function toggle(id) {
    id = parseInt(id);      // Force l'entier (sécurité)
    const ids = load();     // Charge l'état actuel
    const idx = ids.indexOf(id); // -1 si absent, index si présent

    if (idx === -1) {
        ids.push(id);        // Absent → on ajoute à la fin
    } else {
        ids.splice(idx, 1); // Présent → on retire exactement 1 élément à l'index trouvé
    }

    save(ids);
    updateAllBadges();      // Synchronise le compteur dans le header
    return idx === -1;      // true = ajouté, false = retiré
}
📖 indexOf vs includes — quelle différence ?
includes(val)
Retourne true/false. Utilisé dans isFavori() quand on veut juste savoir si l'ID est présent — simple, lisible.
indexOf(val)
Retourne l'index (-1 si absent). Utilisé dans toggle() car on a besoin de l'index pour le splice(). Faire includes() puis indexOf() serait deux parcours là où indexOf() seul suffit.
splice(idx, 1)
Modifie le tableau en place : retire exactement 1 élément à la position idx. splice(2, 1) sur [1,7,42] donne [1,7].

updateAllBadges() — synchroniser tous les compteurs

javascript
function updateAllBadges() {
    const count = load().length;
    document.querySelectorAll('#fav-count, .fav-badge').forEach(el => {
        el.textContent      = count > 0 ? count : '';
        el.dataset.count    = count;
        el.style.display    = count > 0 ? '' : 'none'; // Cache le badge si 0
    });
}
// Le sélecteur CSS '#fav-count, .fav-badge' cible à la fois l'ID (header desktop)
// et la classe (potentiellement présente plusieurs fois, ex: header mobile).
// Tous sont mis à jour en un seul forEach.
03 — Délégation d'événements globale

Un seul écouteur pour tous les boutons cœur de tout le site

Chaque page du projet peut avoir des dizaines de boutons cœur — sur les grilles produits, les cartes catalogue, les pages de détail. Attacher un écouteur sur chacun serait coûteux. La délégation résout ça avec un unique addEventListener sur le document.

Structure HTML générée par buildCard(product)
.fav-card[data-product-id]
├─ .fav-card-img
│  └─ <img src="{p.image}">
├─ .fav-card-body
│  ├─ .fav-card-name → {p.name}
│  ├─ .fav-card-price → {formatPrice(p.price)}
│  └─ .fav-card-actions
    ├─ .btn-add-cart[data-product-id] → panier.js
    └─ .btn-remove-fav[data-fav-id] → toggle()
Les attributs data-product-id et data-fav-id permettent à la délégation (panier.js et favoris.js) d'identifier le produit sans fermeture sur la variable product.
javascript — délégation sur document
document.addEventListener('click', function(e) {
    // closest() remonte dans les ancêtres jusqu'à trouver [data-fav-btn]
    // Gère le cas où le clic est sur le SVG interne (enfant du bouton)
    const btn = e.target.closest('[data-fav-btn]');
    if (!btn) return; // Pas un bouton favori → ignore

    e.preventDefault();  // Empêche tout comportement natif (lien, form)
    e.stopPropagation(); // Empêche la propagation vers un lien parent (ex: fiche produit)

    const id    = parseInt(btn.dataset.favBtn);
    const added = toggle(id); // true = ajouté, false = retiré

    updateHeartBtn(btn, added);    // SVG plein ↔ vide

    // Si on est sur la page favoris et qu'on vient de désaimer → retirer la card
    if (document.querySelector('.favoris-grid') && !added) {
        removeCardFromGrid(id);
    }

    showToast(added ? 'Ajouté aux favoris' : 'Retiré des favoris');
});
Pourquoi e.target.closest() plutôt que e.target directement
Structure HTML du bouton :
<button data-fav-btn="7" class="btn-fav"> ← l'écouteur veut ça
  <svg aria-hidden="true"> ← mais le clic peut atterrir ici
    <path d="M20.84 ..."/> ← ou même ici
  </svg>
</button>

e.target → peut être <path> ou <svg> → data-fav-btn introuvable
e.target.closest() → remonte jusqu'au <button> → data-fav-btn trouvé ✓

Cette fonction ne s'active que si #favoris-grid existe dans le DOM — c'est-à-dire uniquement sur page_favoris.html. Elle inclut un mécanisme d'attente si PearTechDB n'est pas encore disponible.

javascript — attente de PearTechDB par polling
if (window.PearTechDB) {
    render(); // Disponible immédiatement → rendu direct
} else {
    let attempts = 0;
    const t = setInterval(() => {
        attempts++;
        if (window.PearTechDB) {
            clearInterval(t);  // DB disponible → arrêt du polling
            render();          // Rendu enfin possible
        } else if (attempts > 50) {
            clearInterval(t);  // Abandon après 50 × 100ms = 5 secondes
        }
    }, 100); // Vérifie toutes les 100ms
}
💡
Pourquoi le polling ? Les scripts HTML se chargent dans l'ordre de leur balise <script>. Si data.js est placé après favoris.js dans le HTML, window.PearTechDB n'existe pas encore au moment de DOMContentLoaded. Le polling vérifie toutes les 100ms jusqu'à ce que data.js ait fini de s'exécuter et d'exposer window.PearTechDB.

removeCard() — suppression animée d'une card

javascript
function removeCard(id) {
    const card = grid.querySelector(`[data-product-id="${id}"]`);
    if (!card) return;

    // 1. Lance l'animation de disparition
    card.style.transition = 'opacity 0.3s ease, transform 0.3s ease';
    card.style.opacity    = '0';           // Fondu sortant
    card.style.transform  = 'scale(0.95)'; // Légère réduction

    // 2. Après l'animation, retire la card du DOM et réévalue l'état
    setTimeout(() => {
        card.remove();
        render(); // Réévalue : page vide ? label "X produits" ? bouton "Tout supprimer" ?
    }, 300);
}

window._removeFavCard = removeCard; // Exposé pour être appelé par la délégation globale
🔗
window._removeFavCard — pont entre la délégation globale et la fonction locale. La délégation sur document (scope global) ne peut pas accéder directement à removeCard (scope local d'initFavorisPage). L'exposer sur window crée une passerelle. Le préfixe _ signale que c'est un détail d'implémentation interne.

buildCard() — tout le HTML d'une card favori

Cette fonction transforme un objet produit issu de PearTechDB en une chaîne HTML complète. Elle gère 5 sous-calculs avant de générer le HTML.

Les 5 calculs de buildCard()
discount Math.round((1 - price / oldPrice) * 100) — pourcentage de réduction
badge Nouveau → Promo (-X%) → Meilleure vente → rien (priorité décroissante)
stockHtml ≤ 3 → "Plus que N en stock !" · ≤ 10 → "Stock limité" · sinon vide

06 — Notification

showToast() — créer, afficher, puis supprimer

Cycle de vie du toast — création, affichage, suppression
t = 0ms
document.createElement('div') → class="fav-toast" → body.appendChild(toast)
t ≈ 16ms
CSS opacity: 0 → 1 via transition (ajouté au DOM = visible immédiatement)
t = 2200ms
toast.classList.add('fade-out') → transition opacity 1 → 0 en 300ms
t = 2500ms
toast.remove() — supprimé du DOM
querySelectorAll('.fav-toast').forEach(t => t.remove()) en début de fonction → un seul toast à la fois, jamais d'empilement
javascript
function showToast(msg) {
    // Supprime tout toast déjà visible pour éviter les empilements
    document.querySelectorAll('.fav-toast').forEach(t => t.remove());

    const toast = document.createElement('div');
    toast.className = 'fav-toast';
    toast.innerHTML = `<span class="material-symbols-outlined">favorite</span>${msg}`;
    document.body.appendChild(toast); // Visible dès cet ajout (grâce au CSS)

    setTimeout(() => {              // Après 2.5s : lance la disparition
        toast.style.transition = 'opacity 0.3s ease, transform 0.3s ease';
        toast.style.opacity    = '0';
        toast.style.transform  = 'translateY(10px)'; // Glisse légèrement vers le bas
        setTimeout(() => toast.remove(), 300); // Supprime après la fin de l'animation
    }, 2500);
}
Cycle de vie du toast
t = 0ms
createElement + appendChild → visible (opacity:1 via CSS .fav-toast)
t = 2500ms
transition installée → opacity:0 + translateY(10px) — animation 300ms
t = 2800ms
toast.remove() — supprimé du DOM
