Comprendre
panier.js

Ce fichier gère tout ce qui touche au panier sur toutes les pages du site : stockage dans le navigateur, badge de compteur dans la navigation, et ajout de produits depuis une carte ou depuis une fiche produit. Un seul script, plusieurs pages couvertes.

panier.js · 8 fonctions · ~252 lignes · actif sur toutes les pages
00 — Avant de commencer

Délégation globale et API window.PearTechCart

panier.js s'exécute sur chaque page du site. Il lit le panier sauvegardé dans le navigateur, affiche le bon nombre sur le badge du header, et intercepte tous les clics "Ajouter au panier" — qu'ils viennent d'une page d'accueil, d'un catalogue, ou d'une fiche produit.

Analogie : C'est la caisse centrale d'un supermarché. Peu importe dans quel rayon un client ajoute un article à son caddie (accueil, catalogue, fiche produit), c'est toujours la même caisse qui enregistre l'ajout, met à jour le total affiché au-dessus de l'entrée, et stocke le contenu du caddie dans un casier numéroté du magasin (localStorage).

Vocabulaire essentiel — lisez ça en premier

Termes utilisés dans ce fichier
try / catch
Structure de protection contre les erreurs. Le code dans try est exécuté. Si une erreur survient, le code dans catch prend le relais — sans faire planter toute la page. Indispensable pour localStorage qui peut être bloqué.
QuotaExceededError
Erreur lancée par le navigateur quand localStorage est plein. Fréquent en navigation privée où le quota est très limité. Sans try/catch, cette erreur ferait planter le script.
e.target.closest()
element.closest('.classe') remonte l'arbre HTML depuis l'élément cliqué jusqu'à trouver un ancêtre avec la classe donnée. Si aucun ancêtre ne correspond, retourne null. Utilisé pour la délégation d'événements.
Délégation d'événements
Au lieu d'installer un écouteur sur chaque bouton "Ajouter au panier" (qui peuvent être nombreux et injectés dynamiquement), on en installe UN SEUL sur document. Tous les clics remontent jusqu'à lui. Il vérifie si le clic a touché un bouton intéressant.
JSON.stringify / JSON.parse
localStorage ne peut stocker que du texte. JSON.stringify(tableau) convertit le tableau panier en texte. JSON.parse(texte) reconvertit ce texte en tableau au moment de le relire. Sans ça, on stockerait "[object Object]".
parseFloat
parseFloat("1 299,00 €") extrait le nombre décimal d'une chaîne. Ici on nettoie d'abord la chaîne avec .replace(/[^\d,]/g, '') pour retirer tout ce qui n'est pas chiffre ou virgule, puis on remplace la virgule par un point.
window.PearTechCart
L'API publique exportée par panier.js. En exposant { add, getCart, clear } sur window, n'importe quel autre script peut appeler window.PearTechCart.add(produit) pour ajouter un article au panier.
01 — Stockage sécurisé

storageGet et storageSet — accès localStorage blindé

localStorage peut tomber en panne pour trois raisons : navigation privée (quota très faible), cookies bloqués par l'utilisateur, ou stockage plein. Sans protection, le site planterait silencieusement. Ces deux fonctions encapsulent tous les accès.

Ce que try/catch protège concrètement
Sans try/catch
function storageGet(key) {
  return localStorage.getItem(key);
  // Si localStorage bloqué :
  // SecurityError non attrapée
  // tout le script s'arrête
}
Une erreur ici ferait planter l'ensemble de panier.js. Le badge ne s'afficherait pas, aucun ajout ne fonctionnerait.
Avec try/catch
function storageGet(key) {
  try {
    return localStorage.getItem(key);
  } catch (e) {
    console.warn('échoué :', e);
    return null;
  }
}
Si localStorage est bloqué, on renvoie null. Le reste du code gère ce cas (|| [] donne un panier vide). Le site fonctionne en mode dégradé.
javascript
function storageGet(key) {
try {
return localStorage.getItem(key);
} catch (e) {
console.warn('localStorage.getItem échoué :', e);
return null;
}
}
function storageSet(key, value) {
try {
localStorage.setItem(key, value);
return true;   // succès } catch (e) {
// QuotaExceededError = stockage plein (fréquent en navigation privée) console.warn('localStorage.setItem échoué (quota ou mode privé ?) :', e);
return false;  // appelant peut savoir que ça a échoué }
}
storageSet retourne true ou false — et saveCart() utilise ce retour. Si storageSet renvoie false, la fonction saveCart() affiche un toast d'avertissement à l'utilisateur. Le panier reste fonctionnel EN MÉMOIRE (dans la variable cart), mais ne survivra pas au rechargement de la page.

Chargement initial du panier — triple protection

javascript
let cart = [];
try {
cart = JSON.parse(storageGet(CART_STORAGE_KEY)) || [];
// Couche 1 : storageGet  renvoie null si localStorage bloqué
// Couche 2 : JSON.parse(null)  renvoie null (sans erreur)
// Couche 3 : || []  si null, panier vide par défaut
// Couche 4 : try/catch  si JSON corrompu, JSON.parse lance une SyntaxError
} catch (e) {
// Données corrompues dans localStorage (ex: coupure réseau pendant l'écriture) console.warn('Panier corrompu dans localStorage, réinitialisation :', e);
cart = [];
}
Les 4 couches de protection du chargement
localStorage contient : '[{"id":5,"name":"iPhone","quantity":2}]'
storageGet('peartech-cart')
Retourne la chaîne… ou null si localStorage bloqué
JSON.parse(chaîne)
Convertit en vrai tableau JS… ou null si la chaîne était null
|| []
Panier chargé — ou tableau vide si tout a échoué
Si JSON.parse plante (données corrompues) catch cart = []

JSON.stringify pour comparer des objets : En JavaScript, obj1 === obj2 est toujours faux même si les deux objets ont les mêmes propriétés. C'est parce que === compare les références (les adresses mémoire), pas le contenu. JSON.stringify(obj1) === JSON.stringify(obj2) compare les textes produits — si le contenu est identique, les textes sont identiques. C'est le seul moyen simple de comparer deux objets en profondeur.
javascript
function addToCart(product) {
const existing = cart.find(item => item.id === product.id &&
JSON.stringify(item.options) === JSON.stringify(product.options)
);
// Cherche un article IDENTIQUE : même id ET mêmes options
// {ram:"16Go", storage:"256Go"} ≠ {ram:"32Go", storage:"256Go"}
//  deux lignes séparées dans le panier if (existing) {
existing.quantity += product.quantity || 1;
// Produit déjà dans le panier avec ces options  +1 à la quantité
// product.quantity || 1 : si quantity n'est pas fourni, on ajoute 1 } else {
cart.push({
id:            product.id,
name:          product.name,
basePrice:     product.basePrice,
price:         product.price,        // prix AVEC options image:         product.image,
specs:         product.specs || '',
options:       product.options || [],
optionSummary: product.optionSummary || '',
quantity:      product.quantity || 1 });
}
saveCart();
}

L'API publique — window.PearTechCart

javascript
window.PearTechCart = {
add:     addToCart,
// Exposé pour product_grid.js, product_detail.js, etc. // Utilisation : window.PearTechCart.add({ id: 5, name: 'iPhone', price: 999, ... }) getCart: () => cart,
// Retourne le tableau en mémoire — utile pour la page panier
// () => cart = arrow function sans paramètre qui retourne cart clear: () => { cart = []; saveCart(); }
// Vide le panier en mémoire et dans localStorage en une ligne
};
Pourquoi exposer une API plutôt que d'utiliser window.addToCart directement ? En regroupant les fonctions dans un objet nommé PearTechCart, on évite de polluer l'espace global avec des noms génériques comme "addToCart" qui pourraient entrer en conflit avec d'autres scripts. C'est l'équivalent d'un espace de noms (namespace).
04 — Délégation d'événements

Un seul listener pour tous les boutons "Ajouter au panier"

Les boutons "Ajouter au panier" peuvent être créés dynamiquement (par product_grid.js ou product_detail.js), exister en nombre variable, et être présents sur plusieurs pages différentes. Installer un écouteur sur chacun serait fragile. La délégation résout tout cela.

e.target.closest('.btn-add-cart')

Quand on clique sur un bouton, e.target peut être l'icône SVG à l'intérieur du bouton — pas le bouton lui-même. .closest('.btn-add-cart') remonte depuis l'icône jusqu'au bouton parent qui a cette classe. Si le clic n'est pas dans un bouton "Ajouter au panier", closest() retourne null, et le if (!addButton) return fait sortir immédiatement.
Délégation vs Écouteur direct — comparaison
Sans délégation — écouteur direct
// Il faudrait faire ça pour CHAQUE bouton :
document.querySelectorAll('.btn-add-cart')
  .forEach(btn => {
    btn.addEventListener('click', …)
  });
Ne fonctionne pas sur les boutons ajoutés plus tard dynamiquement
Installe N écouteurs (un par bouton)
Doit être ré-exécuté à chaque injection de HTML
Avec délégation — utilisé ici
// Un seul écouteur, pour toujours :
document.addEventListener('click', function(e) {
  const btn = e.target.closest('.btn-add-cart');

});

Glossaire

Concepts et termes techniques utilisés dans ce fichier

Définitions des termes importants rencontrés dans cette documentation.

Document Object Model
Représentation en mémoire de la page HTML sous forme d'arbre. Chaque balise devient un nœud. JavaScript modifie cet arbre via getElementById(), querySelector(), innerHTML. Modifier le DOM = modifier ce que l'utilisateur voit en temps réel.
Immediately Invoked Function Expression
Fonction qui s'exécute immédiatement à sa déclaration : (function() { ... })(). Crée un scope isolé — tout ce qui est déclaré dedans reste privé et ne pollue pas window. Tous les fichiers JS du projet utilisent ce pattern.
DOMContentLoaded
Événement déclenché quand le HTML est parsé et le DOM prêt — avant images et CSS. Moment idéal pour initialiser les modules JS. Distinct de window.load qui attend tous les assets réseau.
localStorage
Stockage clé-valeur persistant dans le navigateur — survit à la fermeture. Capacité ~5 Mo. Clé/valeur uniquement en string : les objets passent par JSON.stringify() avant stockage et JSON.parse() à la lecture.
JSON — JavaScript Object Notation
Format texte pour sérialiser des objets. JSON.stringify(obj) → string stockable. JSON.parse(str) → objet utilisable. Obligatoire pour stocker/lire des objets dans localStorage ou sessionStorage.
Délégation d'événements
Un seul listener sur un parent au lieu d'un listener par enfant. L'événement remonte (bubble) de l'enfant vers le parent. e.target.closest('.btn') identifie l'élément source. Avantage : fonctionne pour les éléments ajoutés dynamiquement après l'initialisation.
CustomEvent
Événement personnalisé émis par le code : document.dispatchEvent(new CustomEvent('nom', {detail: données})). Permet à des modules de communiquer sans se connaître — couplage zéro. L'abonné écoute via document.addEventListener('nom', fn).
dataset
Interface qui expose les attributs data-* d'un élément. el.dataset.productId lit data-product-id. Permet de stocker des données métier directement dans le HTML sans variables globales. La conversion camelCase est automatique : data-product-id → dataset.productId.
Template literal
Chaîne avec backtick `...` qui permet d'insérer des expressions avec ${variable} et des retours à la ligne. Remplace la concaténation de strings. Utilisé massivement pour générer le HTML des cards dynamiquement.
