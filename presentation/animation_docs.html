Comprendre
animation.js

Ce fichier gère 4 effets visuels présents sur toutes les pages : l'apparition progressive des cartes au scroll (scroll reveal), l'ombre du header, l'agrandissement de la barre de recherche au clic, et le défilement fluide vers les ancres. Il utilise deux APIs modernes du navigateur : IntersectionObserver et MutationObserver.

Termes nouveaux dans ce fichier
IntersectionObserver
API navigateur qui surveille si un élément HTML est visible dans la fenêtre. Plus efficace qu'écouter l'événement scroll et calculer les positions manuellement. Le callback est déclenché uniquement quand l'élément entre ou sort du viewport.
MutationObserver
API navigateur qui surveille les modifications du DOM. Ici il détecte quand de nouvelles cartes produits sont ajoutées dynamiquement par JavaScript (product_grid.js, catalogue.js) pour leur appliquer l'animation d'entrée.
threshold: 0.1
Seuil de déclenchement de l'IntersectionObserver. 0.1 = l'animation se déclenche quand 10% de l'élément est visible. 0 = dès qu'un pixel est visible. 1 = seulement quand tout l'élément est visible.
rootMargin
Marge virtuelle autour de la zone observée. '0px 0px -50px 0px' = la zone visible est réduite de 50px en bas. Concrètement, l'animation se déclenche 50px avant que l'élément atteigne le bord visible de l'écran.
getBoundingClientRect()
Méthode qui retourne la position et les dimensions d'un élément par rapport à la fenêtre. rect.top = distance entre le haut de l'élément et le haut de la fenêtre. Utilisé pour savoir si un élément est déjà visible au chargement.
dataset.scrollObserved
Attribut HTML data-scroll-observed utilisé comme flag. Une fois qu'une carte a été prise en charge par observeCard(), ce flag empêche de la traiter deux fois si le MutationObserver la détecte à nouveau.
childList: true, subtree: true
Options du MutationObserver. childList = surveille les ajouts/suppressions d'enfants directs. subtree = surveille aussi tous les descendants. Nécessaire pour détecter les cartes peu importe leur niveau d'imbrication.

IntersectionObserver — comment il détecte l'entrée dans le viewport

Analogie : L'IntersectionObserver est comme un vigile posté à l'entrée d'une salle. Au lieu de regarder en permanence si quelqu'un arrive (ce que ferait addEventListener('scroll', ...)), il attend sagement et est notifié automatiquement quand un élément passe la porte (entre dans le viewport).
javascript
const observer = new IntersectionObserver(function(entries) {
entries.forEach(entry => {
if (entry.isIntersecting) {         // L'élément vient d'entrer dans le viewport entry.target.style.opacity    = '1';
entry.target.style.transform  = 'translateY(0)'; // Annule le décalage vertical initial }
});
}, {
threshold:  0.1,                       // 10% visible  déclenche rootMargin: '0px 0px -50px 0px' // Zone visible réduite de 50px en bas
});
Comportement de rootMargin: '0px 0px -50px 0px'
Sans rootMargin (threshold:0.1)
L'animation se déclenche quand la carte atteint le tout bas de l'écran. L'utilisateur ne voit l'animation qu'une fraction de seconde avant que la carte soit complètement visible — peu naturel.
Avec rootMargin: -50px
L'animation se déclenche 50px avant que la carte atteigne le bas visible. La carte a déjà commencé à apparaître quand l'utilisateur l'aperçoit — effet plus doux et naturel.

observeCard() — la logique de décision

javascript
function observeCard(card) {
if (card.dataset.scrollObserved) return; // Déjà traitée  ignore (idempotence) card.dataset.scrollObserved = '1';       // Marque comme traitée const rect = card.getBoundingClientRect();
const isVisible = rect.top < window.innerHeight && rect.bottom > 0;
// isVisible = la carte est actuellement visible dans la fenêtre if (isVisible) {
// Déjà visible au chargement : afficher directement sans animation card.style.opacity   = '1';
card.style.transform = 'translateY(0)';
} else {
// Hors écran : préparer l'état initial + observer pour l'animation d'entrée card.style.opacity    = '0';
card.style.transform  = 'translateY(20px)'; // 20px plus bas que sa position finale card.style.transition = 'opacity 0.5s ease, transform 0.5s ease'; // Transition fluide observer.observe(card); // L'observer commence à surveiller cet élément }
}

02 — Cartes dynamiques

MutationObserver — surveiller le DOM en temps réel ¶

MutationObserver — surveiller les éléments injectés dynamiquement
Sans MO
product_grid.js injecte cartes
nouvelles cartes sans animation (observées trop tard)
Avec MO
product_grid.js injecte cartes
MO détecte les nouveaux nœuds
IntersectionObserver relancé
cartes animées
observer.observe({childList: true, subtree: true}) détecte ajouts/suppressions dans tout le DOM

Un problème : les cartes produits sont générées après le chargement de la page par product_grid.js ou catalogue.js. Au moment où animation.js s'exécute, ces cartes n'existent pas encore. Le MutationObserver résout ce problème.

javascript
const mutationObs = new MutationObserver(function(mutations) {
mutations.forEach(function(mutation) {
mutation.addedNodes.forEach(function(node) {
if (node.nodeType !== 1) return; // Ignore les nœuds texte (type 3) et commentaires (type 8) // Cas 1 : le nœud ajouté EST directement une card if (node.classList && node.classList.contains('product-card')) {
observeCard(node);
}
// Cas 2 : le nœud est un conteneur (ex: innerHTML réécrit) //  cherche toutes les .product-card à l'intérieur node.querySelectorAll && node.querySelectorAll('.product-card').forEach(observeCard);
});
});
});
const productsGrid = document.getElementById('products-grid');
if (productsGrid) {
mutationObs.observe(productsGrid, { childList: true, subtree: true });
}
Pourquoi deux cas (node direct vs conteneur) ?
Cas 1 — Node direct
Quand un script fait productsGrid.appendChild(card). Le nœud ajouté est directement la card. node.classList.contains('product-card') est vrai.
Cas 2 — Conteneur
Quand un script fait productsGrid.innerHTML = '...'. Tout le HTML est remplacé d'un coup. Le MutationObserver détecte 1 gros nœud il faut chercher les cards en profondeur avec querySelectorAll.
catalogue.js et product_grid.js utilisent innerHTML = html (cas 2). Sans le querySelectorAll, les cartes seraient générées mais aucune n'aurait l'animation d'entrée.

Configuration du MutationObserver

javascript
const gridObserver = new MutationObserver(function(mutations) {
    mutations.forEach(mutation => {
        mutation.addedNodes.forEach(node => {
            if (node.nodeType === 1) { // Élément DOM (pas texte)
                if (node.classList.contains('product-card')) {
                    observeCard(node); // Observe la card directement
                }
                // Cherche aussi les cards imbriquées dans le nœud ajouté
                node.querySelectorAll('.product-card').forEach(observeCard);
            }
        });
    });
});
// { childList: true } — surveille uniquement les ajouts/suppressions directs
// (pas les modifications d'attributs ni le contenu textuel)
gridObserver.observe(productsGrid, { childList: true });
💡
{ childList: true } est le mode le moins coûteux du MutationObserver. { subtree: true } surveille TOUTE la sous-arborescence — utile mais beaucoup plus de callbacks. Ici on surveille uniquement le premier niveau d'enfants de #products-grid ce qui suffit.

03 — Header Shadow

Ombre sous le header au scroll ¶

L'effet d'ombre sous le header est un retour visuel subtil qui renforce la hiérarchie entre la navigation fixe et le contenu de la page. Au repos (en haut de page), le header se fond dans l'arrière-plan. Dès que l'utilisateur scrolle, une ombre apparaît pour marquer la séparation.

Le code complet

javascript
const header = document.querySelector('.header'); // Sélectionne le header

if (header) { // Guard clause : vérifie que le header existe sur cette page
    window.addEventListener('scroll', function() {
        if (window.pageYOffset > 50) { // Si scrollé de plus de 50px
            header.style.boxShadow = '0 4px 12px rgba(0,0,0,0.3)'; // Ombre portée
        } else {
            header.style.boxShadow = 'none'; // Supprime l'ombre en haut de page
        }
    });
}

window.pageYOffset — Mesurer le défilement vertical

window.pageYOffset retourne le nombre de pixels que le document a défilé verticalement depuis son origine. À position initiale (tout en haut), cette valeur est 0. Au seuil de 50px, l'ombre s'applique. Pourquoi 50 et pas 1 ? Un seuil de 1px serait trop sensible — un microdéfilement accidentel suffirait à déclencher l'ombre. 50px correspond à environ 2-3 lignes de contenu défilées, ce qui est intentionnel.

boxShadow vs classe CSS — Deux approches, un choix

Ici on modifie header.style.boxShadow directement (style inline). L'alternative serait d'ajouter une classe CSS (header.classList.add('scrolled')) et de définir l'ombre dans le CSS. L'approche CSS est plus maintenable en production : les animations sont gérées par le GPU, et le style est centralisé. L'approche inline est plus simple pour un comportement dynamique ponctuel.

💡
window.addEventListener('scroll', ...) se déclenche à chaque pixel scrollé, ce qui peut être fréquent (60 fois par seconde en défilement fluide). Pour des effets visuels simples comme une ombre, c'est acceptable. Pour des calculs lourds, on utiliserait requestAnimationFrame ou un debounce pour limiter les appels.

Pourquoi 200ms de délai au blur

Sans délai, si l'utilisateur clique sur un résultat de recherche, le blur se déclenche avant le clic — le résultat disparaît avant d'être activé. Le délai de 200ms laisse le temps au clic de se propager avant que l'animation de fermeture ne commence.

05 — Smooth Scroll

Défilement fluide vers les ancres ¶

Le smooth scroll intercepte les clics sur les liens ancres (href="#section") pour remplacer le saut brutal du navigateur par une animation de défilement fluide. C'est une amélioration UX pure — aucun impact fonctionnel.

Le code complet

javascript
// Sélectionne tous les liens ancres (href commence par "#")
document.querySelectorAll('a[href^="#"]').forEach(anchor => {
    anchor.addEventListener('click', function(e) {
        e.preventDefault(); // Annule le saut brutal par défaut

        const target = document.querySelector(this.getAttribute('href'));
        if (target) {
            target.scrollIntoView({
                behavior: 'smooth', // Animation fluide
                block: 'start'      // Aligne le haut de la cible avec le haut de la fenêtre
            });
        }
    });
});

Le sélecteur CSS a[href^='#'] — cibler uniquement les ancres

Le sélecteur a[href^="#"] cible uniquement les <a> dont l'attribut href commence par "#". L'opérateur ^= signifie 'commence par'. Cela exclut les vrais liens vers d'autres pages (href="page.html") et ne cible que les liens ancres internes à la page.

e.preventDefault() — Annuler le comportement natif

e.preventDefault() annule l'action par défaut de l'événement. Pour un clic sur un lien, le comportement par défaut du navigateur est de naviguer vers la cible — instantanément dans le cas d'une ancre. En l'annulant, on reprend le contrôle total pour implémenter l'animation avec scrollIntoView().

scrollIntoView({behavior:'smooth', block:'start'})

La méthode element.scrollIntoView() est native au navigateur et défile la page pour rendre l'élément visible. L'option behavior: 'smooth' active l'animation CSS (gérée par le navigateur, sans JS supplémentaire). block: 'start' aligne le haut de l'élément cible avec le haut du viewport.

⚠️
Cette implémentation ne tient pas compte du scroll-margin-top CSS. Si le header est fixe (position:fixed), l'élément cible se retrouvera caché sous le header. La solution propre est d'ajouter scroll-margin-top: 70px (hauteur du header) sur les éléments cibles en CSS, ou d'ajuster la position via window.scrollTo() manuellement.

scrollIntoView vs window.scrollTo — 2 approches

❌ window.scrollTo (coordonnées)

Architecture d'animation.js ¶

Voici l'intégralité du module d'animation et les 4 effets qu'il produit, leur déclencheur et leur portée.

Les 4 effets — déclencheur et portée

1. Scroll Reveal — IntersectionObserver observe chaque .category-card et .service-card statique + MutationObserver surveille les .product-card injectées dynamiquement. Déclencheur : entrée dans le viewport (threshold 10%). Effet : opacity:0→1 + translateY(20px→0) en 0.5s.

2. Header Shadow — window.scroll vérifie pageYOffset > 50. Déclencheur : défilement de plus de 50px. Effet : boxShadow sur le .header.

3. Search Focus — focus/blur sur #search-input. Déclencheur : focus clavier ou clic sur le champ. Effet : scale(1.02) sur .search-bar avec délai de 200ms au blur.

4. Smooth Scroll — click sur a[href^="#"]. Déclencheur : clic sur un lien ancre. Effet : scrollIntoView({behavior:'smooth'}) remplace le saut natif.

07 — Ordre du code

Pourquoi chaque bloc est placé là où il est ¶

Chaque ligne répond à une contrainte précise. La règle principale : créer les dépendances avant de les utiliser, traiter le présent avant de surveiller le futur.

1
IsolationIIFE — (function(){ … })()
Doit englober absolument tout le code. L'IIFE crée un scope privé : aucune variable déclarée à l'intérieur ne peut fuir dans window. Sans elle, const observer, const mutationObs… deviendraient des propriétés de window, accessibles et écrasables par n'importe quel autre script.
Sans IIFE : const observer = new IntersectionObserver(…) → window.observer. Si un plugin tiers redéfinit window.observer, toutes les animations cessent.
↓ tout le code DOM attend DOMContentLoaded
2
Déclencheur DOMdocument.addEventListener('DOMContentLoaded', …)
Tout accès au DOM doit attendre que le HTML soit parsé. Le script est souvent chargé dans le <head> ou en début de body — à ce moment, .category-card, .header, #products-grid n'existent pas encore. DOMContentLoaded garantit leur présence avant tout querySelector.
Sans DOMContentLoaded : document.querySelector('.header') → null → observer.observe(null) → TypeError immédiat au chargement.
↓ l'observer doit exister avant la fonction qui l'utilise
3
Observer de scrollobserverOptions + new IntersectionObserver(callback, options)
Dépendance directe de observeCard(). La fonction observeCard() (étape suivante) appelle observer.observe(card). En JS, const n'est pas hoistée — si observer était déclaré après observeCard(), son premier appel lèverait ReferenceError: Cannot access 'observer' before initialization.
Si déclaré après observeCard() : premier observeCard(card) → observer.observe(card) → ReferenceError car const est dans la TDZ (Temporal Dead Zone).
↓ observeCard doit être définie avant le MutationObserver qui la passe en référence
4
Fonction d'observationfunction observeCard(card) { … }
Définie avant le MutationObserver qui la référence comme callback. Le MutationObserver (étape 6) passe observeCard comme référence : mutations.forEach(m => m.addedNodes.forEach(n => observeCard(n))). Les function declarations sont hoistées, donc l'ordre technique est flexible, mais l'écrire avant signale clairement qu'elle est une dépendance du MutationObserver.
Utilise : observer (étape 3). Appelée par : querySelectorAll().forEach (étape 5) et MutationObserver (étape 6).
↓ traiter les éléments existants AVANT de surveiller les futurs
5
Éléments statiquesquerySelectorAll('.category-card, .service-card').forEach(observeCard)
Le MutationObserver ne détecte que les ajouts futurs — il est aveugle aux éléments déjà présents. Ce querySelectorAll traite toutes les cards déjà dans le HTML au moment du chargement. Si on démarrait d'abord le MutationObserver, ces cards existantes ne recevraient jamais l'IntersectionObserver et resteraient visibles sans animation d'entrée.
Si MutationObserver démarrait avant : les .category-card et .service-card du HTML initial ne seraient jamais passées à observeCard() → aucune animation sur les éléments statiques.
Principe fondamental : traiter le présent (éléments existants), puis surveiller le futur (ajouts dynamiques). Cette règle s'applique dans tous les fichiers du projet.
↓ surveiller les cards injectées par product_grid.js et catalogue.js
6
Observer dynamiquenew MutationObserver(…).observe(productsGrid, {childList:true})
Surveille les cards ajoutées dynamiquement après le chargement initial. product_grid.js et catalogue.js injectent des .product-card après DOMContentLoaded via innerHTML. Sans ce MutationObserver, ces nouvelles cards n'auraient aucune animation d'apparition. Le flag data-scroll-observed sur chaque card empêche une double observation si observeCard était appelée deux fois sur la même card.
Dépend de : observeCard() (étape 4). Surveille : #products-grid (peuplé par product_grid.js).
↓ effets secondaires indépendants — ordre de priorité
7
Header shadowwindow.addEventListener('scroll', …) — ombre du header
Effet visuel secondaire, sans dépendance avec les étapes précédentes. Ce listener est fonctionnellement indépendant de tout le système d'animation de cards. Il est placé après car c'est un effet de moindre importance que le scroll reveal. L'ordre entre étapes 7, 8 et 9 est purement conventionnel (du plus important au plus anecdotique).
↓
8
Focus barre de recherchesearchInput.addEventListener('focus'/'blur', …)
UX secondaire — le délai de 200ms sur blur est crucial. Sans ce délai, si l'utilisateur clique sur un résultat de recherche, le blur se déclenche avant le clic (blur précède toujours click dans l'ordre des événements navigateur), fait rétrécir la barre et le clic 'rate' visuellement. Le setTimeout(200) laisse le clic s'exécuter en premier.
↓
9
Smooth scrollquerySelectorAll('a[href^="#"]').forEach(…) — défilement fluide
En dernier car c'est le seul effet qui concerne la navigation, pas les cards. Il intercepte les clics sur les liens ancres internes (href="#section") et remplace le saut brutal par un défilement animé. Placé en dernier : les liens ancres sont statiques dans le HTML, ils n'ont aucune dépendance sur les étapes précédentes — leur position dans le fichier n'a pas d'impact technique.

Concepts et termes techniques utilisés dans ce fichier ¶

Définitions des termes importants rencontrés dans cette documentation.

Document Object Model
Représentation en mémoire de la page HTML sous forme d'arbre. Chaque balise devient un nœud. JavaScript modifie cet arbre via getElementById(), querySelector(), innerHTML. Modifier le DOM = modifier ce que l'utilisateur voit en temps réel.
Immediately Invoked Function Expression
Fonction qui s'exécute immédiatement à sa déclaration : (function() { ... })(). Crée un scope isolé — tout ce qui est déclaré dedans reste privé et ne pollue pas window. Tous les fichiers JS du projet utilisent ce pattern.
IntersectionObserver
API qui détecte quand un élément entre dans le viewport. Plus performant que d'écouter scroll et calculer des positions — le navigateur gère la détection hors du thread principal. Utilisé pour les animations d'apparition au scroll.
MutationObserver
API qui observe les modifications du DOM en temps réel et appelle un callback quand des nœuds sont ajoutés ou supprimés. Plus efficace que poller le DOM avec setInterval. Utilisé pour détecter les cards produit injectées dynamiquement.
DOMContentLoaded
Événement déclenché quand le HTML est parsé et le DOM prêt — avant images et CSS. Moment idéal pour initialiser les modules JS. Distinct de window.load qui attend tous les assets réseau.
scrollHeight
Hauteur totale d'un élément incluant la partie non visible (défilable). Toujours >= hauteur visible. Utilisé pour mesurer la hauteur réelle d'un contenu avant de l'animer : height:0 → height:scrollHeight+'px'.
getBoundingClientRect()
Retourne les coordonnées d'un élément dans le viewport (top, bottom, left, right). Valeurs relatives à la fenêtre visible. Appeler cette méthode force un reflow synchrone — utilisé pour forcer le navigateur à prendre en compte une valeur CSS avant de lancer une transition.
querySelector / querySelectorAll
querySelector(sélecteur) → premier élément correspondant ou null. querySelectorAll() → NodeList de tous les éléments. La NodeList n'est pas un Array — utiliser Array.from() ou [...nodeList] pour accéder aux méthodes Array.

Animations performantes : ce que le navigateur fait sous le capot ¶

💡
Analogie — IntersectionObserver : Imagine un vigile à l'entrée d'une salle de concert. Au lieu de regarder sans arrêt si des gens arrivent (polling), il attend que quelqu'un passe la porte (callback). IntersectionObserver est ce vigile : il attend qu'un élément entre dans le viewport avant d'agir.
💡
Analogie — requestAnimationFrame : Le navigateur peint l'écran 60 fois par seconde. requestAnimationFrame est une invitation à synchroniser ton code avec ce rythme — comme sauter dans un train en marche plutôt qu'essayer de courir à côté.
🚨 PIÈGE — Animer width/height au lieu de transform
Animer width ou height déclenche un reflow complet : le navigateur recalcule la position de tous les éléments. Animer transform: scale() ou opacity ne déclenche que la couche composite — 60 fps garanti même sur mobile.
🚨 PIÈGE — Oublier will-change sur les éléments animés fréquemment
Sans will-change: transform, le navigateur crée la couche GPU au premier frame de l'animation, causant un jank visible. Ajouter will-change avant l'animation prépare la couche en avance. Attention : ne l'applique pas à tous les éléments — consommation mémoire excessive.
PRO TIP
Utilise prefers-reduced-motion en CSS et JS pour désactiver les animations chez les utilisateurs qui ont activé ce réglage dans leur OS. C'est une question d'accessibilité, pas d'esthétique.
PRO TIP
Dans DevTools → onglet Rendering → coche "Frame Rendering Stats". Tu vois en temps réel le nombre de FPS et si des frames sont droppées lors de tes animations.

CSS Transitions

Délègue l'animation au moteur CSS. Plus performant que JS pour les transitions simples déclenchées par changement de classe.

IntersectionObserver

Observe l'entrée dans le viewport sans polling. Zéro coût quand rien ne se passe.

requestAnimationFrame

Synchronise le code JS avec le cycle de rendu du navigateur (60fps). Remplace setTimeout(fn, 16).

Composite layers

transform et opacity sont animés sur la carte GPU, sans toucher au layout HTML.

// ============================================
// animation.js - Animations au scroll
// Gère l'apparition progressive des éléments
// et les effets visuels au défilement de page
// ============================================

// IIFE (Immediately Invoked Function Expression) :
// encapsule tout le code pour éviter de polluer le scope global
(function() {
    'use strict'; // Active le mode strict JS pour détecter les erreurs silencieuses

    // On attend que le DOM soit entièrement chargé avant d'exécuter le code
    document.addEventListener('DOMContentLoaded', function() {

        // ============================================
        // SCROLL REVEAL - Apparition des cartes au scroll
        // Utilise l'API IntersectionObserver pour détecter
        // quand un élément entre dans la zone visible de l'écran
        // ============================================

        // Options de configuration de l'IntersectionObserver
        const observerOptions = {
            threshold: 0.1,          // Déclenche quand 10% de l'élément est visible
            rootMargin: '0px 0px -50px 0px' // Marge négative en bas : l'animation se déclenche 50px avant le bord
        };

        // Crée un observateur qui surveille l'entrée des éléments dans le viewport
        const observer = new IntersectionObserver(function(entries) {
            // On parcourt chaque élément observé
            entries.forEach(entry => {
                if (entry.isIntersecting) { // Si l'élément est maintenant visible
                    entry.target.style.opacity = '1';             // On le rend visible
                    entry.target.style.transform = 'translateY(0)'; // On annule le décalage vertical initial
                }
            });
        }, observerOptions); // On passe les options définies ci-dessus

        // ── Fonction d'attachement de l'observer à une carte ──────

        // Attache l'observer à une carte produit ou catégorie
        function observeCard(card) {
            if (card.dataset.scrollObserved) return; // Empêche d'observer la même carte deux fois
            card.dataset.scrollObserved = '1'; // Marque la carte comme déjà observée

            // Vérifie si la carte est déjà dans la zone visible de l'écran
            const rect = card.getBoundingClientRect(); // Retourne la position et les dimensions de l'élément
            // Une carte est visible si son haut est au-dessus du bas de l'écran ET son bas est en dessous du haut
            const isVisible = rect.top < window.innerHeight && rect.bottom > 0;

            if (isVisible) {
                // La carte est déjà visible : on l'affiche directement sans animation
                card.style.opacity = '1';
                card.style.transform = 'translateY(0)';
            } else {
                // La carte est hors écran : on la prépare pour l'animation d'entrée
                card.style.opacity = '0';              // Invisible au départ
                card.style.transform = 'translateY(20px)'; // Légèrement décalée vers le bas
                card.style.transition = 'opacity 0.5s ease, transform 0.5s ease'; // Transition fluide de 0.5s
                observer.observe(card); // L'IntersectionObserver commence à surveiller cet élément
            }
        }

        // ── Observer les éléments statiques déjà présents dans le DOM ──

        // Sélectionne et observe toutes les cartes de catégories et de services déjà dans la page
        document.querySelectorAll('.category-card, .service-card').forEach(observeCard);

        // ── MutationObserver : surveille les éléments ajoutés dynamiquement ──
        // Nécessaire car les product-cards sont générées par JS après le chargement initial

        // Crée un observateur du DOM pour détecter les nouveaux éléments ajoutés
        const mutationObs = new MutationObserver(function(mutations) {
            mutations.forEach(function(mutation) { // Pour chaque modification du DOM
                mutation.addedNodes.forEach(function(node) { // Pour chaque nœud ajouté
                    if (node.nodeType !== 1) return; // Ignore les nœuds qui ne sont pas des éléments HTML
                    // Si le nœud ajouté est directement une product-card
                    if (node.classList && node.classList.contains('product-card')) {
                        observeCard(node); // On l'observe immédiatement
                    }
                    // Cas où un conteneur entier est re-rendu (ex: grille entière rechargée)
                    // On cherche aussi les product-cards enfants du nœud ajouté
                    node.querySelectorAll && node.querySelectorAll('.product-card').forEach(observeCard);
                });
            });
        });

        // Récupère la grille de produits si elle existe sur la page courante
        const productsGrid = document.getElementById('products-grid');
        if (productsGrid) {
            // Lance la surveillance des enfants directs et de tous les descendants
            mutationObs.observe(productsGrid, { childList: true, subtree: true });
        }

        // Observe également les product-cards déjà présentes au chargement de la page
        document.querySelectorAll('.product-card').forEach(observeCard);

        console.log('Scroll reveal activé (MutationObserver inclus)'); // Log de confirmation

        // ============================================
        // HEADER SCROLL EFFECT
        // Ajoute une ombre sous le header quand on défile
        // pour renforcer la séparation visuelle
        // ============================================

        const header = document.querySelector('.header'); // Sélectionne l'élément header

        if (header) { // Vérifie que le header existe sur cette page
            // Écoute l'événement scroll sur la fenêtre
            window.addEventListener('scroll', function() {
                if (window.pageYOffset > 50) { // Si on a scrollé de plus de 50px
                    header.style.boxShadow = '0 4px 12px rgba(0,0,0,0.3)'; // Ajoute une ombre portée
                } else {
                    header.style.boxShadow = 'none'; // Supprime l'ombre en haut de page
                }
            });
        }

        // ============================================
        // SEARCH BAR FOCUS ANIMATION
        // Légère mise en avant de la barre de recherche
        // quand l'utilisateur clique dessus
        // ============================================

        const searchInput = document.getElementById('search-input'); // Champ de saisie de recherche
        const searchBar = document.querySelector('.search-bar');     // Conteneur de la barre de recherche

        if (searchInput && searchBar) { // Vérifie que les deux éléments existent
            // Quand l'utilisateur clique sur le champ de recherche
            searchInput.addEventListener('focus', function() {
                searchBar.style.transform = 'scale(1.02)'; // Légère augmentation de taille (zoom 2%)
            });

            // Quand l'utilisateur quitte le champ de recherche
            searchInput.addEventListener('blur', function() {
                // Délai de 200ms pour éviter un effet saccadé lors du clic sur un résultat
                setTimeout(function() {
                    searchBar.style.transform = 'scale(1)'; // Retour à la taille normale
                }, 200);
            });
        }

        // ============================================
        // SMOOTH SCROLL - Défilement fluide
        // Intercepte les clics sur les liens ancres (#)
        // pour animer le défilement au lieu d'un saut brutal
        // ============================================

        // Sélectionne tous les liens dont le href commence par "#" (liens ancres)
        document.querySelectorAll('a[href^="#"]').forEach(anchor => {
            anchor.addEventListener('click', function(e) {
                e.preventDefault(); // Annule le comportement par défaut (saut brutal)
                const target = document.querySelector(this.getAttribute('href')); // Récupère l'élément cible
                if (target) { // Vérifie que la cible existe
                    target.scrollIntoView({
                        behavior: 'smooth', // Animation fluide du défilement
                        block: 'start'      // Aligne le haut de la cible avec le haut de la fenêtre
                    });
                }
            });
        });

        console.log('Animations module initialisé'); // Confirmation que le module est chargé
    });

})(); // Fin de l'IIFE : le code s'exécute immédiatement

IntersectionObserver — observer sans polling

javascript
const observer = new IntersectionObserver(function(entries) {

  // entries = tableau des changements depuis le dernier callback
  // Souvent 1 élément, parfois plusieurs si plusieurs entrent en même temps
  entries.forEach(function(entry) {

    // isIntersecting = true quand l'élément entre dans le viewport
    if (entry.isIntersecting) {

      // Ajouter 'visible' déclenche la transition CSS
      // CSS : .visible { opacity:1; transform:translateY(0) }
      entry.target.classList.add('visible');

      // IMPORTANT : arrêter d'observer — l'élément est déjà animé
      // Sur une page avec 200 éléments : 200 observers en moins
      observer.unobserve(entry.target);
    }
  });

}, {
  // threshold:0.15 = déclenche quand 15% de l'élément est visible
  threshold: 0.15,

  // rootMargin'-50px' = déclenche 50px avant que l'élément soit visible
  rootMargin: '0px 0px -50px 0px'
});

// Attacher l'observer à chaque élément cible
document.querySelectorAll('.animate-on-scroll').forEach(function(el) {
  observer.observe(el);
});

requestAnimationFrame — animation JS synchronisée

javascript
function animateCounter(element, target, duration) {
  const start = performance.now(); // timestamp précis en ms

  function step(currentTime) {
    const elapsed  = currentTime - start;
    const progress = Math.min(elapsed / duration, 1); // 0 → 1

    // Easing : commence vite, ralentit à la fin (courbe ease-out)
    const eased    = 1 - Math.pow(1 - progress, 2);

    element.textContent = Math.round(target * eased);

    // Continuer tant que pas terminé
    if (progress < 1) {
      requestAnimationFrame(step); // prochain frame (~16ms à 60fps)
    }
  }

  requestAnimationFrame(step); // démarrer
}
⚠️
Ne jamais utiliser setInterval(fn, 16) à la place de requestAnimationFrame. setInterval ne se synchronise pas avec le cycle de rendu, créant du tearing (déchirement). rAF s'arrête aussi automatiquement quand l'onglet est en arrière-plan — économie de batterie.

① IntersectionObserver — configuration et callback

javascript
const observer = new IntersectionObserver((entries) => {
  entries.forEach(entry => {
    if (!entry.isIntersecting) return;
    // isIntersecting = true quand l'élément entre dans le viewport.
    // = false quand il sort. On ne veut agir qu'à l'entrée.

    entry.target.classList.add('visible');
    // Ajouter 'visible' déclenche la transition CSS :
    // opacity: 0 → opacity: 1, transform: translateY(20px) → translateY(0)

    observer.unobserve(entry.target);
    // Arrête d'observer cet élément. Il est déjà visible, plus besoin.
    // Économie de ressources : le callback ne sera plus appelé pour lui.
  });
}, {
  threshold: 0.15,
  // L'élément doit être visible à 15% avant de déclencher.
  // 0 = dès 1px, 1 = entièrement visible.
  // 0.15 = légèrement dans le viewport → animation dès qu'on aperçoit l'élément.

  rootMargin: '0px 0px -50px 0px'
  // Réduit le viewport de 50px en bas.
  // L'animation se déclenche 50px avant le bas de l'écran.
  // Format : 'top right bottom left' comme margin CSS.
});

const targets = document.querySelectorAll(
  '.fade-in, .slide-up, .scale-in, [data-animate]'

// Sélecteur multiple : tous les éléments animables d'un coup.
// querySelectorAll retourne une NodeList (pas un Array).

targets.forEach(el => {
  const delay = el.dataset.delay || 0;
  // data-delay="200" sur le HTML → délai de 200ms avant l'animation.
  // Permet des animations en cascade : el1 à 0ms, el2 à 100ms, el3 à 200ms.

  el.style.transitionDelay = delay + 'ms';
  // Applique le délai CSS. L'animation démarre X ms après l'ajout de 'visible'.

  observer.observe(el);
  // Commence à surveiller cet élément. Le callback sera déclenché
  // quand l'élément atteindra le threshold défini.
});

④ Animation de compteur — requestAnimationFrame

javascript
function animateCounter(el, target, duration) {
  const start     = performance.now();
  // performance.now() : timestamp haute précision en ms depuis le chargement.
  // Plus précis que Date.now() pour les animations.

  const startVal  = 0;

  function step(now) {
    const elapsed  = now - start;
    const progress = Math.min(elapsed / duration, 1);
    // progress va de 0 à 1. Math.min évite de dépasser 1.

    const eased    = 1 - Math.pow(1 - progress, 3);
    // Easing "ease-out cubic" : démarre vite, ralentit à l'arrivée.
    // progress=0   → eased=0     (début)
    // progress=0.5 → eased=0.875 (déjà loin)
    // progress=1   → eased=1     (fin)

    el.textContent = Math.round(startVal + (target - startVal) * eased);
    // Interpolation linéaire * easing = valeur actuelle à afficher.

    if (progress < 1) requestAnimationFrame(step);
    // Tant que l'animation n'est pas finie, demande le prochain frame.
    // Le navigateur appelle step() avant chaque repaint (~16ms à 60fps).
    // Quand progress atteint 1, on arrête de demander des frames.
  }

  requestAnimationFrame(step); // Lance la boucle
}
