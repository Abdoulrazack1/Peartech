Comprendre
carousel.js

Ce fichier gère le défilement automatique du carrousel de la page d'accueil. Il change de slide toutes les 5 secondes en cochant des input[type="radio"] HTML — une approche CSS-first où c'est le CSS qui anime les slides, et le JS qui pilote simplement les cases à cocher.

00 — Avant de commencer

Le principe : JS pilote, CSS anime ¶

Analogie : Le carrousel fonctionne comme un juke-box à boutons. Chaque slide correspond à un bouton radio. Appuyer sur un bouton (cocher le radio) change la chanson (affiche la slide). carousel.js est le mécanisme automatique qui appuie sur le bouton suivant toutes les 5 secondes — mais si quelqu'un appuie manuellement, le chronomètre repart de zéro.
Répartition des responsabilités CSS vs JS
CSS — tout ce qui est visuel
Positionnement des slides
Transition de glissement
#slide1:checked ~ .slide-1 { opacity:1 }
Points indicateurs actifs
Durée de la transition
JS — uniquement la logique
Avancer au slide suivant (modulo)
Démarrer/arrêter l'intervalle
Pause au survol de la souris
Réinitialiser au clic manuel
Sync de currentIndex
Avantage de cette approche : Le carrousel fonctionne même sans JavaScript (CSS pur). Si JS est désactivé ou plante, les slides restent visibles et navigables manuellement via les labels liés aux radios.
Termes essentiels
setInterval
Exécute une fonction de façon répétée à un intervalle fixe (en millisecondes). setInterval(fn, 5000) appelle fn toutes les 5 secondes. Retourne un identifiant numérique utilisé par clearInterval pour arrêter.
clearInterval
Annule l'intervalle dont l'identifiant est passé en paramètre. Sans ça, l'intervalle continue indéfiniment même après avoir appelé startAutoSlide() une deuxième fois — ce qui doublerait la vitesse.
Sans clearInterval — l'empilement des intervalles
startAutoSlide() appelé 3 fois sans clearInterval
Appel 1
Intervalle #1 tick toutes les 5s
Appel 2
Intervalle #2 tick toutes les 5s (2e)
Appel 3
Intervalle #3 tick toutes les 5s (3e)
Résultat
3 nextSlide()/5s = 1 slide/1.67s
Avec clearInterval avant chaque setInterval
Appel 1
clearInterval(undefined) rien · Intervalle #1
Appel 2
clearInterval(#1) tué · Intervalle #2
Appel 3
clearInterval(#2) tué · Intervalle #3
Résultat
1 seul intervalle actif — 1 slide/5s
opérateur modulo %
n % m retourne le reste de la division. 2 % 3 = 2, 3 % 3 = 0. Utilisé ici pour revenir au début : quand on atteint le dernier slide, le slide suivant est le premier ((2+1) % 3 = 0).
radio.checked = true
Cocher programmatiquement un input[type="radio"]. Le CSS qui surveille :checked réagit instantanément comme si l'utilisateur avait cliqué — déclenchant la transition CSS vers ce slide.
mouseenter / mouseleave
Deux événements de survol. mouseenter se déclenche quand la souris entre dans l'élément, mouseleave quand elle sort. Contrairement à mouseover/mouseout, ils ne se propagent pas aux éléments enfants.
.filter(r => r !== null)
Crée un nouveau tableau en excluant les valeurs null. Nécessaire car getElementById retourne null si l'élément n'existe pas. Sans ce filtre, un null dans le tableau ferait planter radio.checked = true.
01 — La technique "radio as state"

Pourquoi utiliser des inputs radio pour un carrousel ¶

C'est une technique CSS classique : les boutons radio stockent l'état actif du carrousel. Le sélecteur CSS :checked permet d'appliquer des styles conditionnels en fonction du radio sélectionné — sans JavaScript, ni variable, ni classe.

Mécanisme CSS :checked — pas de JavaScript pour la visibilité
HTML — 3 radios + 3 slides
<input type="radio" id="slide1" checked>
<input type="radio" id="slide2">
<div class="slide">Slide 1</div>
<div class="slide">Slide 2</div>
CSS — sélecteur :checked + sibling ~
#slide1:checked ~ .slides .slide:nth-child(1) {
opacity: 1; transform: translateX(0);
}
#slide2:checked ~ .slides .slide:nth-child(2) {
opacity: 1; transform: translateX(0);
}
JS fait uniquement : radios[1].checked = true — le CSS prend le relais pour masquer/afficher les slides via la transition.
HTML — structure invisible dans le DOM
<!-- Les radios sont cachés visuellement mais fonctionnels -->
<input type="radio" name="carousel" id="slide1" checked>
<input type="radio" name="carousel" id="slide2">
<input type="radio" name="carousel" id="slide3">
<!-- Les labels (points indicateurs) cochent les radios correspondants -->
<label for="slide1" class="indicator"></label>
<label for="slide2" class="indicator"></label>
<label for="slide3" class="indicator"></label>
CSS — le sélecteur :checked pilote l'affichage

setInterval — le moteur de l'automatisme ¶

stopAutoSlide + startAutoSlide — reset complet de l'intervalle
Sans clearInterval avant re-start
Clic user → startAutoSlide()

Clic user → startAutoSlide()
interval #2 actif (5s)
interval #1 toujours actif

Avec clearInterval avant re-start
Clic user → stopAutoSlide()

Pourquoi clearInterval avant setInterval ?
Sans clearInterval — accumulation
1er mouseenter → stopAutoSlide()
1er mouseleave → interval #1 créé
2e mouseenter → stopAutoSlide()

3e cycle → 3 intervals simultanés
Slide change 3× trop vite
Avec clearInterval — propre
startAutoSlide() appelée

Toujours 1 seul interval actif
Peu importe combien de fois
startAutoSlide() est appelée

t = 0ms

t = 5000ms
nextSlide() currentIndex = (0+1)%3 = 1 radios[1].checked = true
t = 10000ms

javascript
let currentIndex = 0;  // 0 = slide 1, 1 = slide 2, 2 = slide 3
let interval;          // Stocke l'identifiant de l'intervalle actif (ou undefined)
const intervalTime = 5000; // 5 000 ms = 5 secondes function nextSlide() {
currentIndex = (currentIndex + 1) % radios.length; // Avance avec retour au début (modulo) radios[currentIndex].checked = true;              // Coche le radio  CSS change la slide
}
function startAutoSlide() {
if (interval) clearInterval(interval); //  Crucial : évite d'empiler les intervalles interval = setInterval(nextSlide, intervalTime);
}
function stopAutoSlide() {
if (interval) {
clearInterval(interval);
interval = null; // Remet à null pour que les vérifications if(interval) fonctionnent }
}
startAutoSlide(); // Lance dès le chargement
Pourquoi le clearInterval dans startAutoSlide ? Sans lui, appeler startAutoSlide() deux fois créerait deux intervalles parallèles qui se chevauchent. Le carrousel s'emballerait — changer de slide toutes les 2.5s, puis 1.67s, etc. Le clearInterval(interval) garantit qu'il n'existe jamais plus d'un intervalle actif.

Gestion du timer — 3 cas critiques

startAutoSlide()
clearInterval(interval) avant de créer
Évite l'accumulation de timers
Appelé au init + mouseleave
stopAutoSlide()
clearInterval(interval)
interval = null — état propre
Appelé au mouseenter + clic manuel
Clic utilisateur
stop + start immédiat
Repart de 0 après clic — évite le saut brusque
L'utilisateur contrôle le rythme

Pourquoi clearInterval avant setInterval

Sans clearInterval avant de créer un nouveau setInterval : au premier mouseleave après mouseenter, 2 timers tournent → la slide avance 2x plus vite. Au 5ème cycle, 6 timers → la slide avance 6x plus vite. Le clearInterval au début de startAutoSlide() garantit qu'il n'existe toujours qu'un seul timer actif.

05 — Reset au clic manuel

Réinitialiser l'intervalle après une action utilisateur ¶

Quand l'utilisateur clique sur un point indicateur ou une flèche, le chronomètre repart de zéro. Sans ça, la prochaine transition automatique pourrait arriver une fraction de seconde après le clic — une expérience désagréable.

Pourquoi clearInterval avant setInterval — le problème de l'accumulation
Sans clearInterval — accumulation
t=0 : startAutoSlide() intervalle #1
t=2 : clic manuel startAutoSlide()
intervalle #2 créé, #1 toujours actif
t=3 : clic intervalle #3...
Résultat : slides changent 3× plus vite
Avec clearInterval — un seul intervalle
t=0 : startAutoSlide() intervalle #1
t=2 : clic clearInterval(#1) d'abord
puis startAutoSlide() intervalle #2
Un seul actif à tout moment
Résultat : 5s exactes depuis le dernier clic
if (interval) clearInterval(interval); · interval = setInterval(nextSlide, 5000);
javascript
const controls = document.querySelectorAll('.indicator, .carousel-prev, .carousel-next');
controls.forEach(control => {
control.addEventListener('click', function() {
stopAutoSlide();  // Annule l'intervalle en cours startAutoSlide(); // En recrée un nouveau (repart de 5s) });
});
// Synchronisation de currentIndex quand l'utilisateur clique manuellement un radio
radios.forEach((radio, index) => {
radio.addEventListener('change', function() {
if (this.checked) {
currentIndex = index; // Met à jour l'index pour que nextSlide() parte du bon endroit }
});
});

comportement imprévisible
Avec addEventListener 'change'

passe à la slide 1
séquence correcte

La technique radio-as-state

Les inputs radio avec name='carousel' sont mutuellement exclusifs : cocher l'un décoche les autres. Le CSS utilise le sélecteur #slide1:checked ~ .slide-1 pour afficher uniquement la slide correspondant au radio coché. JS ne gère que quel radio est coché — l'affichage est entièrement CSS. C'est la séparation des responsabilités : état (radio), logique (JS), affichage (CSS).

(currentIndex + 1) % radios.length — La rotation circulaire

L'opérateur modulo % garantit que l'index revient à 0 après la dernière slide. Pour 3 slides : (2+1)%3 = 0 → retour au début. Sans modulo, l'index passerait à 3 (inexistant) → radios[3] = undefined → erreur.

07 — Ordre du code

Pourquoi chaque bloc est placé là où il est ¶

carousel.js suit la séquence ressources → état partagé → logique → démarrage → événements. L'état (currentIndex, interval) doit être dans le bon scope pour être partagé entre toutes les fonctions.

1
Isolation + attente DOMIIFE + DOMContentLoaded
Les 3 radios HTML doivent exister avant toute récupération. Les éléments id="slide1/2/3" font partie du HTML — ils n'existent que quand le DOM est parsé. Toute tentative de les récupérer avant DOMContentLoaded retournerait null, le garde clause stopperait tout, et le carrousel ne s'initialiserait jamais.
Sans DOMContentLoaded : document.getElementById('slide1') → null → radios.filter(Boolean) → tableau vide → radios.length !== 3 → return immédiat.
↓ fail fast : vérifier avant d'aller plus loin
2
Ressource + protectionconst radios = […].filter() + if (radios.length !== 3) return
Les radios sont la ressource centrale — le guard clause protège tout le code suivant. Le filtre filter(Boolean) élimine les null si un radio était absent. Le guard clause if (radios.length !== 3) stoppe immédiatement si le HTML a été modifié (radio manquant, page différente). Pattern fail-fast : échouer tôt plutôt que propager l'erreur.
Sans guard clause : radios[0].checked = true avec un tableau vide → TypeError: Cannot set properties of undefined. Difficile à déboguer car l'erreur surgit loin de la cause réelle.
↓ état partagé entre nextSlide, startAutoSlide et stopAutoSlide
3
État partagélet currentIndex = 0 + let interval + const intervalTime = 5000
Ces 3 variables sont partagées entre toutes les fonctions — elles doivent être dans le scope commun. currentIndex persiste entre les appels de nextSlide() pour se souvenir de la slide actuelle. interval est créé par startAutoSlide() et annulé par stopAutoSlide() — les deux fonctions doivent accéder à la même variable.
Si let interval était dans startAutoSlide() : stopAutoSlide() ferait clearInterval(interval) sur une variable locale non définie → clearInterval(undefined) → le carrousel ne s'arrête jamais au survol. Bug invisible jusqu'aux tests.
↓ nextSlide est passée comme callback à setInterval
4
Logique de rotationfunction nextSlide()
Définie avant startAutoSlide() qui la passe à setInterval. startAutoSlide() fait setInterval(nextSlide, intervalTime) — elle référence nextSlide par son nom. Les function declarations sont hoistées (elles pourraient être écrites après), mais l'ordre logique — définir avant utiliser — est maintenu pour la lisibilité et pour éviter toute confusion si on refactorisait en arrow function (non hoistée).
Utilise : currentIndex, radios. Passée à setInterval() dans startAutoSlide().
↓ start avant stop : la chronologie reflète l'usage
5
Contrôle autofunction startAutoSlide() + function stopAutoSlide()
start avant stop — elle crée l'intervalle que stop détruit. startAutoSlide() commence par clearInterval(interval) pour éviter les doublons (pattern guard), puis crée un nouvel intervalle. stopAutoSlide() ne fait qu'annuler ce que startAutoSlide() a créé. L'ordre de définition reflète l'ordre de dépendance logique.
Les deux partagent la variable interval du scope englobant (étape 3).
↓ lancer le carrousel avant d'attacher les événements de pause
6
Démarrage effectifstartAutoSlide() — appel initial
Appelé après toutes les définitions, avant les listeners de pause. Le carrousel doit tourner avant que les événements de pause puissent l'arrêter — on ne peut pas mettre en pause ce qui ne tourne pas encore. Placer cet appel avant les listeners garantit que interval est défini quand stopAutoSlide() essaiera de l'annuler.
L'état initial est toujours établi avant les handlers qui le modifient. Configurer, puis écouter.
↓ pause au survol — nécessite que le carrousel soit actif
7
Pause survolmouseenter / mouseleave sur carouselContainer
Après le démarrage — la pause n'a de sens que si quelque chose tourne. Ces listeners appellent stopAutoSlide() et startAutoSlide(), toutes deux déjà définies. C'est un comportement UX secondaire : empêcher qu'une slide change au moment où l'utilisateur la lit. Non critique au fonctionnement du carrousel.
↓ clic manuel — synchroniser l'index interne
8
Contrôle manuelControls click + radios change — synchronisation de currentIndex
En dernier — c'est le cas d'usage le moins fréquent (interaction volontaire). Le listener change sur les radios synchronise currentIndex quand l'utilisateur clique directement sur un indicateur de slide. Sans cette synchronisation, après un clic sur la slide 3, le prochain défilement automatique repartirait depuis l'index interne (ex: 1) plutôt que depuis 3. Le reset de l'intervalle évite le 'double saut'.
Utilise : stopAutoSlide(), startAutoSlide(), currentIndex. Tous définis avant.
Règle générale de ce fichier

La décision architecturale clé : let interval dans le scope commun (pas dans startAutoSlide) pour que stopAutoSlide puisse l'annuler. Les événements de pause et de clic manuel viennent toujours après le démarrage effectif du système.

Concepts et termes techniques utilisés dans ce fichier ¶

Définitions des termes importants rencontrés dans cette documentation.

Document Object Model
Représentation en mémoire de la page HTML sous forme d'arbre. Chaque balise devient un nœud. JavaScript modifie cet arbre via getElementById(), querySelector(), innerHTML. Modifier le DOM = modifier ce que l'utilisateur voit en temps réel.
Immediately Invoked Function Expression
Fonction qui s'exécute immédiatement à sa déclaration : (function() { ... })(). Crée un scope isolé — tout ce qui est déclaré dedans reste privé et ne pollue pas window. Tous les fichiers JS du projet utilisent ce pattern.
DOMContentLoaded
Événement déclenché quand le HTML est parsé et le DOM prêt — avant images et CSS. Moment idéal pour initialiser les modules JS. Distinct de window.load qui attend tous les assets réseau.
setInterval / clearInterval
setInterval(fn, ms) répète une fonction toutes les ms ms. Retourne un ID. clearInterval(id) stoppe les répétitions. Sans clearInterval avant un re-start, les intervalles s'accumulent en doublon.
dataset
Interface qui expose les attributs data-* d'un élément. el.dataset.productId lit data-product-id. Permet de stocker des données métier directement dans le HTML sans variables globales. La conversion camelCase est automatique : data-product-id → dataset.productId.

Carousels : gestion des états et de la navigation ¶

💡
Analogie — l'index courant : L'index du carousel est comme un marque-page dans un livre. Peu importe combien de pages il y a, tu sais toujours où tu en es. Passer à la suivante, c'est juste déplacer le marque-page — pas réécrire le livre.
💡
Analogie — le modulo pour la boucle infinie : (index + 1) % total est comme une horloge. Après 11h59, il est 12h00, pas 12h60. Le modulo remet le compteur à zéro automatiquement.
🚨 PIÈGE — Modifier le DOM à chaque slide plutôt que déplacer un pointeur
Cacher/montrer les slides en changeant display à chaque clic force un reflow. La bonne approche : garder tous les slides dans le DOM, déplacer seulement un currentIndex et changer la classe active. Le CSS fait le reste avec transition: transform.
🚨 PIÈGE — Auto-play sans pause au hover
Un carousel qui défile automatiquement pendant que l'utilisateur essaie de lire est une mauvaise UX. Toujours pauser l'auto-play sur mouseenter et reprendre sur mouseleave. Stocker l'ID de l'intervalle dans une variable pour pouvoir l'annuler avec clearInterval.
PRO TIP
Ajoute des attributs aria-label sur les boutons précédent/suivant et role="region" aria-label="carousel" sur le container. Les lecteurs d'écran peuvent alors naviguer correctement.
PRO TIP
Pour des transitions fluides, utilise transform: translateX(-${index * 100}%) sur un container qui englobe tous les slides plutôt que d'afficher/masquer chacun. Une seule propriété CSS animée = GPU uniquement.

currentIndex

Le seul état à gérer. Toute la logique découle de ce nombre.

Modulo %

Rend la navigation circulaire sans if/else. (i+1) % n revient à 0 après le dernier.

Dots indicateurs

Synchronisés avec currentIndex. Régénérés ou mis à jour à chaque changement de slide.

setInterval

Auto-play. Toujours stocker l'ID dans une variable pour pouvoir clearInterval au besoin.

// ============================================
// carousel.js - Défilement automatique du carrousel
// Gère le changement automatique des slides toutes
// les 5 secondes via les boutons radio HTML
// ============================================

// Utilisation d'une IIFE (Immediately Invoked Function Expression) pour isoler le code
// et éviter les conflits de variables avec d'autres scripts
(function() {
    'use strict'; // Active le mode strict pour une meilleure sécurité et optimisation

    // Attend que le DOM soit complètement chargé avant d'exécuter le code
    document.addEventListener('DOMContentLoaded', function() {

        // Récupère les trois boutons radio du carrousel par leur ID
        // Utilise un tableau et filtre les éventuels null si un élément est manquant
        const radios = [
            document.getElementById('slide1'), // Premier bouton radio
            document.getElementById('slide2'), // Deuxième bouton radio
            document.getElementById('slide3')  // Troisième bouton radio
        ].filter(r => r !== null); // Garde uniquement les éléments existants (supprime les null)

        // Vérifie qu'on a bien trois radios (si le HTML est correct)
        if (radios.length !== 3) {
            // Affiche un avertissement dans la console si les radios sont introuvables
            console.warn('Carrousel automatique : radios introuvables');
            return; // Arrête l'exécution du script
        }

        let currentIndex = 0; // Index de la slide actuellement affichée (commence à 0)
        let interval; // Variable pour stocker l'identifiant du setInterval (pour pouvoir l'arrêter)
        const carouselContainer = document.querySelector('.carousel-container'); // Récupère le conteneur du carrousel
        const intervalTime = 5000; // Durée entre chaque slide (5000 ms = 5 secondes)

        /**
         * Passe à la slide suivante
         */
        function nextSlide() {
            // Calcule l'index suivant : (index actuel + 1) modulo le nombre total de slides
            // Le modulo permet de revenir à 0 après la dernière slide
            currentIndex = (currentIndex + 1) % radios.length;
            // Coche le bouton radio correspondant à la nouvelle slide
            // Ceci déclenche automatiquement le changement d'image via le CSS
            radios[currentIndex].checked = true;
        }

        /**
         * Démarre le défilement automatique
         */
        function startAutoSlide() {
            // Si un intervalle existe déjà, on le nettoie pour éviter les doublons
            if (interval) clearInterval(interval);
            // Crée un nouvel intervalle qui appelle nextSlide toutes les intervalTime ms
            interval = setInterval(nextSlide, intervalTime);
        }

        /**
         * Arrête le défilement automatique
         */
        function stopAutoSlide() {
            // Si un intervalle existe, on l'arrête et on remet la variable à null
            if (interval) {
                clearInterval(interval);
                interval = null; // Indique qu'il n'y a plus d'intervalle actif
            }
        }

        // Lance le défilement automatique au chargement de la page
        startAutoSlide();

        // Si le conteneur du carrousel existe, on ajoute des écouteurs pour la pause au survol
        if (carouselContainer) {
            // Quand la souris entre dans le carrousel, on arrête le défilement
            carouselContainer.addEventListener('mouseenter', stopAutoSlide);
            // Quand la souris quitte le carrousel, on redémarre le défilement
            carouselContainer.addEventListener('mouseleave', startAutoSlide);
        }

        // Sélectionne tous les contrôles manuels : indicateurs (points), flèches précédent/suivant
        const controls = document.querySelectorAll('.indicator, .carousel-prev, .carousel-next');
        // Pour chaque contrôle, on ajoute un écouteur de clic
        controls.forEach(control => {
            control.addEventListener('click', function() {
                // Lors d'un clic manuel, on réinitialise le minuteur :
                // on arrête l'ancien intervalle et on en redémarre un nouveau
                stopAutoSlide();
                startAutoSlide();
            });
        });

        // Pour chaque bouton radio, on surveille le changement d'état (lorsque l'utilisateur clique sur un indicateur)
        radios.forEach((radio, index) => {
            radio.addEventListener('change', function() {
                // Si ce bouton radio est maintenant coché, on met à jour l'index courant
                if (this.checked) {
                    currentIndex = index; // L'index devient celui de la slide affichée
                }
            });
        });

        // Message de confirmation dans la console pour le débogage
        console.log('Carrousel automatique activé (intervalle 5s)');
    });
})();
