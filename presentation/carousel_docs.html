Comprendre
carousel.js

Ce fichier gère le défilement automatique du carrousel de la page d'accueil. Il change de slide toutes les 5 secondes en cochant des input[type="radio"] HTML — une approche CSS-first où c'est le CSS qui anime les slides, et le JS qui pilote simplement les cases à cocher.

00 — Avant de commencer

Le principe : JS pilote, CSS anime

Analogie : Le carrousel fonctionne comme un juke-box à boutons. Chaque slide correspond à un bouton radio. Appuyer sur un bouton (cocher le radio) change la chanson (affiche la slide). carousel.js est le mécanisme automatique qui appuie sur le bouton suivant toutes les 5 secondes — mais si quelqu'un appuie manuellement, le chronomètre repart de zéro.
Répartition des responsabilités CSS vs JS
CSS — tout ce qui est visuel
Positionnement des slides
Transition de glissement
#slide1:checked ~ .slide-1 { opacity:1 }
Points indicateurs actifs
Durée de la transition
JS — uniquement la logique
Avancer au slide suivant (modulo)
Démarrer/arrêter l'intervalle
Pause au survol de la souris
Réinitialiser au clic manuel
Sync de currentIndex
Avantage de cette approche : Le carrousel fonctionne même sans JavaScript (CSS pur). Si JS est désactivé ou plante, les slides restent visibles et navigables manuellement via les labels liés aux radios.
Termes essentiels
setInterval
Exécute une fonction de façon répétée à un intervalle fixe (en millisecondes). setInterval(fn, 5000) appelle fn toutes les 5 secondes. Retourne un identifiant numérique utilisé par clearInterval pour arrêter.
clearInterval
Annule l'intervalle dont l'identifiant est passé en paramètre. Sans ça, l'intervalle continue indéfiniment même après avoir appelé startAutoSlide() une deuxième fois — ce qui doublerait la vitesse.
Sans clearInterval — l'empilement des intervalles
startAutoSlide() appelé 3 fois sans clearInterval

Avec clearInterval avant chaque setInterval
Appel 1

Appel 2
clearInterval(#1) tué · Intervalle #2
Appel 3
clearInterval(#2) tué · Intervalle #3
Résultat
1 seul intervalle actif — 1 slide/5s
opérateur modulo %
n % m retourne le reste de la division. 2 % 3 = 2, 3 % 3 = 0. Utilisé ici pour revenir au début : quand on atteint le dernier slide, le slide suivant est le premier ((2+1) % 3 = 0).
radio.checked = true
Cocher programmatiquement un input[type="radio"]. Le CSS qui surveille :checked réagit instantanément comme si l'utilisateur avait cliqué — déclenchant la transition CSS vers ce slide.
mouseenter / mouseleave
Deux événements de survol. mouseenter se déclenche quand la souris entre dans l'élément, mouseleave quand elle sort. Contrairement à mouseover/mouseout, ils ne se propagent pas aux éléments enfants.
.filter(r => r !== null)
Crée un nouveau tableau en excluant les valeurs null. Nécessaire car getElementById retourne null si l'élément n'existe pas. Sans ce filtre, un null dans le tableau ferait planter radio.checked = true.
01 — La technique "radio as state"

Pourquoi utiliser des inputs radio pour un carrousel

C'est une technique CSS classique : les boutons radio stockent l'état actif du carrousel. Le sélecteur CSS :checked permet d'appliquer des styles conditionnels en fonction du radio sélectionné — sans JavaScript, ni variable, ni classe.

Mécanisme CSS :checked — pas de JavaScript pour la visibilité
HTML — 3 radios + 3 slides
<input type="radio" id="slide1" checked>
<input type="radio" id="slide2">
<div class="slide">Slide 1</div>
<div class="slide">Slide 2</div>
CSS — sélecteur :checked + sibling ~
#slide1:checked ~ .slides .slide:nth-child(1) {
opacity: 1; transform: translateX(0);
}
#slide2:checked ~ .slides .slide:nth-child(2) {
opacity: 1; transform: translateX(0);
}
JS fait uniquement : radios[1].checked = true — le CSS prend le relais pour masquer/afficher les slides via la transition.

javascript
let currentIndex = 0;  // 0 = slide 1, 1 = slide 2, 2 = slide 3
let interval;          // Stocke l'identifiant de l'intervalle actif (ou undefined)
const intervalTime = 5000; // 5 000 ms = 5 secondes function nextSlide() {
currentIndex = (currentIndex + 1) % radios.length; // Avance avec retour au début (modulo) radios[currentIndex].checked = true;              // Coche le radio  CSS change la slide
}
function startAutoSlide() {
if (interval) clearInterval(interval); //  Crucial : évite d'empiler les intervalles interval = setInterval(nextSlide, intervalTime);
}
function stopAutoSlide() {
if (interval) {
clearInterval(interval);
interval = null; // Remet à null pour que les vérifications if(interval) fonctionnent }
}
startAutoSlide(); // Lance dès le chargement
Pourquoi le clearInterval dans startAutoSlide ? Sans lui, appeler startAutoSlide() deux fois créerait deux intervalles parallèles qui se chevauchent. Le carrousel s'emballerait — changer de slide toutes les 2.5s, puis 1.67s, etc. Le clearInterval(interval) garantit qu'il n'existe jamais plus d'un intervalle actif.

05 — Reset au clic manuel

Réinitialiser l'intervalle après une action utilisateur

Quand l'utilisateur clique sur un point indicateur ou une flèche, le chronomètre repart de zéro. Sans ça, la prochaine transition automatique pourrait arriver une fraction de seconde après le clic — une expérience désagréable.

Pourquoi clearInterval avant setInterval — le problème de l'accumulation
Sans clearInterval — accumulation
t=0 : startAutoSlide() intervalle #1
t=2 : clic manuel startAutoSlide()
intervalle #2 créé, #1 toujours actif
t=3 : clic intervalle #3...
Résultat : slides changent 3× plus vite
Avec clearInterval — un seul intervalle
t=0 : startAutoSlide() intervalle #1
t=2 : clic clearInterval(#1) d'abord
puis startAutoSlide() intervalle #2
Un seul actif à tout moment
Résultat : 5s exactes depuis le dernier clic

Glossaire

Concepts et termes techniques utilisés dans ce fichier

Définitions des termes importants rencontrés dans cette documentation.

Document Object Model
Représentation en mémoire de la page HTML sous forme d'arbre. Chaque balise devient un nœud. JavaScript modifie cet arbre via getElementById(), querySelector(), innerHTML. Modifier le DOM = modifier ce que l'utilisateur voit en temps réel.
Immediately Invoked Function Expression
Fonction qui s'exécute immédiatement à sa déclaration : (function() { ... })(). Crée un scope isolé — tout ce qui est déclaré dedans reste privé et ne pollue pas window. Tous les fichiers JS du projet utilisent ce pattern.
DOMContentLoaded
Événement déclenché quand le HTML est parsé et le DOM prêt — avant images et CSS. Moment idéal pour initialiser les modules JS. Distinct de window.load qui attend tous les assets réseau.
setInterval / clearInterval
setInterval(fn, ms) répète une fonction toutes les ms ms. Retourne un ID. clearInterval(id) stoppe les répétitions. Sans clearInterval avant un re-start, les intervalles s'accumulent en doublon.
dataset
Interface qui expose les attributs data-* d'un élément. el.dataset.productId lit data-product-id. Permet de stocker des données métier directement dans le HTML sans variables globales. La conversion camelCase est automatique : data-product-id → dataset.productId.
