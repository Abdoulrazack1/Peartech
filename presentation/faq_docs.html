Comprendre
faq.js

Ce fichier gère la page FAQ. Il implémente un accordéon animé (ouverture/fermeture avec transition sur la hauteur), une recherche par mot-clé avec debounce, et un filtrage par catégorie — les trois combinés pour n'afficher que les questions pertinentes.

Avant d'animer la hauteur, il faut mettre le contenu dans un état initial précis : height: 0 + overflow: hidden. Et si le wrapper .faq-answer-inner manque dans le HTML, on le crée dynamiquement.

État initial posé par initItems() — 3 propriétés CSS critiques
PropriétéValeur initialePourquoi
height'0'Contenu caché au départ — permet d'animer vers scrollHeight
overflow'hidden'Masque ce qui dépasse pendant l'animation height
display'block'Obligatoire — display:none bloque la transition CSS
javascript
function initItems() {
getItems().forEach(function(item) {
const answer = item.querySelector('.faq-answer');
// Crée le wrapper interne si absent du HTML (compatibilité) let inner = answer.querySelector('.faq-answer-inner');
if (!inner) {
inner = document.createElement('div');
inner.className = 'faq-answer-inner';
while (answer.firstChild) inner.appendChild(answer.firstChild); // Déplace tout answer.appendChild(inner);
}
// État initial : fermé answer.style.height   = '0';
answer.style.overflow = 'hidden';
answer.style.display  = 'block'; // display:block requis pour que height s'anime item.querySelector('.faq-question').addEventListener('click', function() {
if (item.classList.contains('active')) {
closeItem(item);
} else {
// Ferme tous les autres d'abord (un seul ouvert à la fois) getItems().forEach(other => {
if (other !== item && other.classList.contains('active')) closeItem(other);
});
openItem(item);
}
});
});
}
// getItems() est une fonction, pas une variable — pour toujours avoir
// les items actuels même si le DOM change après l'initialisation

return Array.from(faqList.querySelectorAll('.faq-item'));
}

02 — Animation de hauteur

Le problème de height:auto — impossible à animer directement en CSS

En CSS, on ne peut pas écrire transition: height 0.3s de 0 vers auto. Le navigateur ne sait pas calculer le point d'arrivée. La solution : mesurer la hauteur réelle en JS, puis animer vers cette valeur fixe.

openItem() — les 4 étapes pour animer height:0 height:auto
①
height = 'auto' le navigateur calcule la vraie hauteur
②
scrollHeight lu ex: 148px (valeur fixe mémorisée)
③
height = '0' retour à 0, instantané (pas de transition)
④
requestAnimationFrame height = '148px' CSS transition s'active : 0 148px en 0.3s
Le passage par height='0' avant rAF est obligatoire. Sans lui, le navigateur ne voit pas de changement d'état et n'anime pas la transition.
javascript — openItem() — la technique en 4 étapes
function openItem(item) {
const answer = item.querySelector('.faq-answer');
item.classList.add('active');
// Étape 1 : passe en height:auto pour MESURER answer.style.height = 'auto';
const targetHeight = answer.scrollHeight + 'px'; // Ex: "248px" // Étape 2 : remet à 0 pour partir du bon point de départ answer.style.height = '0';
// Étape 3 : FORCE le navigateur à recalculer le DOM (reflow) // Sans ça, le navigateur optimise et saute l'étape 2  pas d'animation answer.getBoundingClientRect();
// Étape 4 : installe la transition et lance l'animation answer.style.transition = 'height 0.32s cubic-bezier(0.4, 0, 0.2, 1)';
answer.style.height     = targetHeight; //  animé de 0 vers 248px // Après l'animation : repasse en auto pour s'adapter au contenu dynamique answer.addEventListener('transitionend', function onEnd() {
answer.removeEventListener('transitionend', onEnd); // Usage unique if (item.classList.contains('active')) answer.style.height = 'auto';
});
}

javascript — closeItem() — l'inverse en 3 étapes
function closeItem(item) {
const answer = item.querySelector('.faq-answer');
// Étape 1 : fige la hauteur actuelle (si elle est en 'auto', on ne peut pas partir d'auto) answer.style.height = answer.scrollHeight + 'px';
// Étape 2 : force le reflow answer.getBoundingClientRect();
// Étape 3 : transition vers 0 (légèrement plus rapide qu'à l'ouverture) answer.style.transition = 'height 0.28s cubic-bezier(0.4, 0, 0.2, 1)';
answer.style.height     = '0';
item.classList.remove('active'); // Retire la classe immédiatement (styles CSS) // Pas besoin de transitionend ici : height:0 est la position finale souhaitée
}
getBoundingClientRect() force ce qu'on appelle un reflow — le navigateur recalcule la géométrie du DOM. Sans ça, le navigateur peut "batching" les changements de style et fusionner height:0 et height:248px en un seul saut, sans transition visible. Cet appel "force" le navigateur à valider l'état height:0 avant de lancer l'animation.

javascript — la logique de filtrage
function applyFilters() {
const term = searchInput ? searchInput.value.toLowerCase().trim() : '';
const category = document.querySelector('.category-btn.active')?.dataset.category || 'all';
let visibleCount = 0;
getItems().forEach(function(item) {
const qText = item.querySelector('.faq-question-text').textContent.toLowerCase();
const aText = item.querySelector('.faq-answer').textContent.toLowerCase();
const iCat = item.dataset.category || '';
const catOk = category === 'all' || iCat === category;
const termOk = term === '' || qText.includes(term) || aText.includes(term);
const show = catOk && termOk; // Les DEUX conditions doivent être vraies item.style.display = show ? '' : 'none';
if (!show && item.classList.contains('active')) closeItem(item); // Ferme si caché if (show) visibleCount++;
});
noResults.classList.toggle('visible', visibleCount === 0); // Message si 0 résultats
}
Logique de filtrage — pour chaque item
catOk
category === 'all' tout passe
sinon item.dataset.category === category sélectionnée
termOk
term === '' tout passe
sinon term dans la question OU la réponse
show = catOk && termOk — les deux doivent être true

Debounce — ne pas filtrer à chaque frappe

javascript
let debounceTimer;
searchInput.addEventListener('input', function() {
clearTimeout(debounceTimer);          // Annule le timer précédent debounceTimer = setTimeout(applyFilters, 200); // Lance après 200ms de silence
});
// "Le"  annulé / "Les"  annulé / "Les p"  annulé / "Les pr"  200ms  applyFilters()
// Sans debounce : applyFilters s'appellerait 6 fois pour "Les pr" au lieu d'1 fois.
clearTimeout sur un ID déjà expiré ne fait rien (pas d'erreur). Donc la première frappe fait clearTimeout(undefined) no-op, puis installe le timer. C'est un pattern parfaitement sûr.

Glossaire

Concepts et termes techniques utilisés dans ce fichier

Définitions des termes importants rencontrés dans cette documentation.

Document Object Model
Représentation en mémoire de la page HTML sous forme d'arbre. Chaque balise devient un nœud. JavaScript modifie cet arbre via getElementById(), querySelector(), innerHTML. Modifier le DOM = modifier ce que l'utilisateur voit en temps réel.
Immediately Invoked Function Expression
Fonction qui s'exécute immédiatement à sa déclaration : (function() { ... })(). Crée un scope isolé — tout ce qui est déclaré dedans reste privé et ne pollue pas window. Tous les fichiers JS du projet utilisent ce pattern.
DOMContentLoaded
Événement déclenché quand le HTML est parsé et le DOM prêt — avant images et CSS. Moment idéal pour initialiser les modules JS. Distinct de window.load qui attend tous les assets réseau.
scrollHeight
Hauteur totale d'un élément incluant la partie non visible (défilable). Toujours >= hauteur visible. Utilisé pour mesurer la hauteur réelle d'un contenu avant de l'animer : height:0 → height:scrollHeight+'px'.
debounce
Regroupe plusieurs appels rapides en un seul différé. Frappe 5 lettres en 200ms → 1 seule recherche lancée 300ms après la dernière frappe. Implémenté avec clearTimeout(timer) + timer = setTimeout(fn, 300).
dataset
Interface qui expose les attributs data-* d'un élément. el.dataset.productId lit data-product-id. Permet de stocker des données métier directement dans le HTML sans variables globales. La conversion camelCase est automatique : data-product-id → dataset.productId.
classList
Interface de manipulation des classes CSS. add(), remove(), toggle(), contains(). Plus fiable que modifier className comme une string. toggle('active', condition) ajoute ou retire selon un booléen.
