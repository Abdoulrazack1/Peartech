data.js

Ce fichier est la base de données centrale du site. Il contient toutes les catégories et tous les produits, expose des fonctions pour les interroger, et rend le tout accessible via window.PearTechDB. Tous les autres scripts en dépendent et doivent être chargés après lui.

L'IIFE — pourquoi encapsuler dans une fonction anonyme

Comme tous les scripts du projet, data.js s'enveloppe dans une IIFE (Immediately Invoked Function Expression). Cela protège les variables internes de l'espace global.

Sans IIFE vs avec IIFE — protection du scope
Sans IIFE — dangereux
const categories = […];
const products = […];

// window.categories = [… // accessible !]
// window.products = [… // accessible !]
// N'importe quel autre script peut
// modifier ou écraser les tableaux
Avec IIFE — sécurisé
(function() {
  const categories = […];
  const products = […];
  // privés — invisibles depuis window
  window.PearTechDB = { …helpers };
  // seul cet objet est accessible
})();
Les variables categories et products restent privées. L'extérieur ne peut accéder aux données qu'à travers les méthodes contrôlées de window.PearTechDB — comme un objet avec des getters.

La structure options — objet imbriqué

Le champ options est le plus complexe. C'est un objet dont les clés sont les noms des groupes d'options, et les valeurs sont des tableaux de choix possibles avec leur surcoût.

Exemple complet du champ options d'un MacBook
options: {
  ram: [
    { label: '8 Go', price: 0 },
    { label: '16 Go', price: 200 },
    { label: '24 Go', price: 400 }
  ],
  storage: [
    { label: '256 Go', price: 0 },
    { label: '512 Go', price: 230 }
  ]
}
product_detail.js lit Object.entries(options) pour générer les boutons de chaque groupe. Le price est mis en data-price sur chaque bouton.
panier.js récupère les options actives dans le DOM et les stocke dans cartItem.options pour la détection de doublons via JSON.stringify.
cart.js affiche item.options.map(opt => opt.name + ': ' + opt.value) pour décrire la configuration choisie.

04 — Recherche textuelle

search(query) — 3 champs simultanément

search(query) — 3 champs vérifiés en même temps, toLowerCase partout
query = "iphone pro" q = "iphone pro" (toLowerCase)
pour chaque produit dans products[]
name.toLowerCase()
"iphone 15 pro max" contient "iphone pro" inclus
category.toLowerCase()
"smartphones" ne contient pas "iphone pro" on continue
tags.join(' ')
"iphone apple 5g pro camera" contient "iphone pro"
Logique OR : suffit qu'UN des 3 champs match pour inclure le produit · toLowerCase des deux côtés = recherche insensible à la casse
javascript
search: function(query) {
const q = query.toLowerCase();  // normalise : "MAC"  "mac" return products.filter(p => p.name.toLowerCase().includes(q) ||        // ex: "macbook air m2" inclut "mac"  p.description.toLowerCase().includes(q) || // si le name ne matche pas, tente ici p.tags.some(tag => tag.includes(q))        // ["macbook","m2"]  "macbook" inclut "mac"  );
}
search("mac") — analyse sur chaque produit
1 — name
"MacBook Air M2"
inclut "mac"
2 — description
"Le MacBook Air le plus…"
Si name ne matche pas
3 — tags[]
["macbook", "m2"]
.some() vérifie chaque tag
Le test s'arrête dès le premier true grâce au ||. Si name matche, description et tags ne sont pas vérifiés.

Glossaire

Concepts et termes techniques utilisés dans ce fichier

Définitions des termes importants rencontrés dans cette documentation.

Document Object Model
Représentation en mémoire de la page HTML sous forme d'arbre. Chaque balise devient un nœud. JavaScript modifie cet arbre via getElementById(), querySelector(), innerHTML. Modifier le DOM = modifier ce que l'utilisateur voit en temps réel.
JSON — JavaScript Object Notation
Format texte pour sérialiser des objets. JSON.stringify(obj) → string stockable. JSON.parse(str) → objet utilisable. Obligatoire pour stocker/lire des objets dans localStorage ou sessionStorage.
Template literal
Chaîne avec backtick `...` qui permet d'insérer des expressions avec ${variable} et des retours à la ligne. Remplace la concaténation de strings. Utilisé massivement pour générer le HTML des cards dynamiquement.
Scope (portée)
Contexte d'accessibilité d'une variable. let/const → block scope (limité au bloc {}). var → function scope (remonte à la fonction). Les IIFE créent un scope fermé pour éviter les collisions de noms entre modules.
