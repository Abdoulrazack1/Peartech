Comprendre
product_detail.js

Ce fichier gère entièrement la page fiche produit. Il lit l'identifiant dans l'URL, charge le bon produit depuis la base de données, génère tout le HTML de la page, puis initialise les onglets, la galerie d'images avec lightbox, le système de prix dynamique selon les options, les produits similaires, et les notes personnelles.

product_detail.js · 10 fonctions · ~639 lignes · une seule page
00 — Avant de commencer

De l'URL vide à la fiche produit complète

product_detail.js transforme une page HTML quasi-vide en fiche produit complète. Il lit l'URL pour savoir quel produit afficher, génère tout le contenu dynamiquement, puis branche les comportements interactifs un par un.

🎬
Analogie : C'est comme un décorateur de plateau de tournage. La page HTML est un plateau vide avec juste quelques points d'ancrage (#product-detail, #breadcrumb…). product_detail.js arrive avec tous les accessoires (données du produit) et aménage entièrement le décor, puis installe tous les mécanismes interactifs.

Vocabulaire essentiel — lisez ça en premier

📖 Termes utilisés dans ce fichier
URLSearchParams
new URLSearchParams(window.location.search) analyse la partie "?id=42" de l'URL. .get('id') retourne la valeur "42" sous forme de texte, ou null si le paramètre n'existe pas.
window.location.search
La chaîne de requête de l'URL courante : pour page_produit.html?id=42, window.location.search vaut "?id=42". C'est depuis là que le fichier sait quel produit afficher.
Template literal
Chaîne délimitée par des backticks ( ` ) qui permet d'insérer des variables avec ${variable} et de s'étendre sur plusieurs lignes. `Bonjour ${prenom} !` est bien plus lisible que 'Bonjour ' + prenom + ' !'.
Object.entries()
Object.entries({ram:"8Go", storage:"256Go"}) retourne [["ram","8Go"], ["storage","256Go"]] — un tableau de paires [clé, valeur]. Utilisé pour générer les boutons d'options dynamiquement.
for...of
Boucle qui parcourt les éléments d'un tableau un par un. for (const [key, values] of Object.entries(options)) déstructure chaque paire en deux variables key et values.
Lightbox
Superposition (overlay) qui affiche une image en grand format au-dessus de la page. L'arrière-plan est assombri. L'utilisateur peut naviguer entre les images, fermer via Échap, ou swiper sur mobile.
touchstart / touchend
Événements déclenchés sur les appareils tactiles. touchstart enregistre la position du doigt au début du geste. touchend calcule la distance parcourue. Si > 50px vers la gauche → image suivante. Vers la droite → précédente.
?. (optional chaining)
document.querySelector('.titre')?.textContent : si querySelector retourne null, le ?. court-circuite et renvoie undefined au lieu de lancer une erreur. Évite le classique "Cannot read properties of null".

// URLSearchParams analyse la chaîne "?id=42"
const urlParams  = new URLSearchParams(window.location.search);
const productId = urlParams.get('id');
// urlParams.get('id') = "42" (toujours un texte, jamais un nombre)
// C'est pourquoi plus loin on fait parseInt(productId) pour la recherche en DB

if (!productId) {
    window.location.href = 'page_accueil.html';
    return;  // return OBLIGATOIRE : window.location.href ne stoppe pas l'exécution !
}

const product = PearTechDB.getProductById(parseInt(productId));
// parseInt("42") → 42 (nombre)
// La DB cherche par id numérique, pas par texte
⚠️
window.location.href ne stoppe pas le code ! Écrire window.location.href = 'page_accueil.html' demande au navigateur de naviguer, mais le code JavaScript CONTINUE de s'exécuter pendant quelques millisecondes. Sans le return juste après, les lignes suivantes s'exécuteraient aussi — et planteraient puisque productId est null.

Génération conditionnelle des options

javascript
let optionsHtml = '';

if (product.options && Object.keys(product.options).length > 0) {
    // product.options = { ram: [{label:"8Go",price:0}, {label:"16Go",price:150}], storage: [...] }
    // Object.keys() retourne ["ram", "storage"] — vérifie que ce n'est pas vide

    optionsHtml = '<div class="product-options">';

    for (const [key, values] of Object.entries(product.options)) {
        // key = "ram", values = [{label:"8Go",price:0}, {label:"16Go",price:150}]
        // Destructuration : chaque entrée de [["ram", [...]], ["storage", [...]]] est
        // décomposée en deux variables [key, values]

        const optionLabel = key === 'ram' ? 'Mémoire RAM' :
                           key === 'storage' ? 'Stockage' :
                           key === 'color' ? 'Couleur' :
                           key.charAt(0).toUpperCase() + key.slice(1);
        // Traduction des clés techniques en labels lisibles
        // Dernier cas : si clé inconnue, juste mettre la 1ère lettre en majuscule

        optionsHtml += `
            <div class="option-group" data-option-group="${key}">
                <span class="option-label">${optionLabel}:</span>
                <div class="option-buttons">
                    ${values.map((opt, index) =>
                        `<button class="option-btn ${index === 0 ? 'active' : ''}"
                                  data-price="${opt.price}"
                                  data-value="${opt.label}">${opt.label}</button>`
                    ).join('')}
                </div>
            </div>
        `;
        // index === 0 ? 'active' : '' → le premier bouton de chaque groupe est actif par défaut
    }
    optionsHtml += '</div>';
}

03 — Onglets

initTabs() — afficher un panneau à la fois

Les onglets (Description, Fiche technique, Avis, FAQ, Notes) suivent le même pattern que filtre.js : retirer active de tout, puis l'ajouter sur la cible.

Les 5 onglets — data-tab → #tab-{id}

①Clic sur "Fiche technique" → tabId = 'specs'
②tabBtns.forEach → classList.remove('active') sur tous
③tabContents.forEach → classList.remove('active') sur tous
④btn.classList.add('active') → bouton "Fiche technique" marqué
⑤getElementById('tab-specs').classList.add('active') → panneau visible
javascript
function initTabs() {
    const tabBtns     = document.querySelectorAll('.tab-btn');
    const tabContents = document.querySelectorAll('.tab-content');

    tabBtns.forEach(btn => {
        btn.addEventListener('click', () => {
            const tabId = btn.dataset.tab;
            // data-tab="specs" → tabId = "specs"

            tabBtns.forEach(b => b.classList.remove('active'));     // 1. Tout effacer
            tabContents.forEach(c => c.classList.remove('active')); // 2. Cacher tous les panneaux
            btn.classList.add('active');                              // 3. Marquer le bouton cliqué

            const activeTab = document.getElementById(`tab-${tabId}`);
            if (activeTab) activeTab.classList.add('active');          // 4. Afficher le panneau
            // template literal : `tab-${"specs"}` → "tab-specs" → cherche #tab-specs
        });
    });
}

Création de l'overlay — une seule fois

function initLightbox(product) {
    // Garde : si l'overlay existe déjà, on ne le recrée pas
    if (document.getElementById('lightbox-overlay')) return;

    const overlay = document.createElement('div');
    overlay.id        = 'lightbox-overlay';
    overlay.className = 'lightbox-overlay';
    overlay.innerHTML = `
        <div class="lightbox-container">
            <button class="lightbox-close">✕</button>
            <button class="lightbox-prev">←</button>
            <img class="lightbox-img" id="lightbox-img">
            <button class="lightbox-next">→</button>
            <div class="lightbox-counter"></div>
        </div>
    `;
    document.body.appendChild(overlay);
    // L'overlay est ajouté au body et reste dans le DOM toujours.
    // Il est visible seulement quand il a la classe "active".
}

Navigation circulaire — l'opérateur modulo %

javascript
// Navigation "suivant" — circulaire grâce au modulo
currentIndex = (currentIndex + 1) % images.length;

// Exemple avec 4 images (index 0 à 3) :
// currentIndex=0 → (0+1) % 4 = 1  ✓
// currentIndex=2 → (2+1) % 4 = 3  ✓
// currentIndex=3 → (3+1) % 4 = 0  ← revient au début

// Navigation "précédent" — légèrement plus complexe
currentIndex = (currentIndex - 1 + images.length) % images.length;

// Pourquoi + images.length ?
// currentIndex=0 → (0-1) % 4 = -1 en JS → problème !
// Avec correction : (0-1+4) % 4 = 3 % 4 = 3 ← va à la dernière image ✓
// currentIndex=2 → (2-1+4) % 4 = 5 % 4 = 1 ✓

Swipe tactile — détection du glissement

javascript
let touchStartX = 0;

overlay.addEventListener('touchstart', (e) => {
    touchStartX = e.touches[0].clientX;
    // e.touches[0] = premier doigt en contact. clientX = position horizontale en pixels
}, { passive: true });
// { passive: true } = on promet de ne pas appeler e.preventDefault()
// → le navigateur peut gérer le scroll en parallèle, plus fluide

overlay.addEventListener('touchend', (e) => {
    const diff = touchStartX - e.changedTouches[0].clientX;
    // diff > 0 : doigt a glissé VERS LA GAUCHE → image suivante
    // diff < 0 : doigt a glissé VERS LA DROITE → image précédente

    if (Math.abs(diff) > 50) {  // seuil : au moins 50 pixels de déplacement
        if (diff > 0) {
            currentIndex = (currentIndex + 1) % images.length;
        } else {
            currentIndex = (currentIndex - 1 + images.length) % images.length;
        }
        updateLightbox(currentIndex);
    }
}, { passive: true });

05 — Prix dynamique

initOptions() — le prix se recalcule à chaque sélection

Quand l'utilisateur choisit une option (16 Go RAM, 512 Go SSD…), le prix affiché se met à jour immédiatement. La logique est simple : prix de base + somme des surcoûts des options actives.

Calcul du prix avec options — exemple MacBook
Prix de base
product.basePrice = 1 299 €
Option RAM sélect.
16 Go → dataset.priceAddon = +200
Option SSD sélect.
512 Go → dataset.priceAddon = +100
Prix affiché
1299 + 200 + 100 = 1 599 €
querySelectorAll('.option-btn.active') → reduce((sum, btn) => sum + Number(btn.dataset.priceAddon || 0), basePrice) → innerHTML mis à jour
javascript
function initOptions(product) {
    const optionGroups = document.querySelectorAll('.option-group');
    const priceElement = document.getElementById('current-price');
    if (!priceElement) return;

    let basePrice = product.basePrice || product.price;
    // basePrice || product.price : si basePrice n'existe pas → utiliser price

    function updatePrice() {
        let total = basePrice;

        optionGroups.forEach(group => {
            const activeBtn = group.querySelector('.option-btn.active');
            if (activeBtn && activeBtn.dataset.price) {
                total += parseFloat(activeBtn.dataset.price);
                // data-price="150" → 150 (surcoût de l'option active)
            }
        });

        priceElement.textContent = total.toFixed(2).replace('.', ',') + ' €';
        // toFixed(2) = toujours 2 décimales : 999 → "999.00"
        // .replace('.', ',') = format français : "999.00" → "999,00"

        const paymentInfo = document.querySelector('.payment-info');
        if (paymentInfo) {
            paymentInfo.textContent = `ou 12 × ${(total/12).toFixed(2).replace('.', ',')} € sans frais`;
        }
    }

    optionGroups.forEach(group => {
        const btns = group.querySelectorAll('.option-btn');
        btns.forEach(btn => {
            btn.addEventListener('click', () => {
                btns.forEach(b => b.classList.remove('active'));
                btn.classList.add('active');
                updatePrice();
            });
        });
    });

    updatePrice();  // Calcule le prix initial (options par défaut)
}

06 — Produits similaires

generateSimilarProducts() — même catégorie, sauf le courant

javascript
function generateSimilarProducts(currentProduct) {
    const grid = document.getElementById('similar-products-grid');
    if (!grid) return;

    const categorySlug = getCategorySlugFromId(currentProduct.categoryId);

    let similar = [];
    if (categorySlug && PearTechDB.getProductsByCategory) {
        similar = PearTechDB
            .getProductsByCategory(categorySlug)  // tous les produits de cette catégorie
            .filter(p => p.id !== currentProduct.id)    // exclure le produit courant
            .slice(0, 6);                               // max 6 produits
    }

    if (similar.length === 0) {
        grid.innerHTML = '<p>Aucun produit similaire pour le moment.</p>';
        return;
    }

    grid.innerHTML = similar.map(p => `
        <article class="product-card" data-product-id="${p.id}">
            <a href="page_produit.html?id=${p.id}" class="product-image">
                <img src="${p.images[0]}" onerror="this.src='/asset/image/no-image.png'">
            </a>
            …
        </article>
    `).join('');
    // .map() transforme chaque produit en HTML
    // .join('') colle tous les morceaux ensemble sans séparateur
}

07 — Notes personnelles

initNotes() — sauvegarder des notes par produit

L'onglet "Mes notes" permet à l'utilisateur de saisir du texte lié à ce produit. Les notes sont sauvegardées dans localStorage avec une clé unique par produit. Elles persistent entre les sessions.

Pipeline generateSimilarProducts(currentProduct)
source
PearTechDB.getProductsByCategory(slug) → tous les produits de la catégorie
filtre
.filter(p => p.id !== currentProduct.id) → exclu lui-même
limite
.slice(0, 6) → max 6 produits similaires
rendu
similar.length === 0 → section cachée · sinon → buildCard() × N → grid
🔤
Clé unique par produit : 'peartech-notes-42' pour l'iPhone id=42, 'peartech-notes-7' pour le MacBook id=7. En ajoutant l'id à la clé, chaque produit a son propre espace de stockage. Sans ça, toutes les fiches produits partageraient la même note — et s'effaceraient mutuellement.
Isolation des notes dans localStorage — une clé par produit
page_produit.html?id=5
clé localStorage :
'peartech-notes-5'
Valeur : "À comparer avec l'iPhone 15 Pro…"
page_produit.html?id=42
clé localStorage :
'peartech-notes-42'
Valeur : "Prix vu moins cher sur Cdiscount…"
page_produit.html?id=99
clé localStorage :
'peartech-notes-99'
Valeur : "" (aucune note)
Chaque visite sur une fiche produit charge uniquement sa propre clé. Les notes de l'iPhone ne contaminent jamais celles du MacBook.
javascript
function initNotes() {
    const urlId = new URLSearchParams(window.location.search).get('id');
    if (!urlId) return;

    const STORAGE_KEY = 'peartech-notes-' + urlId;
    // Exemple : 'peartech-notes-42' — unique par produit

    const textarea  = document.getElementById('notes-textarea');
    const saveBtn   = document.getElementById('notes-btn-save');
    const clearBtn  = document.getElementById('notes-btn-clear');
    const charCount = document.getElementById('notes-char-count');
    const indicator = document.getElementById('notes-saved-indicator');

    // Charger la note existante pour ce produit
    const saved = localStorage.getItem(STORAGE_KEY) || '';
    textarea.value = saved;
    updateCharCount(saved.length);

    // Raccourci Ctrl+S / Cmd+S (Mac)
    textarea.addEventListener('keydown', function(e) {
        if ((e.ctrlKey || e.metaKey) && e.key === 's') {
            e.preventDefault();  // empêche le dialogue "Enregistrer la page" du navigateur
            localStorage.setItem(STORAGE_KEY, textarea.value);
            showIndicator('✓ Sauvegardée', false);
        }
    });
}

Étape 1 — DOMContentLoaded
Lecture de l'URL
URLSearchParams extrait l'id depuis ?id=42. Si absent → redirection accueil. Si PearTechDB absent → message erreur. Si produit introuvable → "Produit introuvable".
Étape 2
Génération de la page
getCategorySlugFromId() trouve la catégorie. breadcrumb.innerHTML construit le fil d'Ariane. generateProductHTML(product) génère tout le HTML de la fiche et l'injecte dans #product-detail.
Étape 3
Initialisation des boutons d'action
Le bouton "Acheter maintenant" reçoit son handler : simule un clic sur "Ajouter au panier", puis redirige vers la page panier après 300ms. Le bouton wishlist synchronise son aria-label et son title via MutationObserver.
Étape 4
Initialisation des 5 modules
initTabs() : onglets de la fiche produit.
initThumbnails() : galerie + lightbox avec navigation, clavier, swipe.
initOptions() : prix recalculé à chaque sélection d'option.
generateSimilarProducts() : injecte 6 produits de la même catégorie.
initNotes() : notes persistantes en localStorage.
En permanence
Interactions utilisateur
Chaque module gère ses propres événements indépendamment. Les modules communiquent peu entre eux : initOptions met à jour le prix dans le DOM que panier.js lira plus tard. initThumbnails sync l'index courant que la lightbox utilise.
