URLSearchParams
API navigateur pour lire les paramètres d'une URL. new URLSearchParams(window.location.search) sur ?categorie=apple permet d'écrire params.get('categorie') 'apple'.
Map
Structure de données clévaleur. Contrairement à un objet, les clés peuvent être de n'importe quel type et l'ordre d'insertion est garanti. Utilisée ici pour compter les produits par sous-catégorie ou marque.

Collection sans doublons. featSet.add('5G') plusieurs fois ne l'insère qu'une seule fois. Utilisé pour lister les caractéristiques disponibles.
Object.values().flat()
Object.values(obj) retourne les valeurs d'un objet en tableau. .flat() aplatit un tableau de tableaux en un seul. Ex : {a:[1,2], b:[3]} [1,2,3].
prédicat
Fonction qui retourne true ou false. Dans SUBCAT_MATCHERS, chaque entrée est un prédicat : 'iPhone': p => p.tags.includes('iphone') retourne vrai si le produit a le tag "iphone".
.every()
Retourne true si tous les éléments du tableau satisfont la condition (AND logique). Utilisé pour les features : le produit doit posséder toutes les caractéristiques cochées.
.some()
Retourne true si au moins un élément satisfait la condition (OR logique). Utilisé pour les sous-catégories : le produit doit appartenir à au moins une des sous-catégories cochées.
aria-*
Attributs HTML d'accessibilité pour les lecteurs d'écran. aria-label donne un nom vocal à un élément, aria-current="page" indique la page active, aria-pressed indique l'état d'un bouton toggle.

Pourquoi les variables d'état sont hors de DOMContentLoaded

Les variables currentCategory, allProducts, currentPage, etc. sont déclarées dans le scope IIFE mais à l'extérieur de DOMContentLoaded. Elles doivent être accessibles depuis plusieurs fonctions (applyFilters(), updatePagination(), buildFilters()…) qui sont toutes définies dans le même scope.

javascript
// Scope IIFE — accessible par toutes les fonctions
let currentCategory  = null;
let allProducts      = [];
let currentPage      = 1;
const PRODUCTS_PER_PAGE = 12;
let currentFilters   = { brands: [], price: null, features: [] };
let currentSort      = 'default';

document.addEventListener('DOMContentLoaded', function() {
    // Ici on initialise les refs DOM
    // et on attache les listeners
    // qui lisent/modifient les variables ci-dessus
});
💡
Si let currentPage = 1 était déclarée dans DOMContentLoaded, elle ne serait accessible que depuis ce callback et les fonctions définies à l'intérieur. updatePagination() définie ailleurs ne pourrait pas la lire. Le scope IIFE sert de 'module state' partagé entre toutes les fonctions du fichier.

Lecture de l'URL pour détecter la catégorie

new URLSearchParams(window.location.search).get('category') lit le paramètre ?category=apple de l'URL. Si absent, currentCategory reste null — la page affiche tous les produits sans filtre de catégorie. C'est le point d'entrée dynamique : la même page HTML sert toutes les catégories selon l'URL.

La dépendance à PearTechDB — deux scénarios

javascript
if (window.PearTechDB) {
    allProducts = PearTechDB.products; // data.js déjà chargé
    buildFilters();
    applyFilters();
} else {
    document.addEventListener('PearTechDBReady', function() {
        allProducts = PearTechDB.products;
        buildFilters();
        applyFilters();
    });
}

Si data.js est chargé avant catalogue.js (ordre normal des scripts), la branche if s'exécute immédiatement. Si pour une raison data.js chargait après (async, réseau lent), la branche else attend le signal. La page ne reste jamais bloquée dans un état vide.

buildFilters() vs applyFilters() — deux responsabilités distinctes

buildFilters() construit la sidebar statique des options de filtre (marques, prix, fonctionnalités). Elle s'exécute une seule fois à l'init — les options disponibles ne changent pas pendant la session.
applyFilters() filtre, trie, pagine et affiche les produits. Elle s'exécute à chaque interaction (changement de filtre, de tri, de page). La séparation évite de reconstruire la sidebar inutilement à chaque filtre.

getBrand() — détecter la marque via les tags

javascript
const BRAND_MAP = {
'apple': 'Apple', 'iphone': 'Apple', 'ipad': 'Apple',
'samsung': 'Samsung', 'google': 'Google', 'pixel': 'Google',
// … 12 marques au total
};
function getBrand(product) {
for (const tag of product.tags) {
const b = BRAND_MAP[tag.toLowerCase()];
if (b) return b; // Retourne dès le premier tag reconnu }
return product.name.split(' ')[0]; // Fallback : premier mot du nom ("Samsung Galaxy"  "Samsung")
}

updateActiveFilters() — Les pastilles de filtres actifs

La zone sous la toolbar affiche une pastille par filtre actif avec un bouton ×. Cliquer le × appelle applyFilters() après avoir retiré le filtre du state. Les pastilles sont régénérées entièrement à chaque applyFilters() — même principe que renderCart() : re-rendu total pour garantir la cohérence.

applyFilters() — la fonction centrale ¶

applyFilters() — pipeline de filtrage dans l'ordre d'exécution
1
matchesSubcategory(p) — OR entre les sous-catégories cochées (iPhone OU iPad OU…)
Échec false immédiat, le produit est rejeté sans vérifier les autres filtres
si passe

Les sliders min/max initialisés sur les valeurs réelles du catalogue
si passe

Ignoré si aucune marque cochée (tableau vide condition ignorée)

Produit inclus dans filteredProducts trié displayCurrentPage()

C'est la fonction appelée à chaque changement d'un filtre. Elle refiltre allProducts entièrement à chaque fois — approche simple et lisible, suffisante pour ~34 produits.

javascript
function applyFilters() {
filteredProducts = allProducts.filter(p => {
if (!matchesSubcategory(p, filters.subcategories))             return false;
if (p.basePrice < filters.priceMin || p.basePrice > filters.priceMax) return false;
if (filters.brands.length   && !filters.brands.includes(getBrand(p))) return false;
if (filters.features.length && !filters.features.every(f => matchesFeature(p, f))) return false;
return true;
});
// Tri selon le critère actif switch (sortBy) {
case 'price-asc':  filteredProducts.sort((a,b) => a.basePrice - b.basePrice);       break;
case 'price-desc': filteredProducts.sort((a,b) => b.basePrice - a.basePrice);       break;
case 'rating':     filteredProducts.sort((a,b) => (b.rating||0) - (a.rating||0)); break;
case 'new':        filteredProducts.sort((a,b) => (b.isNew?1:0) - (a.isNew?1:0)); break;
}
currentPage = 1;       // Toujours revenir page 1 après un changement de filtre displayCurrentPage();  // Découpe et affiche les 12 premiers updatePagination();    // Régénère les boutons de navigation updateActiveFilters(); // Met à jour les tags de filtres actifs
}
La logique AND/OR des filtres combinés
Entre les 4 filtres = AND (tous)
Un produit doit passer les 4 vérifications pour apparaître. Si la sous-catégorie ne matche pas le produit est rejeté sans même vérifier le prix.
Dans features = AND (.every)
Si "5G" et "GPS" sont cochés, le produit doit avoir les deux. .every() retourne faux si une seule feature manque.
Dans sous-catégories = OR (.some)
Si "iPhone" et "iPad" sont cochés, le produit doit appartenir à au moins un des deux.

displayCurrentPage() — la pagination en une ligne

javascript
function displayCurrentPage() {
const start = (currentPage - 1) * PRODUCTS_PER_PAGE; // Ex page 3 : (3-1)*12 = 24 displayedProducts = filteredProducts.slice(start, start + PRODUCTS_PER_PAGE);
// .slice(24, 36) extrait les produits 24 à 35 (12 produits max) renderProducts(displayedProducts);
}

1
Lire l'état courant — currentCategory + currentFilters + currentSort + currentPage
↓
2
Filtrer par catégorie — allProducts.filter(p => !currentCategory || p.category === currentCategory)
↓
3
Filtrer par marque — .filter(p => !filters.brands.length || filters.brands.includes(getBrand(p)))
↓
4
Filtrer par prix — .filter(p => !filters.price || p.basePrice <= filters.price)
↓
5
Filtrer par features — 5G, GPS, etc. via helpers has5G(p), hasGPS(p)
↓
6
Trier — .sort((a,b) => ...) selon currentSort — price-asc/desc, name, default
↓
7
Paginer — .slice((currentPage-1)*PER_PAGE, currentPage*PER_PAGE)
↓
8
Afficher — renderProducts(paginated) + updatePagination(total) + updateActiveFilters()

Pourquoi recalculer depuis zéro à chaque appel

💡
applyFilters() ne maintient pas de résultat intermédiaire. À chaque appel elle repart de allProducts (tous les produits). C'est intentionnel : les filtres sont combinatoires (marque ET prix ET feature). Maintenir des états intermédiaires pour chaque combinaison serait complexe et error-prone. La lisibilité prime ici sur la micro-optimisation.

Pourquoi chaque bloc est placé là où il est ¶

catalogue.js a une architecture forte : état global et utilitaires hors DOMContentLoaded, comportements DOM dedans. La séparation déclaration/initialisation des références DOM résout le conflit portée/timing.

1
État global HORS DOMlet currentCategory, allProducts, filteredProducts, currentPage, sortBy, viewMode, filters
Hors de DOMContentLoaded pour être accessibles dans toutes les fonctions. Ces variables sont partagées entre applyFilters(), renderProducts(), buildFilters(), renderPagination() et les listeners. En JS, une variable let déclarée dans un bloc (comme DOMContentLoaded) est inaccessible à l'extérieur. Les déclarer dans le scope de l'IIFE (niveau supérieur) les rend disponibles partout dans le module.
Si dans DOMContentLoaded : function getBrand(product) définie en dehors de DOMContentLoaded ne pourrait pas accéder à allProducts déclaré à l'intérieur → ReferenceError ou valeur undefined selon l'ordre d'exécution.
↓ DOM déclaré maintenant, initialisé plus tard dans DOMContentLoaded
2
Références DOM HORS DOMlet categoryTitle, productsGrid, sortSelect … (déclarations sans valeur)
Déclarées dans le scope IIFE mais initialisées dans DOMContentLoaded — solution au problème portée/timing. On veut accéder à ces éléments depuis des fonctions définies en dehors de DOMContentLoaded (ex: renderProducts()). Si on les déclarait dans DOMContentLoaded, elles seraient hors scope pour ces fonctions. Si on les initialisait hors DOMContentLoaded, le DOM n'existerait pas encore. La déclaration hors DOM + initialisation dans DOM est la seule solution.
Déclarées ici avec let. Initialisées via getElementById() dans DOMContentLoaded (étape 4). Utilisées dans toutes les fonctions de rendu.
↓ utilitaires purs avant DOMContentLoaded — pas de dépendance au DOM
3
Utilitaires métierBRAND_MAP + getBrand(product) + has5G(product) + hasGPS(product)
Avant DOMContentLoaded — fonctions pures sans accès au DOM. Ces fonctions analysent les données produit (tags, options, specs) sans toucher au DOM. Les placer avant DOMContentLoaded signale clairement qu'elles ne dépendent pas de l'état de la page — elles peuvent s'exécuter à n'importe quel moment. Elles sont utilisées dans buildFilters() et applyFilters(), définies plus loin.
↓ point d'entrée DOM — récupération puis vérification
4
DOMContentLoaded — récupération DOM + vérification PearTechDBInitialisation des références DOM + if (!window.PearTechDB) return
Les éléments DOM sont récupérés en premier dans DOMContentLoaded. Toutes les fonctions qui suivent les utilisent. La vérification de PearTechDB vient immédiatement après car sans données, rien ne peut s'afficher. Pattern fail-fast : si PearTechDB est absent, on affiche un message d'erreur et on stoppe — inutile de continuer.
Sans vérification PearTechDB : PearTechDB.categories.find(…) → TypeError: Cannot read properties of undefined. La page resterait blanche avec une erreur console difficile à diagnostiquer.
↓ données disponibles → construire les filtres
5
Lecture URL + chargement produitsURLSearchParams + PearTechDB.getCategoryBySlug() + getProductsByCategory()
Après la vérification de PearTechDB — on est certain que la DB est disponible. On lit l'URL pour connaître la catégorie demandée (?categorie=apple), puis on charge les produits correspondants dans allProducts. Ces données alimentent buildFilters() et applyFilters(). L'ordre est une contrainte de données : avoir les produits avant de construire les filtres de prix, marque, etc.
↓ construire les éléments DOM avant d'écouter dessus
6
Construction filtres + listenersbuildFilters() → attachFilterEvents()
buildFilters() avant attachFilterEvents() — contrainte DOM absolue. buildFilters() génère les checkboxes, sliders et boutons dans le DOM. attachFilterEvents() attache des listeners sur ces éléments. Si on inversait l'ordre, querySelectorAll('.filter-checkbox') retournerait 0 résultats — les éléments n'existent pas encore.
Si attachFilterEvents() avant buildFilters() : querySelectorAll('.filter-checkbox') → 0 éléments → forEach sur tableau vide → aucun listener → les cases à cocher s'affichent mais ne déclenchent jamais de filtrage.
↓ déclencher l'affichage initial — tout est prêt
7
Affichage initialapplyFilters() — premier rendu
En dernier dans DOMContentLoaded — déclenche le premier affichage. applyFilters() applique l'état initial des filtres (tous à zéro), trie les produits selon sortBy (relevance par défaut), pagine et appelle renderProducts(). Elle doit être appelée après buildFilters() (les filtres existent) et attachFilterEvents() (les listeners sont en place). C'est le 'bouton démarrer' du catalogue.
La séquence buildFilters → attachFilterEvents → applyFilters est obligatoire et non interchangeable. Construire le DOM, le rendre interactif, puis afficher l'état initial.

Concepts et termes techniques utilisés dans ce fichier ¶

Définitions des termes importants rencontrés dans cette documentation.

Document Object Model
Représentation en mémoire de la page HTML sous forme d'arbre. Chaque balise devient un nœud. JavaScript modifie cet arbre via getElementById(), querySelector(), innerHTML. Modifier le DOM = modifier ce que l'utilisateur voit en temps réel.
Immediately Invoked Function Expression
Fonction qui s'exécute immédiatement à sa déclaration : (function() { ... })(). Crée un scope isolé — tout ce qui est déclaré dedans reste privé et ne pollue pas window. Tous les fichiers JS du projet utilisent ce pattern.
DOMContentLoaded
Événement déclenché quand le HTML est parsé et le DOM prêt — avant images et CSS. Moment idéal pour initialiser les modules JS. Distinct de window.load qui attend tous les assets réseau.
localStorage
Stockage clé-valeur persistant dans le navigateur — survit à la fermeture. Capacité ~5 Mo. Clé/valeur uniquement en string : les objets passent par JSON.stringify() avant stockage et JSON.parse() à la lecture.
dataset
Interface qui expose les attributs data-* d'un élément. el.dataset.productId lit data-product-id. Permet de stocker des données métier directement dans le HTML sans variables globales. La conversion camelCase est automatique : data-product-id → dataset.productId.
classList
Interface de manipulation des classes CSS. add(), remove(), toggle(), contains(). Plus fiable que modifier className comme une string. toggle('active', condition) ajoute ou retire selon un booléen.
debounce
Regroupe plusieurs appels rapides en un seul différé. Frappe 5 lettres en 200ms → 1 seule recherche lancée 300ms après la dernière frappe. Implémenté avec clearTimeout(timer) + timer = setTimeout(fn, 300).
URLSearchParams
Lit les paramètres d'une URL : ?id=12&cat=apple. new URLSearchParams(window.location.search).get('id') → '12'. Utilisé sur la page produit pour lire l'ID du produit à afficher depuis l'URL.
Template literal
Chaîne avec backtick `...` qui permet d'insérer des expressions avec ${variable} et des retours à la ligne. Remplace la concaténation de strings. Utilisé massivement pour générer le HTML des cards dynamiquement.

💡

Ajouter les éléments un par un avec appendChild, c'est faire 50 voyages au marché. Une seule injection = un seul reflow.
🚨 PIÈGE — Relire les données depuis le DOM pour filtrer
Ne jamais parser le HTML pour récupérer les données de filtrage (el.textContent, el.dataset.price). Toujours filtrer le tableau de données JS d'origine (PearTechDB.products), puis régénérer le HTML. Le DOM est une représentation, pas la source de vérité.
🚨 PIÈGE — Recréer tous les event listeners après chaque filtrage
Si tu injectes le HTML des produits via innerHTML et réattaches ensuite les listeners "Ajouter au panier" à chaque filtrage, tu cumules des listeners sur les anciens éléments détruits. Utilise la délégation d'événements sur le container parent : container.addEventListener('click', e => { if(e.target.matches('.btn-add')) … }).

Mémoïse le résultat du filtrage courant dans une variable currentProducts. Le tri peut ensuite s'appliquer sur ce résultat filtré sans refaire le filtre. Deux états séparés : activeFilters et sortOrder.
PRO TIP
Pour de la pagination ou un scroll infini, slice le tableau filtré : currentProducts.slice(0, page * perPage). Pas besoin de librairie externe.

    // Table de correspondance : tag produit → nom de marque lisible
    // Nécessaire car les tags sont en minuscules (ex: 'iphone') mais on affiche 'Apple'
    const BRAND_MAP = {
        'apple':    'Apple',   'iphone':   'Apple',  'ipad': 'Apple', // Produits Apple
        'samsung':  'Samsung',                                          // Produits Samsung
        'google':   'Google',  'pixel':    'Google',                   // Produits Google
        'xiaomi':   'Xiaomi',                                           // Produits Xiaomi
        'oneplus':  'OnePlus',                                          // Produits OnePlus
        'amazon':   'Amazon',  'fire':     'Amazon',                   // Produits Amazon
        'fitbit':   'Fitbit',                                           // Montres Fitbit
        'garmin':   'Garmin',                                           // Montres/GPS Garmin
        'motorola': 'Motorola',                                         // Téléphones Motorola
        'oppo':     'Oppo',                                             // Téléphones Oppo
        'sony':     'Sony',                                             // Produits Sony
        'huawei':   'Huawei'                                            // Produits Huawei
    };

    // Détermine la marque d'un produit à partir de ses tags
    function getBrand(product) {
        for (const tag of product.tags) { // Parcourt chaque tag du produit
            const b = BRAND_MAP[tag.toLowerCase()]; // Cherche le tag (en minuscules) dans la table
            if (b) return b; // Retourne la marque dès qu'on la trouve
        }
        return product.name.split(' ')[0]; // Fallback : prend le premier mot du nom (ex: "Samsung Galaxy" → "Samsung")
    }

    // Vérifie si un produit est compatible 5G en inspectant ses options
    function has5G(product) {
        if (!product.options) return false; // Pas d'options définies : pas de 5G
        // Object.values donne un tableau de tableaux → flat() l'aplatit en un seul tableau
        // some() retourne true dès qu'une option contient "5G" dans son label
        return Object.values(product.options).flat()
            .some(o => o.label && o.label.includes('5G'));
    }

    // Vérifie si un produit intègre le GPS en cherchant dans toutes ses specs et options
    function hasGPS(product) {
        // JSON.stringify convertit l'objet en chaîne pour faciliter la recherche de texte
        // On cherche "gps" (minuscules) dans les specs ET les options combinées
        const haystack = JSON.stringify(product.specs || {}).toLowerCase()
                       + JSON.stringify(product.options || {}).toLowerCase();
        return haystack.includes('gps'); // True si "gps" est trouvé quelque part
    }

        // On fait ça en une seule fois pour éviter des querySelector répétés

        categoryTitle       = document.getElementById('category-title');
        categoryDescription = document.getElementById('category-description');
        resultsCount        = document.getElementById('results-count');
        productsGrid        = document.getElementById('products-grid');
        paginationDiv       = document.getElementById('pagination');
        resetFiltersBtn     = document.getElementById('reset-filters');

        // ── Vérification que PearTechDB est chargé ────────────────
        // data.js doit être chargé avant catalogue.js dans le HTML

        if (!window.PearTechDB) {

            if (productsGrid) productsGrid.innerHTML = '<p class="no-results">Impossible de charger les produits.</p>';

        // Ex: page_catalogue.html?categorie=apple → slug = "apple"

        const categorySlug = urlParams.get('categorie'); // Récupère la valeur du paramètre "categorie"

                subcategories: [] // Pas de sous-catégories prédéfinies pour "tous"

            allProducts = PearTechDB.products; // Tous les produits de la base

        if (sortSelect) sortSelect.addEventListener('change', function () {

        if (priceMaxSlider) priceMaxSlider.addEventListener('input', function () {

    // CONSTRUCTION DES FILTRES SIDEBAR
    // Génère dynamiquement les checkboxes à partir des produits réels

    // ════════════════════════════════════════════════════════════════

    function initFilters() {

        // ── Sous-catégories ───────────────────────────────────────
        // Map : sous-catégorie → nombre de produits correspondants
        // Utilisée pour afficher le compteur "(X)" à côté de chaque checkbox

        const subcatMap = new Map(); // Map clé = nom, valeur = nombre de produits

        if (currentCategory.subcategories && currentCategory.subcategories.length) {
            // Catégorie spécifique : utilise les sous-catégories définies dans la DB
            currentCategory.subcategories.forEach(sub => {
                const subLower = sub.toLowerCase(); // Normalise en minuscules pour la comparaison
                const count = allProducts.filter(p =>
                    p.name.toLowerCase().includes(subLower) || // Le nom du produit contient la sous-cat
                    p.tags.some(t => subLower.includes(t) || t.includes(subLower)) // Ou un tag correspond
                ).length;
                if (count > 0) subcatMap.set(sub, count); // N'ajoute que si au moins 1 produit correspond
            });
        } else {
            // Page "tous" : regroupe par nom de catégorie principale (Apple, Android, Montres...)
            PearTechDB.categories.forEach(cat => {
                const count = allProducts.filter(p => p.categoryId === cat.id).length; // Produits de cette catégorie
                if (count > 0) subcatMap.set(cat.name, count); // Ajoute avec le compteur
            });
        }

        // Injection des checkboxes de sous-catégories dans le DOM
        const subcatDiv = document.getElementById('subcategories-filter');
        if (subcatDiv) {
            subcatDiv.innerHTML = ''; // Vide le conteneur avant de le remplir
            subcatMap.forEach((count, label) => { // Pour chaque sous-catégorie
                const el = document.createElement('label');
                el.className = 'checkbox-item'; // Classe CSS pour le style aligné
                el.innerHTML = `<input type="checkbox" class="subcategory-checkbox" value="${label}">
                    <span>${label}</span><span class="checkbox-count">${count}</span>`;
                subcatDiv.appendChild(el); // Ajoute la checkbox dans le DOM
            });

            // Écoute les changements sur les checkboxes de sous-catégories
            subcatDiv.querySelectorAll('.subcategory-checkbox').forEach(cb =>
                cb.addEventListener('change', () => {
                    // Collecte toutes les checkboxes sous-catégories cochées → tableau de valeurs
                    filters.subcategories = [...document.querySelectorAll('.subcategory-checkbox:checked')].map(c => c.value);
                    applyFilters(); // Relance le filtrage
                })
            );
        }

        // ── Marques ────────────────────────────────────────────────
        // Construit la liste des marques présentes dans les produits actuels

        const brandsMap = new Map(); // Map : marque → nombre de produits
        allProducts.forEach(p => {
            const b = getBrand(p); // Détermine la marque du produit
            brandsMap.set(b, (brandsMap.get(b) || 0) + 1); // Incrémente le compteur (0 si première fois)
        });

        // Injection des checkboxes de marques, triées par nombre de produits décroissant
        const brandsDiv = document.getElementById('brands-filter');
        if (brandsDiv) {
            brandsDiv.innerHTML = ''; // Vide avant de remplir
            // [...brandsMap.entries()] convertit la Map en tableau de [clé, valeur]
            // .sort((a,b) => b[1]-a[1]) trie par valeur (compte) décroissant (plus de produits en premier)
            [...brandsMap.entries()].sort((a,b) => b[1]-a[1]).forEach(([label, count]) => {
                const el = document.createElement('label');
                el.className = 'checkbox-item';
                el.innerHTML = `<input type="checkbox" class="brand-checkbox" value="${label}">
                    <span>${label}</span><span class="checkbox-count">${count}</span>`;
                brandsDiv.appendChild(el);
            });
            // Écoute les changements sur les checkboxes de marques
            brandsDiv.querySelectorAll('.brand-checkbox').forEach(cb =>
                cb.addEventListener('change', () => {
                    filters.brands = [...document.querySelectorAll('.brand-checkbox:checked')].map(c => c.value);
                    applyFilters();
                })
            );
        }

        // ── Caractéristiques ────────────────────────────────────────
        // Génère dynamiquement les filtres selon les caractéristiques réelles des produits
        // (pas de liste en dur : si aucun produit 5G, le filtre 5G n'apparaît pas)

        const featSet = new Set(); // Set = ensemble sans doublons (chaque feature une seule fois)
        allProducts.forEach(p => {
            if (p.specs.os) { // Si le produit a un OS défini
                const os = p.specs.os.toLowerCase(); // Normalise en minuscules
                if (os.includes('ios') || os.includes('ipados')) featSet.add('iOS / iPadOS'); // Produit Apple
                else if (os.includes('android'))                  featSet.add('Android');      // Produit Android
                else if (os.includes('fire'))                     featSet.add('Fire OS');       // Tablette Amazon Fire
            }
            if (has5G(p))  featSet.add('Compatible 5G');   // Ajoute "5G" si le produit le supporte
            if (hasGPS(p)) featSet.add('GPS intégré');     // Ajoute "GPS" si présent dans les specs/options
            if (p.specs.ram) { // Si la RAM est renseignée
                const r = parseInt(p.specs.ram); // Convertit en entier (ex: "8 Go" → 8)
                if (r <= 4)       featSet.add('RAM ≤ 4 Go');    // Entrée de gamme
                else if (r <= 8)  featSet.add('RAM 6–8 Go');    // Milieu de gamme
                else if (r <= 12) featSet.add('RAM 12 Go');     // Haut de gamme
                else              featSet.add('RAM 16 Go et +'); // Ultra haut de gamme
            }
            if (p.oldPrice && p.oldPrice > p.basePrice) featSet.add('En promotion');  // En promo si ancien prix > prix actuel
            if (p.isNew)        featSet.add('Nouveautés');        // Marqué comme nouveau
            if (p.isBestSeller) featSet.add('Meilleures ventes'); // Marqué comme bestseller
        });

        // Injection des checkboxes de caractéristiques
        const featDiv = document.getElementById('features-filter');
        if (featDiv) {
            featDiv.innerHTML = ''; // Vide avant de remplir
            featSet.forEach(feat => { // Pour chaque caractéristique trouvée
                const el = document.createElement('label');
                el.className = 'checkbox-item';
                // Pas de compteur ici : plus difficile à calculer, pas indispensable UX
                el.innerHTML = `<input type="checkbox" class="feature-checkbox" value="${feat}">
                    <span>${feat}</span>`;
                featDiv.appendChild(el);
            });
            featDiv.querySelectorAll('.feature-checkbox').forEach(cb =>
                cb.addEventListener('change', () => {
                    filters.features = [...document.querySelectorAll('.feature-checkbox:checked')].map(c => c.value);
                    applyFilters();
                })
            );
        }

        const minPrice = Math.floor(Math.min(...prices));    // Prix le plus bas (arrondi à l'inférieur)
        const maxPrice = Math.ceil(Math.max(...prices));     // Prix le plus haut (arrondi au supérieur)

        if (priceMinSlider) { priceMinSlider.min = minPrice; priceMinSlider.max = maxPrice; priceMinSlider.value = minPrice; priceMinSlider.step = 10; }
        if (priceMaxSlider) { priceMaxSlider.min = minPrice; priceMaxSlider.max = maxPrice; priceMaxSlider.value = maxPrice; priceMaxSlider.step = 10; }

        // Affiche les valeurs textuelles initiales
        if (priceMinDisplay) priceMinDisplay.textContent = minPrice.toLocaleString('fr-FR') + ' €';
        if (priceMaxDisplay) priceMaxDisplay.textContent = maxPrice.toLocaleString('fr-FR') + ' €';

        // Initialise les filtres de prix avec les vraies valeurs min/max
        filters.priceMin = minPrice;
        filters.priceMax = maxPrice;
    }

    // ════════════════════════════════════════════════════════════════
    // MATCHING DES SOUS-CATÉGORIES
    // Prédicats explicites pour éviter les faux positifs
    // (ex: "apple watch" contient "watch" ce qui matchait Samsung Galaxy Watch)
    // ════════════════════════════════════════════════════════════════

    const SUBCAT_MATCHERS = {
        // ── Apple ──
        'iPhone':              p => p.tags.includes('iphone'),                                // Produits avec le tag "iphone"
        'iPad':                p => p.tags.includes('ipad'),                                  // Produits avec le tag "ipad"
        'Apple Watch':         p => p.categoryId === 'cat_wearables' && p.tags.includes('apple'), // Montres Apple uniquement
        'Accessoires':         p => p.tags.some(t => ['accessoire','accessoires'].includes(t)), // Tag accessoire
        // ── Android ──
        'Samsung':             p => p.tags.includes('samsung') && !p.tags.includes('tablette') && !p.tags.some(t => ['watch','montre'].includes(t)), // Samsung mais pas tablette ni montre
        'Google Pixel':        p => p.tags.includes('google') || p.tags.includes('pixel'),  // Google ou Pixel
        'Xiaomi':              p => p.tags.includes('xiaomi'),                               // Marque Xiaomi
        'OnePlus':             p => p.tags.includes('oneplus'),                              // Marque OnePlus
        // ── Wearables ──
        'Samsung Galaxy Watch':p => p.tags.includes('samsung') && p.tags.some(t => ['watch','montre'].includes(t)), // Samsung + watch
        'Fitbit':              p => p.tags.includes('fitbit'),                               // Marque Fitbit
        'Garmin':              p => p.tags.includes('garmin'),                               // Marque Garmin
        // ── Tablettes ──
        'Samsung Galaxy Tab':  p => p.tags.includes('samsung') && p.tags.includes('tablette'), // Samsung + tablette
        'Amazon Fire':         p => p.tags.includes('amazon') || p.tags.includes('fire'),   // Amazon ou Fire OS
        // ── Fallback universel ──
        'Autres':              p => true, // Accepte tout : utilisé si aucun autre matcher ne correspond
    };

    // Vérifie si un produit correspond à la liste de sous-catégories filtrées
    function matchesSubcategory(product, subs) {
        if (!subs.length) return true; // Aucune sous-catégorie cochée : tous les produits passent

        return subs.some(sub => { // Au moins une sous-catégorie cochée doit correspondre (OR logique)
            // 1. Utilise le matcher explicite si disponible (plus précis)
            const matcher = SUBCAT_MATCHERS[sub];
            if (matcher) return matcher(product); // Retourne le résultat du prédicat
            // 2. Fallback : cherche la catégorie par son nom (pour la page "tous")
            const cat = PearTechDB.categories.find(c => c.name === sub);
            if (cat) return product.categoryId === cat.id; // Vérifie l'ID de catégorie
            // 3. Fallback final : cherche le nom dans le nom du produit
            return product.name.toLowerCase().includes(sub.toLowerCase());
        });
    }

    // Vérifie si un produit possède une caractéristique spécifique
    function matchesFeature(product, feat) {
        switch (feat) {
            case 'iOS / iPadOS':    return product.specs.os && /ios|ipados/i.test(product.specs.os); // Regex insensible à la casse
            case 'Android':         return product.specs.os && /android/i.test(product.specs.os);
            case 'Fire OS':         return product.specs.os && /fire/i.test(product.specs.os);
            case 'Compatible 5G':   return has5G(product);  // Délègue à la fonction utilitaire
            case 'GPS intégré':     return hasGPS(product); // Délègue à la fonction utilitaire
            case 'RAM ≤ 4 Go':      return product.specs.ram && parseInt(product.specs.ram) <= 4;  // RAM faible
            case 'RAM 6–8 Go':      return product.specs.ram && parseInt(product.specs.ram) >= 6 && parseInt(product.specs.ram) <= 8; // RAM moyenne
            case 'RAM 12 Go':       return product.specs.ram && parseInt(product.specs.ram) === 12; // RAM haute
            case 'RAM 16 Go et +':  return product.specs.ram && parseInt(product.specs.ram) >= 16; // RAM très haute
            case 'En promotion':    return product.oldPrice && product.oldPrice > product.basePrice; // Prix barré
            case 'Nouveautés':      return product.isNew === true;       // Marqué isNew dans data.js
            case 'Meilleures ventes': return product.isBestSeller === true; // Marqué isBestSeller dans data.js
            default: return true; // Feature inconnue : le produit passe par défaut (ne bloque pas)
        }
    }

    // ════════════════════════════════════════════════════════════════
    // APPLICATION DES FILTRES ET DU TRI
    // Fonction centrale appelée après chaque modification d'un filtre
    // ════════════════════════════════════════════════════════════════

    function applyFilters() {
        // Filtre allProducts selon tous les critères actifs
        filteredProducts = allProducts.filter(p => {
            if (!matchesSubcategory(p, filters.subcategories))            return false; // Sous-cat ne correspond pas
            if (p.basePrice < filters.priceMin || p.basePrice > filters.priceMax) return false; // Hors fourchette de prix
            if (filters.brands.length   && !filters.brands.includes(getBrand(p)))  return false; // Marque non sélectionnée
            // every() : toutes les features sélectionnées doivent correspondre (AND logique)
            if (filters.features.length && !filters.features.every(f => matchesFeature(p, f))) return false;
            return true; // Le produit passe tous les filtres
        });

        // Tri des produits filtrés selon le critère choisi
        switch (sortBy) {
            case 'price-asc':  filteredProducts.sort((a,b) => a.basePrice - b.basePrice);        break; // Prix croissant
            case 'price-desc': filteredProducts.sort((a,b) => b.basePrice - a.basePrice);        break; // Prix décroissant
            case 'rating':     filteredProducts.sort((a,b) => (b.rating||0) - (a.rating||0));   break; // Meilleures notes
            case 'new':        filteredProducts.sort((a,b) => (b.isNew?1:0) - (a.isNew?1:0));   break; // Nouveautés d'abord
            // 'relevance' (défaut) : pas de tri, garde l'ordre de la DB
        }

        // Met à jour le compteur "X résultat(s)"
        if (resultsCount) resultsCount.textContent = filteredProducts.length + ' résultat'
            + (filteredProducts.length !== 1 ? 's' : ''); // Accorde "résultat" avec le nombre

        currentPage = 1;        // Repart toujours à la page 1 après un changement de filtre

    function addToCart(productId, btn) {
        const product = allProducts.find(p => p.id === productId); // Cherche le produit dans les données
        if (!product) return; // Produit introuvable : on arrête

        const CART_KEY = 'peartech-cart'; // Clé localStorage du panier
        const cart     = JSON.parse(localStorage.getItem(CART_KEY)) || []; // Charge le panier ou tableau vide
        const existing = cart.find(i => i.id === productId); // Vérifie si déjà dans le panier

        if (existing) {
            existing.quantity++; // Déjà présent : incrémente la quantité
        } else {
            cart.push({ // Nouveau produit : ajoute avec toutes les infos nécessaires
                id:       product.id,
                name:     product.name,
                image:    product.images[0] || '', // Première image ou chaîne vide
                price:    product.basePrice,        // Prix actuel
                // Spécifications en une ligne séparée par " - " (filtre les nulls avec filter(Boolean))
                specs:    [product.specs.processor, product.specs.ram, product.specs.storage].filter(Boolean).join(' - '),
                quantity: 1 // Toujours 1 au premier ajout
            });
        }

        localStorage.setItem(CART_KEY, JSON.stringify(cart)); // Sauvegarde le panier mis à jour
        const total = cart.reduce((acc, i) => acc + i.quantity, 0); // Total d'articles toutes lignes
        localStorage.setItem('peartech-cart-count', total); // Sauvegarde le compteur pour le badge

        // Met à jour le badge dans le header
        const badge = document.getElementById('cart-count');
        if (badge) {
            badge.textContent = total;
            badge.style.display = 'flex'; // S'assure que le badge est visible
            badge.setAttribute('aria-label', total + ' article' + (total > 1 ? 's' : '') + ' dans le panier');

        // ── Feedback visuel : remplace l'icône panier par une coche ──

        // SVG coche verte
        btn.innerHTML = '<svg viewBox="0 0 24 24" width="18" height="18" fill="none" stroke="currentColor" stroke-width="2.5" stroke-linecap="round" stroke-linejoin="round" aria-hidden="true"><polyline points="20 6 9 17 4 12"/></svg>';
        btn.disabled  = true; // Désactive pour éviter les double-clics

  // ── Étape 1 : FILTRER ─────────────────────────────────────────────
  let products = PearTechDB.products.filter(p => {
    const matchCat  = !activeFilters.category
                    || p.category === activeFilters.category;
    // !activeFilters.category = aucun filtre catégorie → tous passent.
    // Sinon : uniquement les produits de la catégorie sélectionnée.

    const matchPrice = p.price <= activeFilters.maxPrice;
    // Filtre prix : toujours actif (maxPrice a une valeur par défaut élevée).

    const matchSearch = !activeFilters.search
                     || p.name.toLowerCase().includes(activeFilters.search);

    return matchCat && matchPrice && matchSearch;
    // Un produit passe SI il satisfait TOUS les filtres actifs (AND logique).
  });

  // ── Étape 2 : TRIER ───────────────────────────────────────────────
  products = [...products].sort((a, b) => {
    switch (activeSort) {
      case 'price-asc':  return a.price - b.price;
      // a - b : si négatif → a avant b (croissant)
      case 'price-desc': return b.price - a.price;
      // b - a : si négatif → b avant a (décroissant)
      case 'name':       return a.name.localeCompare(b.name);
      // localeCompare : tri alphabétique respectant les accents et la locale.
      default:           return 0;
      // 0 = ordre stable (pas de changement de position)
    }
  });
  // [...products] : spread pour copier le tableau avant sort.
  // sort() modifie le tableau en place → si on triait products directement,
  // le prochain filtre partirait du tableau trié, pas de l'original.

  // ── Étape 3 : RENDRE ──────────────────────────────────────────────
  const html = products.map(p => productCardHTML(p)).join('');
  // map : transforme chaque produit en string HTML.
  // join('') : concatène tout en une seule string.
  // Une seule assignation innerHTML = un seul reflow du navigateur.

  catalogue.innerHTML = html
    || `<p class="no-results">Aucun produit trouvé.</p>`;
  // Si html = '' (0 résultats), || affiche le message vide.

  updateResultsCount(products.length);
}

  const btn = e.target.closest('.btn-add');
  // closest remonte l'arbre DOM jusqu'à trouver .btn-add
  // ou null si le clic n'était pas sur/dans un bouton add.
  if (!btn) return;

  const id = +btn.dataset.productId;
  // dataset.productId lit l'attribut data-product-id.
  // Le + devant convertit la string en number (raccourci de parseInt).

  const product = PearTechDB.getById(id);
  if (product) PearTechCart.addToCart(product.id, product.name, product.price);
});
// Un seul listener sur catalogue, valable même après re-render. ✓
