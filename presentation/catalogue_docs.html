catalogue.js

Ce fichier gère entièrement la page catalogue (page_catalogue.html). Il lit la catégorie depuis l'URL, charge les produits correspondants depuis PearTechDB, construit les filtres dynamiques, applique le filtrage multi-critères, trie, pagine, et gère l'ajout au panier depuis la grille.

📖 Termes nouveaux dans ce fichier
URLSearchParams
API navigateur pour lire les paramètres d'une URL. new URLSearchParams(window.location.search) sur ?categorie=apple permet d'écrire params.get('categorie') → 'apple'.
Map
Structure de données clé→valeur. Contrairement à un objet, les clés peuvent être de n'importe quel type et l'ordre d'insertion est garanti. Utilisée ici pour compter les produits par sous-catégorie ou marque.
Set
Collection sans doublons. featSet.add('5G') plusieurs fois ne l'insère qu'une seule fois. Utilisé pour lister les caractéristiques disponibles.
Object.values().flat()
Object.values(obj) retourne les valeurs d'un objet en tableau. .flat() aplatit un tableau de tableaux en un seul. Ex : {a:[1,2], b:[3]} → [1,2,3].
prédicat
Fonction qui retourne true ou false. Dans SUBCAT_MATCHERS, chaque entrée est un prédicat : 'iPhone': p => p.tags.includes('iphone') retourne vrai si le produit a le tag "iphone".
.every()
Retourne true si tous les éléments du tableau satisfont la condition (AND logique). Utilisé pour les features : le produit doit posséder toutes les caractéristiques cochées.
.some()
Retourne true si au moins un élément satisfait la condition (OR logique). Utilisé pour les sous-catégories : le produit doit appartenir à au moins une des sous-catégories cochées.
aria-*
Attributs HTML d'accessibilité pour les lecteurs d'écran. aria-label donne un nom vocal à un élément, aria-current="page" indique la page active, aria-pressed indique l'état d'un bouton toggle.

const urlParams    = new URLSearchParams(window.location.search);
const categorySlug = urlParams.get('categorie');

    const cat = PearTechDB.getCategoryBySlug(categorySlug);
    if (cat) {

03 — Construction des filtres

initFilters() — générer la sidebar dynamiquement

Les checkboxes de la sidebar ne sont pas codées en dur dans le HTML — elles sont générées à partir des produits réels. Si aucun produit du catalogue ne supporte le 5G, le filtre "Compatible 5G" n'apparaît pas.

Les 4 blocs générés dynamiquement depuis les produits réels
Sous-catégories
Map slug→count depuis subcategories[] de la catégorie courante.
Ex : iPhone (14) · Samsung (8)
Marques
getBrand(p) → Map marque→count triée par count décroissant.
Ex : Apple (14) · Samsung (8)
Caractéristiques
Set auto-rempli — OS, 5G, GPS, RAM… N'apparaît que si au moins un produit correspond.
Pas de filtre fantôme.
Sliders prix
min/max calculés depuis les basePrice réels de la catégorie.
Pas de valeurs codées en dur.
Si tous les produits sont d'une seule marque, le bloc marques affiche une seule entrée. Si aucun produit n'a le GPS, le filtre GPS n'est pas créé. La sidebar s'adapte aux données.
Les 4 blocs générés par initFilters()
Sous-catégories
Map slug→count depuis subcategories[] de la catégorie
Marques
Map marque→count via getBrand(), triée par count décroissant
Caractéristiques
Set auto-rempli : OS, 5G, GPS, RAM, promos, isNew…
Sliders prix
min/max calculés depuis les basePrice réels

const featSet = new Set();
allProducts.forEach(p => {
    if (p.specs.os?.toLowerCase().includes('ios')) featSet.add('iOS / iPadOS');
    if (has5G(p))  featSet.add('Compatible 5G');   // Add idempotent : 2e add ignoré
    if (hasGPS(p)) featSet.add('GPS intégré');

        if (r <= 4)       featSet.add('RAM ≤ 4 Go');

        else if (r <= 12) featSet.add('RAM 12 Go');
        else               featSet.add('RAM 16 Go et +');

    if (p.isBestSeller) featSet.add('Meilleures ventes');
});
// featSet contient chaque feature au plus une fois, peu importe combien de produits la partagent

getBrand() — détecter la marque via les tags

javascript
const BRAND_MAP = {
    'apple': 'Apple', 'iphone': 'Apple', 'ipad': 'Apple',
    'samsung': 'Samsung', 'google': 'Google', 'pixel': 'Google',
    // … 12 marques au total
};

function getBrand(product) {
    for (const tag of product.tags) {
        const b = BRAND_MAP[tag.toLowerCase()];
        if (b) return b; // Retourne dès le premier tag reconnu
    }
    return product.name.split(' ')[0]; // Fallback : premier mot du nom ("Samsung Galaxy" → "Samsung")
}
04 — Matching sous-catégories

SUBCAT_MATCHERS — prédicats précis contre les faux positifs

Une approche naïve (name.includes('Apple Watch')) créerait des faux positifs : "Samsung Galaxy Watch" contient "Watch". La solution : un objet dictionnaire avec un prédicat (test) précis par sous-catégorie.

Naïf vs prédicats précis — les faux positifs du includes()
Approche naïve — faux positifs
p.name.includes("Watch")
→ "Apple Watch" ✓ mais aussi
→ "Samsung Galaxy Watch" ✗
p.name.includes("Apple")
→ "Apple Watch" ✓ mais aussi
→ "Apple AirPods" ✗ (mauvaise catég.)
SUBCAT_MATCHERS — prédicats exacts
'iPhone':
p => p.tags.includes('iphone')
'Apple Watch':
p => p.categoryId === 'cat_wearables'
&& p.tags.includes('apple')
→ 2 conditions AND → 0 faux positif
Usage : if (SUBCAT_MATCHERS[subcat]) return SUBCAT_MATCHERS[subcat](product)
javascript — extraits de SUBCAT_MATCHERS
const SUBCAT_MATCHERS = {
    'iPhone':          p => p.tags.includes('iphone'),           // Tag exact
    'Apple Watch':     p => p.categoryId === 'cat_wearables' && p.tags.includes('apple'),
    // ↑ Double condition pour ne pas inclure l'iPhone qui a aussi le tag 'apple'
    'Samsung':         p => p.tags.includes('samsung')
                            && !p.tags.includes('tablette')
                            && !p.tags.some(t => ['watch', 'montre'].includes(t)),
    // ↑ Samsung téléphones uniquement : exclut tablettes ET montres
    'Samsung Galaxy Watch': p => p.tags.includes('samsung') && p.tags.some(t => ['watch','montre'].includes(t)),
    'Samsung Galaxy Tab':   p => p.tags.includes('samsung') && p.tags.includes('tablette'),
    'Autres':          p => true // Accepte tout — filet de sécurité
};
Problème du faux positif Samsung — résolu par les matchers
❌ Approche naïve
p.name.includes('Samsung')
→ "Samsung Galaxy S24" ✓
→ "Samsung Galaxy Tab" ✓ ← faux positif
→ "Samsung Galaxy Watch"✓ ← faux positif
✅ Matcher précis
samsung && !tablette && !watch
→ "Samsung Galaxy S24" ✓
→ "Samsung Galaxy Tab" ✗
→ "Samsung Galaxy Watch"✗
05 — Application des filtres

applyFilters() — la fonction centrale

️ applyFilters() — pipeline de filtrage dans l'ordre d'exécution
1
matchesSubcategory(p) — OR entre les sous-catégories cochées (iPhone OU iPad OU…)
Échec → false immédiat, le produit est rejeté sans vérifier les autres filtres
↓ si passe
2
p.basePrice ≥ priceMin && ≤ priceMax — fourchette de prix
Les sliders min/max initialisés sur les valeurs réelles du catalogue
↓ si passe
3
filters.brands.includes(getBrand(p)) — marque cochée
Ignoré si aucune marque cochée (tableau vide → condition ignorée)
↓ si passe
4
filters.features.every(f => matchesFeature(p, f)) — AND entre features
Si "5G" ET "GPS" cochés → le produit doit avoir les DEUX
↓ tous les filtres passés
✓
Produit inclus dans filteredProducts → trié → displayCurrentPage()

C'est la fonction appelée à chaque changement d'un filtre. Elle refiltre allProducts entièrement à chaque fois — approche simple et lisible, suffisante pour ~34 produits.

javascript
function applyFilters() {
    filteredProducts = allProducts.filter(p => {
        if (!matchesSubcategory(p, filters.subcategories))             return false;
        if (p.basePrice < filters.priceMin || p.basePrice > filters.priceMax) return false;
        if (filters.brands.length   && !filters.brands.includes(getBrand(p))) return false;
        if (filters.features.length && !filters.features.every(f => matchesFeature(p, f))) return false;
        return true;
    });

    // Tri selon le critère actif
    switch (sortBy) {
        case 'price-asc':  filteredProducts.sort((a,b) => a.basePrice - b.basePrice);       break;
        case 'price-desc': filteredProducts.sort((a,b) => b.basePrice - a.basePrice);       break;
        case 'rating':     filteredProducts.sort((a,b) => (b.rating||0) - (a.rating||0)); break;
        case 'new':        filteredProducts.sort((a,b) => (b.isNew?1:0) - (a.isNew?1:0)); break;
    }

    currentPage = 1;       // Toujours revenir page 1 après un changement de filtre
    displayCurrentPage();  // Découpe et affiche les 12 premiers
    updatePagination();    // Régénère les boutons de navigation
    updateActiveFilters(); // Met à jour les tags de filtres actifs
}
La logique AND/OR des filtres combinés
Entre les 4 filtres = AND (tous)
Un produit doit passer les 4 vérifications pour apparaître. Si la sous-catégorie ne matche pas → le produit est rejeté sans même vérifier le prix.
Dans features = AND (.every)
Si "5G" et "GPS" sont cochés, le produit doit avoir les deux. .every() retourne faux si une seule feature manque.
Dans sous-catégories = OR (.some)
Si "iPhone" et "iPad" sont cochés, le produit doit appartenir à au moins un des deux.

displayCurrentPage() — la pagination en une ligne

javascript
function displayCurrentPage() {
    const start     = (currentPage - 1) * PRODUCTS_PER_PAGE; // Ex page 3 : (3-1)*12 = 24
    displayedProducts = filteredProducts.slice(start, start + PRODUCTS_PER_PAGE);
    // .slice(24, 36) extrait les produits 24 à 35 (12 produits max)
    renderProducts(displayedProducts);
}

La pagination n'affiche pas tous les numéros si il y en a beaucoup : elle montre toujours la page 1, la dernière, et les 2 pages autour de la page courante, avec des … pour les sauts.

javascript
for (let i = 1; i <= totalPages; i++) {
    if (i === 1 || i === totalPages || (i >= currentPage - 2 && i <= currentPage + 2)) {
        // Toujours afficher : 1re page, dernière page, et fenêtre ±2 autour de la page courante
        html += bouton(i);
    } else if (i === currentPage - 3 || i === currentPage + 3) {
        // Juste en dehors de la fenêtre → afficher "…"
        html += '<span class="page-dots">…</span>';
    }
    // Les autres numéros sont simplement ignorés
}

javascript — suppression d'un tag "brand"
btn.addEventListener('click', function() {
    const type  = this.dataset.filter; // 'sub', 'brand', 'feature', ou 'price'
    const value = this.dataset.value;  // ex: 'Apple'

    if (type === 'brand') {
        // Décoche la checkbox dans la sidebar
        document.querySelectorAll('.brand-checkbox').forEach(cb => {
            if (cb.value === value) cb.checked = false;
        });
        // Retire la valeur du tableau filters.brands
        filters.brands = filters.brands.filter(v => v !== value);
    }
    // … cas 'sub', 'feature', 'price' similaires
    applyFilters(); // Relance tout
});
