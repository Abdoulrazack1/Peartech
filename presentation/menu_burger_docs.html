Contrairement à la plupart des éléments de navigation, le bouton burger et le panneau mobile n'existent pas dans le HTML — ils sont créés et injectés par JavaScript au chargement.

javascript — createBurgerButton()
function createBurgerButton() {
if (document.getElementById('burger-menu')) return; // Guard idempotent : ne crée qu'une fois const burger = document.createElement('button');
burger.id        = 'burger-menu';
burger.className = 'burger-btn';
burger.setAttribute('aria-label', 'Ouvrir le menu');
burger.setAttribute('aria-expanded', 'false');     // État initial : fermé burger.setAttribute('aria-controls', 'mobile-menu'); // Lie le bouton à ce qu'il contrôle burger.innerHTML = `<span class="material-symbols-outlined burger-icon">menu</span>`;
const nav = document.querySelector('nav');
if (nav) nav.insertBefore(burger, nav.firstChild); // Premier enfant de la nav
}
javascript — createMobileMenu() — structure simplifiée
function createMobileMenu() {
if (document.getElementById('mobile-menu')) return;
// Overlay sombre derrière le panneau const overlay = document.createElement('div');
overlay.className = 'mobile-overlay';
overlay.setAttribute('aria-hidden', 'true'); // Invisible aux lecteurs d'écran document.body.appendChild(overlay);
// Panneau latéral (rôle ARIA dialog) const menu = document.createElement('div');
menu.id = 'mobile-menu';
menu.setAttribute('role', 'dialog');
menu.setAttribute('aria-modal', 'true'); // Signale une boîte de dialogue modale menu.innerHTML = `... header + nav + actions ...`;
document.body.appendChild(menu);
}
Attributs ARIA utilisés
aria-label
Texte lu par les lecteurs d'écran à la place du contenu visuel. Indispensable pour les boutons icône (sans texte).
aria-expanded
"false" = le bouton contrôle un élément actuellement fermé. "true" = actuellement ouvert. Mis à jour dynamiquement à chaque ouverture/fermeture.
aria-controls
ID de l'élément que ce bouton contrôle. Permet aux lecteurs d'écran de savoir que le bouton ouvre #mobile-menu.
role="dialog"
Signale que cet élément est une boîte de dialogue. Les lecteurs d'écran annoncent "Boîte de dialogue : Menu de navigation" à l'ouverture.
aria-modal="true"
Indique que le contenu derrière la modale est temporairement inaccessible. Combiné avec le focus trap, ça garantit une navigation clavier cohérente.
aria-current="page"
Ajouté au lien de la page courante par highlightActivePage(). Indique aux lecteurs d'écran quelle page est active.
aria-hidden="true"
Sur l'overlay : l'exclut complètement de l'arbre d'accessibilité. L'overlay est purement décoratif, les lecteurs d'écran l'ignorent.

La séquence de création — 4 étapes ordonnées

1createBurgerButton()

2createMobileMenu()

javascript
function openMobileMenu(burgerBtn, mobileMenu, mobileOverlay) {
mobileMenu.classList.add('active');        // CSS translate ou visibility mobileOverlay.classList.add('active');     // Fond sombre visible burgerBtn.classList.add('active');         // Style actif du bouton (optionnel) burgerBtn.setAttribute('aria-expanded', 'true');
burgerBtn.setAttribute('aria-label', 'Fermer le menu'); // Label mis à jour document.body.style.overflow = 'hidden';  // Bloque le scroll derrière const closeBtn = document.getElementById('mobile-close');
if (closeBtn) closeBtn.focus(); // Focus immédiat sur la croix (accessibilité)
}
function closeMobileMenu(burgerBtn, mobileMenu, mobileOverlay) {
mobileMenu.classList.remove('active');
mobileOverlay.classList.remove('active');
burgerBtn.classList.remove('active');
burgerBtn.setAttribute('aria-expanded', 'false');
burgerBtn.setAttribute('aria-label', 'Ouvrir le menu');
document.body.style.overflow = ''; // Rétablit le scroll normal burgerBtn.focus(); // Retourne le focus au bouton burger
}
Les 4 façons de fermer le menu
Bouton burger / croix
Clic sur #burger-menu ou #mobile-close toggle / close direct
Clic overlay
Clic sur .mobile-overlay closeMobileMenu()
Touche Échap
keydown sur document si Escape && menu.active close
Clic lien nav
.mobile-nav-item click closeMobileMenu() puis navigation
document.body.style.overflow = 'hidden' empêche la page en arrière-plan de scroller pendant que le menu est ouvert. Sans ça, l'utilisateur pourrait scroller la page tout en voyant le menu, créant un effet visuel confus. La remise à '' (chaîne vide) restaure la valeur CSS par défaut.

L'animation CSS des 3 barres → croix

css
/* Barres initiales */
.burger-btn span { display: block; width: 22px; height: 2px;
  background: var(--text); transition: all .3s; }

/* État ouvert — transformation CSS pure */
.burger-btn.open span:nth-child(1) { transform: rotate(45deg) translate(5px, 5px); }
.burger-btn.open span:nth-child(2) { opacity: 0; transform: scaleX(0); }
.burger-btn.open span:nth-child(3) { transform: rotate(-45deg) translate(5px, -5px); }

/* JS ajoute/retire simplement la classe .open */

Le JS ne touche pas aux transform directement — il ajoute/retire la classe .open sur le bouton. Toute la mécanique visuelle est dans le CSS. Avantage : l'animation peut être modifiée sans toucher au JS. L'animation utilise transition: all .3s — les 3 barres bougent simultanément en 300ms.

🚨 Gotcha — Ajout du listener APRÈS que le bouton existe dans le DOM
L'ordre est critique : createBurgerButton() → createMobileMenu() → attachMenuEvents(). Si attachMenuEvents() est appelé avant createBurgerButton(), document.querySelector('.burger-btn') retourne null et le listener est perdu silencieusement.

03 — Geste tactile

attachSwipeToClose() — fermer en glissant vers la gauche ¶

javascript
function attachSwipeToClose(mobileMenu, burgerBtn, mobileOverlay) {
let touchStartX = 0; // Position X mémorisée au début du toucher mobileMenu.addEventListener('touchstart', function(e) {
touchStartX = e.touches[0].clientX; // Premier point de contact }, { passive: true }); // passive: true  n'appelle jamais preventDefault  meilleure perf scroll mobileMenu.addEventListener('touchend', function(e) {
const touchEndX = e.changedTouches[0].clientX;
const deltaX = touchEndX - touchStartX; // Négatif = glissement vers la gauche const THRESHOLD = 60;                   // Seuil min en pixels if (deltaX < -THRESHOLD) {               // < -60 = swipe gauche de plus de 60px closeMobileMenu(burgerBtn, mobileMenu, mobileOverlay);
}
}, { passive: true });
}
Mécanique du swipe — pourquoi deltaX négatif = vers la gauche
Début (touchstart)
touchStartX = 280px (position X du doigt)
Glissement
Le doigt se déplace vers la gauche (X diminue)
Fin (touchend)
touchEndX = 180px · deltaX = 180 - 280 = -100px
Décision
-100 < -60 closeMobileMenu()
Seuil 60px — timeline du swipe de fermeture
touchstart
touchStartX = 280px mémorisé
glissement
le doigt glisse vers la gauche (X diminue)
touchend
touchEndX = 180px · deltaX = 180-280 = -100px
|delta| > 60
closeMobileMenu() — seuil atteint
|delta| ≤ 60
scroll normal, rien ne se passe

Détection des gestes tactiles — touchstart/touchend

Données du geste swipe
touchStartXnumberPosition X du touchstart — mémorisée au début du geste
touchStartYnumberPosition Y — pour distinguer swipe horizontal vs scroll vertical
deltaXnumbertouchEndX - touchStartX — positif = swipe droite, négatif = gauche
thresholdnumber50px minimum pour être reconnu comme swipe intentionnel

Pourquoi 50px comme seuil ?

Un seuil trop petit (10-20px) déclenche sur des micro-mouvements accidentels lors d'un tap. Un seuil trop grand (100px+) demande un geste très prononcé — peu naturel. 50px correspond à environ 1/8 de l'écran d'un smartphone standard — un geste intentionnel mais pas excessif.

javascript
const desktopBreakpoint = window.matchMedia('(min-width: 768px)');
function onBreakpointChange(e) {
if (e.matches && mobileMenu.classList.contains('active')) {
// On passe en mode desktop ET le menu est ouvert  fermer closeMobileMenu(burgerBtn, mobileMenu, mobileOverlay);
}
}
// API moderne : addEventListener sur MediaQueryList
if (desktopBreakpoint.addEventListener) {
desktopBreakpoint.addEventListener('change', onBreakpointChange);
} else {
desktopBreakpoint.addListener(onBreakpointChange); // Fallback Safari < 14
}
MediaQueryList vs window.resize : Écouter window.addEventListener('resize', ...) déclenche un événement à chaque pixel redimensionné (des dizaines de fois par seconde). matchMedia ne se déclenche qu'une fois, au moment exact du franchissement du breakpoint 768px. Beaucoup plus performant.

Gestion du redimensionnement — fermer si desktop

💡
Si l'utilisateur ouvre le menu mobile puis agrandit la fenêtre au-dessus de 768px (breakpoint desktop), le menu resterait ouvert et superposerait la navigation desktop. window.addEventListener('resize', () => { if (window.innerWidth > 768) closeMenu(); }) évite cet état incohérent.

Le listener window.resize utilise un debounce pour éviter d'appeler closeMenu() 60 fois par seconde pendant un redimensionnement. Sans debounce, chaque pixel de redimensionnement déclencherait la callback.

Le resize avec debounce — éviter 100 appels par seconde

❌ Sans debounce (100+ appels/s)
window.addEventListener('resize', function() {
    // Déclenché À CHAQUE pixel de redimensionnement
    // Sur un resize de 200px → 200 appels
    // Chaque appel vérifie le menu et le DOM
    if (window.innerWidth > 768) closeMenu();
});
✓ Avec debounce (1 appel par fin de resize)
let resizeTimeout;
window.addEventListener('resize', function() {
    clearTimeout(resizeTimeout);
    resizeTimeout = setTimeout(function() {
        // Déclenché seulement quand le resize s'arrête
        if (window.innerWidth > 768) closeMenu();
    }, 150); // 150ms après le dernier event
});
Le debounce de 150ms est plus court que celui de la recherche (300ms) car le resize est une action mécanique qui se termine brusquement — pas de délai de réflexion comme la saisie texte.

Fermeture automatique au resize — pourquoi ?

javascript
window.addEventListener('resize', function() {
    if (window.innerWidth > 768) { // Breakpoint desktop
        closeMenu(); // Ferme le menu si on passe en desktop
    }
});
⚠️
Sans ce listener, un scénario problématique : l'utilisateur ouvre le menu mobile → redimensionne la fenêtre en desktop → le menu overlay reste ouvert par-dessus le contenu desktop. Le resize listener garantit que le menu mobile est toujours fermé quand la fenêtre est en mode desktop.

attachFocusTrap() — emprisonner le focus dans le menu ¶

            first.focus(); // Boucle vers le premier
        }
    }
});

Les 10 fonctions et leurs déclencheurs ¶

menu_burger.js crée le menu mobile entièrement en JS : bouton burger, menu overlay, barre de recherche mobile. Tout est généré programmatiquement — rien dans le HTML d'origine.

Les 6 fonctions — séquence obligatoire

createBurgerButton() — Crée et insert le bouton ☰ dans le header. Prerequis : header doit exister dans le DOM.

createMobileMenu() — Crée le menu overlay avec les liens de navigation. Prerequis : DOM prêt.

openMenu() / closeMenu() — Primitives d'animation (translateX). Prerequis : menu créé.

attachMenuEvents() — Attache les listeners sur le bouton burger, les liens, le clic hors zone. Prerequis : bouton ET menu créés.

initMobileSearch() — Clone la barre de recherche desktop pour le mobile. Prerequis : .search-bar présente dans le DOM.

La séquence de DOMContentLoaded — l'ordre est impératif

javascript
document.addEventListener('DOMContentLoaded', function() {
    createBurgerButton(); // 1. Crée le bouton (doit exister avant les listeners)
    createMobileMenu();   // 2. Crée le menu (doit exister avant open/close)
    attachMenuEvents();   // 3. Attache les listeners (bouton ET menu doivent exister)
    initMobileSearch();   // 4. Clone la recherche (optionnel, indépendant)
});
⚠️
Échanger les étapes 2 et 3 causerait une erreur : attachMenuEvents() appelle closeMenu() qui référence le menu DOM. Si le menu n'existe pas encore, closeMenu() tenterait de modifier un null → TypeError. La dépendance séquentielle est explicite ici : créer avant d'utiliser.

Le design en fonctions séparées (plutôt qu'un monobloc) permet de tester chaque étape indépendamment dans la console DevTools : createBurgerButton(), createMobileMenu()… et de diagnostiquer exactement quelle étape échoue si le menu mobile ne fonctionne pas.

Pourquoi chaque bloc est placé là où il est ¶

menu_burger.js suit la règle créer → configurer → écouter. Chaque étape est une condition préalable de la suivante.

1
Bouton burgerfunction createBurgerButton()
Définie en premier — le bouton est le déclencheur du menu. Elle est définie avant createMobileMenu() car l'ordre de définition reflète l'ordre de création : d'abord le déclencheur, puis ce qu'il déclenche. Le check if (document.getElementById('burger-menu')) return au début évite la double création si le script est chargé plusieurs fois.
Appelle : document.querySelector('nav') pour insérer le bouton. Crée : le bouton avec aria-controls='mobile-menu'.
↓ créer le menu après son déclencheur
2
Menu mobile + overlayfunction createMobileMenu()
Après createBurgerButton() — crée ce que le bouton contrôle. Crée l'overlay sombre et le panneau du menu. Les attributs ARIA (role='dialog', aria-modal='true') sont définis ici car ce sont des attributs statiques du composant. attachMenuEvents() attachera ensuite des listeners sur les éléments créés ici — contrainte DOM : créer avant d'écouter.
Crée : #mobile-menu et .mobile-overlay. Ces éléments sont ciblés par openMenu(), closeMenu() et attachMenuEvents().
↓ fonctions de manipulation avant les listeners qui les appellent
3
Ouverture / Fermeturefunction openMenu() + function closeMenu()
Avant attachMenuEvents() qui les passe comme callbacks. openMenu() ajoute la classe active au menu et met à jour aria-expanded. closeMenu() fait l'inverse. Elles sont définies avant attachMenuEvents() pour que celle-ci puisse les référencer directement dans les listeners de clic.
↓ attacher les listeners sur des éléments créés aux étapes 1 et 2
4
Événementsfunction attachMenuEvents() — burger, close, overlay, Échap
Après createBurgerButton() et createMobileMenu() — les éléments doivent exister. Attache les listeners sur le bouton burger, le bouton de fermeture, l'overlay (clic = fermeture) et la touche Échap. Elle référence openMenu() et closeMenu() (étape 3). Dans DOMContentLoaded, elle est appelée après la création des éléments — contrainte DOM absolue.
Référence : openMenu(), closeMenu(). Cible : #burger-menu, #mobile-close, .mobile-overlay — tous créés avant.
↓ recherche mobile — fonctionnalité secondaire
5
Recherche mobilefunction initMobileSearch()
Fonctionnalité secondaire — placée après les comportements principaux (ouverture/fermeture). Gère la barre de recherche dans le menu mobile : frappe → performSearch() si disponible. Le menu peut fonctionner sans elle. Elle est définie en dernier car elle est la moins critique et la plus facilement désactivable.
↓ séquence d'initialisation dans DOMContentLoaded — ordre obligatoire
6
DémarrageDOMContentLoaded → createBurgerButton() → createMobileMenu() → attachMenuEvents() → initMobileSearch()
Séquence obligatoire dans DOMContentLoaded — chaque étape dépend de la précédente. (1) Créer le bouton → (2) Créer le menu → (3) Attacher les listeners sur ces éléments → (4) Initialiser la recherche. Inverser 2 et 3 ferait que attachMenuEvents() ciblerait des éléments qui n'existent pas encore → querySelectorAll('#burger-menu') → null.
La séquence créer → configurer → écouter est invariable. On ne peut attacher des listeners que sur des éléments qui existent dans le DOM.
Règle générale de ce fichier

menu_burger.js crée dynamiquement des éléments DOM plutôt que de les sélectionner. Cette approche impose un ordre strict : on ne peut attacher des listeners sur des éléments créés dynamiquement qu'après leur création. La séquence dans DOMContentLoaded est une contrainte, pas une convention.

Concepts et termes techniques utilisés dans ce fichier ¶

Définitions des termes importants rencontrés dans cette documentation.

Document Object Model
Représentation en mémoire de la page HTML sous forme d'arbre. Chaque balise devient un nœud. JavaScript modifie cet arbre via getElementById(), querySelector(), innerHTML. Modifier le DOM = modifier ce que l'utilisateur voit en temps réel.
Immediately Invoked Function Expression
Fonction qui s'exécute immédiatement à sa déclaration : (function() { ... })(). Crée un scope isolé — tout ce qui est déclaré dedans reste privé et ne pollue pas window. Tous les fichiers JS du projet utilisent ce pattern.
DOMContentLoaded
Événement déclenché quand le HTML est parsé et le DOM prêt — avant images et CSS. Moment idéal pour initialiser les modules JS. Distinct de window.load qui attend tous les assets réseau.
classList
Interface de manipulation des classes CSS. add(), remove(), toggle(), contains(). Plus fiable que modifier className comme une string. toggle('active', condition) ajoute ou retire selon un booléen.
ARIA — Accessible Rich Internet Applications
Attributs HTML pour les technologies d'assistance (lecteurs d'écran). aria-label décrit une action, aria-expanded indique si un menu est ouvert, aria-pressed l'état d'un bouton bascule. Sans ARIA, un lecteur d'écran ne sait pas à quoi sert un bouton sans texte visible.
dataset
Interface qui expose les attributs data-* d'un élément. el.dataset.productId lit data-product-id. Permet de stocker des données métier directement dans le HTML sans variables globales. La conversion camelCase est automatique : data-product-id → dataset.productId.
Bubbling — propagation d'événement
Un événement sur un élément remonte vers ses parents dans l'arbre DOM. Clic sur un <button> dans un <div> → les listeners du <div> se déclenchent aussi. Utilisé par la délégation d'événements. Stoppable avec stopPropagation().
stopPropagation()
Stoppe la remontée (bubbling) d'un événement. Sans ça, un clic sur un bouton qui ouvre un menu déclencherait aussi le listener document.addEventListener('click', fermerMenu). Utilisé pour que le bouton d'ouverture n'annule pas immédiatement l'ouverture.

Menu burger : animation, accessibilité et scroll lock ¶

💡
Analogie — le verrou de scroll : Bloquer le scroll derrière un menu overlay, c'est comme mettre un panneau "Fermé pour travaux" sur la rue derrière une vitrine ouverte. Le menu est au premier plan, le reste de la page ne bouge plus. Sans ça, l'utilisateur scrolle le fond de page en lisant le menu.
💡
Analogie — les trois barres → X : Transformer le burger en × au clic, c'est une convention visuelle universelle. Comme un bouton "Play" qui devient "Pause" — l'icône indique l'action inverse de l'état courant.
🚨 PIÈGE — overflow:hidden sur body qui décale la page
Ajouter overflow: hidden au body pour bloquer le scroll supprime la scrollbar, ce qui peut décaler le contenu de 15–17px. Solution : ajouter padding-right: scrollbarWidth + 'px' au body quand tu bloques, où scrollbarWidth = window.innerWidth - document.documentElement.clientWidth.
🚨 PIÈGE — Ne pas fermer le menu sur resize
Si l'utilisateur ouvre le menu burger sur mobile puis passe en mode paysage (ou redimensionne sur desktop), le menu peut rester ouvert alors que la navigation desktop est affichée. Écouter window.addEventListener('resize', closeMenu) ou utiliser une media query JS pour fermer au-delà d'un breakpoint.
PRO TIP
Piège le focus dans le menu ouvert avec tabIndex et des listeners keydown sur Tab/Shift+Tab. Les utilisateurs clavier doivent naviguer dans le menu sans "s'échapper" vers le fond de page masqué.
PRO TIP
Anime le menu avec transform: translateX(-100%) → translateX(0) plutôt que left: -300px → left: 0. transform utilise le GPU, left déclenche un reflow à chaque frame.

Une classe CSS sur le body ou le container. Tout découle de cet état unique.

Scroll lock

overflow: hidden sur body + compensation scrollbar pour éviter le décalage.

aria-expanded

Annonce l'état du menu aux lecteurs d'écran. true = ouvert, false = fermé.

Fermer sur : clic overlay, touche Escape, clic lien, resize desktop.

    function highlightActivePage() {

        burgerBtn.setAttribute('aria-expanded', 'true');   // Indique l'état ouvert aux lecteurs d'écran
        burgerBtn.setAttribute('aria-label', 'Fermer le menu');
        document.body.style.overflow = 'hidden'; // Bloque le scroll de la page derrière

        // Focus sur le bouton fermer pour une navigation clavier cohérente
        const closeBtn = document.getElementById('mobile-close');
        if (closeBtn) closeBtn.focus();
    }

    function closeMobileMenu(burgerBtn, mobileMenu, mobileOverlay) {
        mobileMenu.classList.remove('active');
        mobileOverlay.classList.remove('active');
        burgerBtn.classList.remove('active');
        burgerBtn.setAttribute('aria-expanded', 'false');  // Indique l'état fermé aux lecteurs d'écran
        burgerBtn.setAttribute('aria-label', 'Ouvrir le menu');
        document.body.style.overflow = ''; // Rétablit le scroll normal de la page

        // Rend le focus au bouton burger après fermeture (bonne pratique clavier)
        burgerBtn.focus();
    }

    function toggleMobileMenu(burgerBtn, mobileMenu, mobileOverlay) {
        if (mobileMenu.classList.contains('active')) {
            closeMobileMenu(burgerBtn, mobileMenu, mobileOverlay);
        } else {
            openMobileMenu(burgerBtn, mobileMenu, mobileOverlay);
        }
    }

    // ============================================
    // SWIPE TO CLOSE (fermeture au glissement)
    // ============================================

    function attachSwipeToClose(mobileMenu, burgerBtn, mobileOverlay) {
        let touchStartX = 0; // Position X au début du toucher

        mobileMenu.addEventListener('touchstart', function(e) {
            touchStartX = e.touches[0].clientX; // Mémorise la position X initiale du doigt
        }, { passive: true }); // passive: true améliore les performances de scroll

        mobileMenu.addEventListener('touchend', function(e) {
            const touchEndX  = e.changedTouches[0].clientX; // Position X finale du doigt
            const deltaX     = touchEndX - touchStartX;     // Déplacement horizontal (positif = vers la droite)
            const SWIPE_THRESHOLD = 60;                     // Seuil minimal en pixels pour déclencher la fermeture

            // Glissement vers la gauche de plus de 60px → ferme le menu
            if (deltaX < -SWIPE_THRESHOLD) {
                closeMobileMenu(burgerBtn, mobileMenu, mobileOverlay);
            }
        }, { passive: true });
    }

    // ============================================
    // GESTION DU RESIZE (fermeture auto en desktop)
    // ============================================

    function attachResizeHandler(burgerBtn, mobileMenu, mobileOverlay) {
        // MediaQueryList : observe le passage mobile ↔ desktop en temps réel
        const desktopBreakpoint = window.matchMedia('(min-width: 768px)');

        function onBreakpointChange(e) {
            // Si on passe en mode desktop ET que le menu est ouvert → fermer proprement
            if (e.matches && mobileMenu.classList.contains('active')) {
                closeMobileMenu(burgerBtn, mobileMenu, mobileOverlay);
            }
        }

        // addEventListener sur MediaQueryList (plus performant qu'écouter window resize en continu)
        if (desktopBreakpoint.addEventListener) {
            desktopBreakpoint.addEventListener('change', onBreakpointChange);
        } else {
            // Fallback pour les navigateurs anciens (Safari < 14)
            desktopBreakpoint.addListener(onBreakpointChange);
        }
    }

    // ============================================
    // SYNCHRONISATION RECHERCHE MOBILE ↔ PRINCIPALE
    // ============================================

    function attachMobileSearch(burgerBtn, mobileMenu, mobileOverlay) {
        const mobileSearchInput = document.getElementById('mobile-search-input');
        const mainSearchInput   = document.getElementById('search-input');

        if (!mobileSearchInput) return; // Champ mobile absent : rien à faire

        if (mainSearchInput) {
            // Synchronise la saisie mobile avec le champ de recherche principal
            mobileSearchInput.addEventListener('input', function() {
                mainSearchInput.value = this.value;
                mainSearchInput.dispatchEvent(new Event('input', { bubbles: true }));
            });

            mobileSearchInput.addEventListener('keydown', function(e) {
                if (e.key === 'Enter') {
                    e.preventDefault();
                    mainSearchInput.dispatchEvent(new KeyboardEvent('keydown', { key: 'Enter', bubbles: true }));
                    closeMobileMenu(burgerBtn, mobileMenu, mobileOverlay); // Ferme le menu après validation
                }
            });
        }
    }

    // ============================================
    // TRAP DE FOCUS (accessibilité clavier)
    // ============================================

    function attachFocusTrap(mobileMenu, burgerBtn, mobileOverlay) {
        mobileMenu.addEventListener('keydown', function(e) {
            if (e.key !== 'Tab') return; // On ne gère que la touche Tab

            // Récupère tous les éléments interactifs focusables dans le menu
            const focusable = Array.from(
                mobileMenu.querySelectorAll('a, button, input, [tabindex]:not([tabindex="-1"])')
            ).filter(el => !el.hasAttribute('disabled') && el.offsetParent !== null);

            if (focusable.length === 0) return;

            const firstEl = focusable[0];                      // Premier élément focusable
            const lastEl  = focusable[focusable.length - 1];   // Dernier élément focusable

            if (e.shiftKey && document.activeElement === firstEl) {
                // Tab+Shift depuis le premier élément → revient au dernier (boucle)
                e.preventDefault();
                lastEl.focus();
            } else if (!e.shiftKey && document.activeElement === lastEl) {
                // Tab depuis le dernier élément → revient au premier (boucle)
                e.preventDefault();
                firstEl.focus();
            }
        });
    }

    // ============================================
    // INITIALISATION PRINCIPALE
    // ============================================

    document.addEventListener('DOMContentLoaded', function() {

        // Création des éléments DOM (dans l'ordre correct)
        createBurgerButton();
        createMobileMenu();

        // Récupération des éléments après création
        const burgerBtn     = document.getElementById('burger-menu');
        const mobileMenu    = document.getElementById('mobile-menu');
        const mobileOverlay = document.querySelector('.mobile-overlay');

        // Sécurité : arrête si les éléments n'ont pas pu être créés
        if (!burgerBtn || !mobileMenu || !mobileOverlay) {
            console.warn('menu_burger.js : éléments introuvables, initialisation annulée');
            return;
        }

        // Mise en évidence du lien de la page active
        highlightActivePage();

        // Attache les comportements
        attachSwipeToClose(mobileMenu, burgerBtn, mobileOverlay);
        attachResizeHandler(burgerBtn, mobileMenu, mobileOverlay);
        attachMobileSearch(burgerBtn, mobileMenu, mobileOverlay);
        attachFocusTrap(mobileMenu, burgerBtn, mobileOverlay);

        // ── Événements principaux ─────────────────────────────────

        // Clic sur le bouton burger : ouvre ou ferme le menu
        burgerBtn.addEventListener('click', function(e) {
            e.stopPropagation(); // Empêche le clic de remonter jusqu'au document
            toggleMobileMenu(burgerBtn, mobileMenu, mobileOverlay);
        });

        // Clic sur l'overlay sombre : ferme le menu
        mobileOverlay.addEventListener('click', function() {
            closeMobileMenu(burgerBtn, mobileMenu, mobileOverlay);
        });

        // Clic sur le bouton croix dans le menu : ferme le menu
        const closeBtn = document.getElementById('mobile-close');
        if (closeBtn) {
            closeBtn.addEventListener('click', function() {
                closeMobileMenu(burgerBtn, mobileMenu, mobileOverlay);
            });
        }

        // Touche Échap : ferme le menu si ouvert
        document.addEventListener('keydown', function(e) {
            if (e.key === 'Escape' && mobileMenu.classList.contains('active')) {
                closeMobileMenu(burgerBtn, mobileMenu, mobileOverlay);
            }
        });

        // Clic sur un lien de navigation : ferme le menu puis navigue
        mobileMenu.querySelectorAll('.mobile-nav-item').forEach(function(item) {
            item.addEventListener('click', function() {
                closeMobileMenu(burgerBtn, mobileMenu, mobileOverlay);
            });
        });

    });

① Ouverture — scroll lock + compensation scrollbar

javascript
function openMenu() {
  // ── 1. Mesurer la scrollbar AVANT de la cacher ───────────────────
  // window.innerWidth inclut la scrollbar.
  // document.documentElement.clientWidth exclut la scrollbar.
  // La différence = largeur de la scrollbar (0 si déjà masquée, 15-17px sinon).
  const scrollbarWidth = window.innerWidth - document.documentElement.clientWidth;

  // ── 2. Compenser le décalage de contenu ──────────────────────────
  // overflow: hidden supprime la scrollbar → le contenu se décale vers la droite.
  // Ajouter padding-right annule ce décalage visuellement.
  document.body.style.paddingRight = scrollbarWidth + 'px';

  // ── 3. Bloquer le scroll ─────────────────────────────────────────
  document.body.classList.add('menu-open');
  // CSS : body.menu-open { overflow: hidden; }
  // L'utilisateur ne peut plus scroller la page derrière le menu.

  // ── 4. Accessibilité ─────────────────────────────────────────────
  burgerBtn.setAttribute('aria-expanded', 'true');
  mobileMenu.setAttribute('aria-hidden', 'false');
  // aria-hidden: false → le menu est visible et lisible par les lecteurs d'écran.

  // ── 5. Piéger le focus ──────────────────────────────────────────
  const firstFocusable = mobileMenu.querySelector('a, button, [tabindex]');
  firstFocusable?.focus();
  // Déplace le focus dans le menu → l'utilisateur clavier navigue dans le menu,
  // pas dans la page derrière.
}

② Fermeture — restaurer l'état

function closeMenu() {
  document.body.classList.remove('menu-open');
  document.body.style.paddingRight = '';
  // Retirer la valeur = revenir à la valeur CSS d'origine (pas "0px").
  // Si on mettait '0px', on écraserait un éventuel padding-right existant.

  burgerBtn.setAttribute('aria-expanded', 'false');
  mobileMenu.setAttribute('aria-hidden', 'true');

  burgerBtn.focus();
  // Retour du focus sur le bouton burger.
  // WCAG : quand un dialogue/menu se ferme, le focus retourne à l'élément déclencheur.
}
