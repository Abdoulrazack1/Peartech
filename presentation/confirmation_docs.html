Comprendre
confirmation.js

Ce fichier gère la page de confirmation de commande. Il lit les données de commande écrites par paiement.js dans sessionStorage, les injecte dans le DOM, nettoie le panier, puis supprime les données de session pour éviter qu'elles s'affichent à nouveau lors d'une prochaine visite.

04 — Nettoyage du panier

Vider le panier après confirmation

Vider le panier après confirmation — pourquoi et comment
Données orderData parsées avec succès
immédiatement après l'affichage
PearTechCart.clear()
localStorage.removeItem('peartech-cart')
sessionStorage.removeItem('peartech-last-order')
Badge panier 0
updateCartBadge() via PearTechCart
Si l'utilisateur recharge la page : panier déjà vide + orderData supprimé fallback générique affiché. C'est le comportement voulu.

Une fois la commande confirmée, le panier doit être vidé. confirmation.js utilise l'API centralisée si disponible, sinon un fallback direct.

javascript
if (window.PearTechCart) {
window.PearTechCart.clear(); // API panier.js  vide + met à jour badge
} else {
localStorage.setItem('peartech-cart-count', '0'); // Fallback direct
}
const badge = document.getElementById('cart-count');
if (badge) {
badge.textContent    = '0';
badge.style.display  = 'none'; // Cache le badge quand il est à 0
}
Le localStorage 'peartech-cart' lui-même n'est pas vidé ici — PearTechCart.clear() s'en charge. En fallback, seul le compteur est mis à zéro. Pour une robustesse maximale, il faudrait aussi faire localStorage.removeItem('peartech-cart') dans le fallback.

Glossaire

Concepts et termes techniques utilisés dans ce fichier

Définitions des termes importants rencontrés dans cette documentation.

Document Object Model
Représentation en mémoire de la page HTML sous forme d'arbre. Chaque balise devient un nœud. JavaScript modifie cet arbre via getElementById(), querySelector(), innerHTML. Modifier le DOM = modifier ce que l'utilisateur voit en temps réel.
Immediately Invoked Function Expression
Fonction qui s'exécute immédiatement à sa déclaration : (function() { ... })(). Crée un scope isolé — tout ce qui est déclaré dedans reste privé et ne pollue pas window. Tous les fichiers JS du projet utilisent ce pattern.
DOMContentLoaded
Événement déclenché quand le HTML est parsé et le DOM prêt — avant images et CSS. Moment idéal pour initialiser les modules JS. Distinct de window.load qui attend tous les assets réseau.
sessionStorage
Identique à localStorage mais limité à l'onglet courant. Effacé à la fermeture de l'onglet. Utilisé ici pour transmettre la commande de paiement.js vers confirmation.js sans la persister.
JSON — JavaScript Object Notation
Format texte pour sérialiser des objets. JSON.stringify(obj) → string stockable. JSON.parse(str) → objet utilisable. Obligatoire pour stocker/lire des objets dans localStorage ou sessionStorage.
